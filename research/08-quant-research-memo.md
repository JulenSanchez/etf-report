## 1. 参数探索方法论

默认路径：**Stratum → Cascade**。

| 场景 | 推荐策略 | 用法 |
|---|---|---|
| **自动化统一优化** | `quant_optimizer.py` | 一条命令替代手工 sweep, 支持 grid/random/bayesian (Optuna TPE) |
| 改模型后首次探索 | Stratum → Cascade | 先逐因子识别敏感维度，再对敏感维度细扫 |
| 时间充裕、维度低 | Full Grid | 参数 ≤ 3 时可全量扫 |
| 单因子行为确认 | One-Factor Sweep / Ridge | 固定其他参数，深扫目标因子 |
| 从冠军参数微调 | Hill-Climb / Ridge | 从当前最优点沿各维小步扰动 |
| 地形未知、多峰 | Funnel | 随机撒点 → top-K 周边补点 |
| 指定收益/回撤范围 | Target Band Filter | 粗扫后筛满足目标带的组合 |
| 找弱点和崩溃区 | Adversarial Search | 反向按最差窗口、最大回撤等排序 |

统一优化器用法见 `runbooks/QUANT_RUNBOOK.md` §6。

每次正式搜索应记录：策略、采样效率、是否漏峰、是否过慢、用户反馈和新适用场景。

## 3. 信心函数与市场状态框架

### 3.1 dz/fz 分数信心函数的问题

旧 `confidence_function(score, dz, fz)` 效果弱：Top-6 综合分集中在窄区间，常常超过 `full_zone`，导致仓位几乎长期满仓。分数适合决定“买什么”，不适合单独决定“买多少”。

### 3.2 regime_confidence 试验

2026-05-06 曾验证基于组合 NAV 推断 regime 的仓位控制：

- `bull=0.95 / choppy=0.75 / bear=0.35`
- `regime_window=8`
- `regime_threshold=0.03`

结果：3 年最大回撤由约 -25% 降到约 -17%，但年化收益下降明显。结论是：市场状态控制仓位有价值，但 NAV 驱动容易循环引用，后续更适合作为被替代的过渡方案。

### 3.3 统一市场信号框架

后续策略转向外部市场信号驱动仓位：

- 用沪深 300 趋势信号决定整体仓位，而不是用自身 NAV。
- 高默认仓位 + 少触发 + 避免踏空。
- 早期验证中，`ma_trend` 比 NAV 驱动或纯分数映射更清晰。

当前经验：选股质量往往比频繁择时更重要；仓位控制应尽量依赖所有人都能观测到的市场客观信号。

## 4. MA Trend 参数优化记录

2026-05-08 的 3 年 coarse sweep 结论：

| 参数 | 优化方向 |
|---|---|
| MA Period | 20 → 26 左右表现更好 |
| Bull Position | 维持高仓位，100% 附近表现强 |
| Bear Position | 40% → 30% 可明显降回撤 |
| Direction Confirm | 开启后提升最大 |

代表结果：

| 指标 | 默认 MA20/B100/B40/DirOFF | 优化 MA26/B100/B30/DirON |
|---|---:|---:|
| Calmar | 0.92 | 1.55 |
| 年化收益 | 22.5% | 24.8% |
| 最大回撤 | -24.4% | -16.0% |

经验：

- `Direction Confirm = ON` 是主要提升来源。
- MA 最优区域大致在 26-40。
- MA=26 更进取，MA=40 更保守。
- MA=40 的部分区间尚未完整细扫，后续如重启优化应优先补。

回测执行经验：

- CodeBuddy 长时间后台任务可能被回收；长 sweep 应采用小批次。
- batch size 建议 ≤ 5。
- 每个 combo 立即写 checkpoint。
- API timeout 不宜过长，优先让脚本可中断、可续跑。
- Tuner 进程挂掉是常态，脚本检测后保存 checkpoint 并优雅退出即可。

## 5. 实盘成交时点过渡

当前执行框架：

- 冷却期从 60 分钟缩短到 10 分钟，15:10 后可拉取当日数据。
- 当前过渡使用 `next_open`：收盘信号 → 次日开盘执行。
- 2026-07-06 后，盘后固定价格交易扩展到全部 A 股 + ETF，可评估切换到 `same_close`：收盘后生成信号 → 盘后以收盘价成交。

三种执行口径：

1. 14:50 盘中信号 + 尾盘成交：反应快，但有信号偏差和成交价偏差。
2. 收盘后信号 + 次日开盘执行：最保守，无信号偏差，但有隔夜跳空。
3. 收盘后信号 + 盘后收盘价成交：理论上最贴近回测，但依赖券商支持盘后固定价格交易。

当前阶段优先 `next_open`；若未来接入 miniQMT，可再评估盘中/尾盘增强。

待开发方向：在 Tuner 页面增加纯展示型预警。

触发场景：

- HS300 当前在 MA 上方，但盘中实时价跌破 MA。
- HS300 当前在 MA 下方，但盘中实时价突破 MA。

约束：

- 只做展示，不进入回测逻辑。
- 不触发实际仓位切换。
- MA 趋势的核心价值是滞后过滤噪声，不能让盘中假突破污染决策。
- 展示口径是“如果今天收在当前价位，MA 信号会怎样变化”。

## 7. ETF 池与回测评价改进假设

2026-05-12 用户参数测试后的待验证方向：

- 日频在体感上显著优于周频，可能得益于更快反应；后续应在执行约束下继续验证。
- 后视镜看，剔除近年持续弱势标的可改善表现，但不能直接规则化。
- 现有 ETF 池覆盖不足，可能漏掉 2024 年初红利股走强、4 月小反弹等热点。
- 回测评价应增加逐 ETF 盈亏归因，识别“白送钱”的交易。
- 债市在 2024 年股市走熊期间走牛，应寻找满足规模、成交额、波动率要求的债券 ETF 候选。
