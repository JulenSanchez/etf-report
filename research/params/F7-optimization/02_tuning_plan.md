# F7 参数调优方案 — 优化 2026-01 逃顶能力

日期：2026-05-13
目标：找到 F7 子参数组合，使其在 2026-01 顶部（卫星 ETF 1月13日、有色 1月29日）的逃顶能力接近或超越 F6

## 一、问题诊断

### 1.1 F7 逃顶滞后的根因

F7 的 Z-score 计算基于 **20 日累计对数收益**：
```
Z = (当前 20 日累计 log_return - μ) / σ
```

**问题**：20 日窗口太长，Z 值变化滞后：
- 行情顶部往往是 **5-10 日** 快速暴涨
- 20 日累计值会被前期的低值"拖慢"，Z 值到达极端需要时间
- 等 Z > +2σ 时，顶部已经过去了

对比 F6：RSI>80 + 放量下跌 2.5% 是 **当日触发**，滞后 0-1 日。

### 1.2 当前 F7 参数

| 参数 | 当前值 | 作用 |
|---|---|---|
| `f7_window` | 20 | 累计对数收益的窗口 |
| `f7_t` | 7 | 幂次，控制两端加速程度 |
| `f7_k` | 3.0 | Z-score 阈值，\|Z\|>k 后开始加速惩罚 |
| `w7` | 0.15 | F7 权重 |

## 二、调优方向

### 方向 A：缩短 `f7_window`（提高对近期暴涨的敏感度）

| f7_window | 效果 |
|---|---|
| 20（当前） | 滞后，20 日累计太平滑 |
| **10** | 对 1-2 周暴涨更敏感，Z 值更快到达极端 |
| **5** | 非常敏感，但可能引入噪声 |

**预期**：f7_window=10 时，2026-01 顶部的 Z 值会提前 3-5 日到达 +2σ，F7 提前减仓。

### 方向 B：降低 `f7_k`（让惩罚更早启动）

| f7_k | 效果 |
|---|---|
| 3.0（当前） | \|Z\|>3 才开始加速惩罚，阈值太高 |
| **2.0** | \|Z\|>2 就开始加速，提前反应 |
| **1.5** | 更激进，但可能误触发 |

**预期**：f7_k=2.0 时，F7 在 Z=+2 时已经有明显的低分输出（~0.05），接近 F6 的触发效果。

### 方向 C：提高 `f7_t`（让极端值惩罚更猛烈）

| f7_t | 效果 |
|---|---|
| 7（当前） | 两端加速适中 |
| **11, 15** | 极端值惩罚更猛烈，Z=+3 时分数更接近 0 |

**预期**：更高的 t 让 F7 在极端值区域更像"开关"，接近 F6 的离散触发行为。

### 方向 D：提高 `w7`（增强 F7 影响力）

| w7 | 效果 |
|---|---|
| 0.15（当前） | F7 影响力有限 |
| **0.25** | F7 影响力接近 F1/F3 |
| **0.35** | F7 主导，但可能过拟合 |

**可接受情况**：F7 在全区间收益仍落后 F6，但在顶部防守明显优于当前 F7（MDD 从 -12.94% 优化到 -10% 左右）。
