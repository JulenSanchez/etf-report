# 动态 C (CS) — 共识感知仓位分配

> **状态**: ✅ Promoted (2026-05-25)  
> **Preset**: preset1, C=0.5, CS=10  
> **基线**: 6Y +910.0%, MDD -17.9%（2020-05-27~2026-05-25, vs 旧静态 C +528.0%, MDD -16.3%）  
> **数据**: `cs_grid_search.json`（42 组 C×CS 网格）

---

## 〇、思考链：从后视镜到动态 C

这条线索的起点是一个持续了数周的问题——**"我们离最优策略还有多远？"**

### 后视镜研究（REQ-189）

2026-05-07，首次后视镜研究：构造上帝视角策略（提前知道明天的涨幅），测量理论天花板。当时发现信息效率仅 6.5%——策略只提取了天花板的 6.5%。

2026-05-25，后视镜重跑（v2）：44 支 ETF 池，6 年数据。核心发现——**V2（上帝视角日频 Top-6）和 preset1 的日均选股重叠仅 1.5/6。** 上帝视角每天选的和策略每天选的，差异巨大。

### 转折点：重叠在什么时候高？

进一步分析发现：重叠不是均匀分布的。在"强者共识明确"的交易日（某支 ETF 的分数碾压式领先），重叠显著更高。**这不只是"策略有时选错"——这说明策略在信号清晰的时候其实选得不错，问题在于"选对之后给了多少仓位"。**

### 从观察到参数

如果共识强的日子 = 策略选得准的日子，那在这些日子里理应给更多仓位。但怎么量化"共识强"？Top-6 的 Z-score 标准差（σ_top6）恰好就是共识度的代理变量。这个量一直存在于回测引擎中，只是从未被使用。

CS 参数的名字由此而来：Consensus Sensitivity——对共识的敏感度。

### 为什么是这条线而不是别的

后视镜研究在需求看板里是 REQ-189，早已标记 done。但它作为"理解策略天花板"的元问题被反复惦念——在策略优化、池子扩容、贡献度分析的间隙中，始终有一条暗线在追问"到底差在哪"。CS 是这条暗线浮出水面的第一个答案。

## 一、动机

（承上）后视镜 V2 的分析直接指向了 CS 的核心假设：**信号强弱不是均匀分布的——有些日子策略"非常确定"该买什么，有些日子只是勉强排出前 6。** 静态 C 把这个差异抹平了——无论信号多清晰，仓位分配的集中度永远相同。如果能区分"强信号日"和"弱信号日"，在强信号日更激进地集中仓位，理论上能捕获更多 alpha。

---

## 二、从直觉到公式

### 2.1 信号共识度的代理变量：σ_top6

每期回测对全池 44 支 ETF 的综合分做 Z-score 标准化后，取 Top-6 的 Z-score，计算它们的标准差 σ_top6。

```
σ_top6 = std(z_scores of top-6)
```

**直觉**：σ_top6 大 → Top-6 内部差距大 → 有一支 ETF 显著优于其他 → "共识强"。σ_top6 小 → Top-6 分数非常接近 → 谁更好只是噪声 → "共识弱"。

实测 6 年 1457 个交易日的 σ_top6 分布：
- 均值 0.55，中位数 0.52
- 范围 0.04（几乎等分）~ 1.81（一骑绝尘）

### 2.2 公式

```
c_mult  = 1 + CS × (σ_top6 − 0.5)
C_eff   = C_base × max(c_mult, 0.1)
```

- CS=0：退化为静态 C
- σ_top6 > 0.5：c_mult > 1 → 放大 C（更集中）
- σ_top6 < 0.5：c_mult < 1 → 缩小 C（更分散）

### 2.3 为什么要减 0.5？

0.5 是 σ_top6 的 6 年实测中位数（0.52）。减去 0.5 的效果是：**一半的交易日 c_mult > 1（放大），一半 < 1（缩小）。** 这不是物理常数，是数据的中性点。

**更深层的含义**：σ_top6 本身是横截面统计量（仅看当天的 44 支 ETF），但 0.5 作为校准基准携带了 6 年历史信息。因此 CS 公式隐式地实现了**横向（截面分数差）× 纵向（历史均值校准）的二维效果**——CS 不直接看历史数据，但通过 0.5 基准间接感知了"今天的信号在历史上算不算强"。

## 三、核心洞察：CS ≠ 加强版 C

### 3.1 CS 和 C 不是"叠加"，是"正交"

一个常被误解的点：C 和 CS 都能产生差异化的仓位分配——这不是 CS 独有的能力。如果你把 C 从 0.5 调到 5，游戏 ETF 同样能被推到 37%。**两者的共同点是"都能拉开仓位差"。**

关键区别在于**它们凭什么拉开**：

| | 静态 C | 动态 CS |
|------|------|------|
| 决策依据 | 策略持有者的事前信念（"我相信集中更好"） | 历史数据（1457 个交易日的 σ 分布） |
| 生效范围 | 每时每刻——包括 σ=0.11 的熊市中继日 | 仅在 σ 显著偏离历史中位数的时刻 |
| 信息维度 | **横截面**：ETF 之间的分数差距 | **纵截面**：今天的信号清晰度在历史上算什么水平 |
| 代表什么 | 人的认知——一种策略哲学 | 数据的涌现——让历史自己说话 |

**CS 没有增加策略持有者的信念权重，反而削弱了它。** 策略持有者不再需要决定"我应该多集中"——这个决定被外包给了 0.5，一个从 6 年历史中自然涌现的中位数。

### 3.2 二维 vs 一维：面积类比

如果把策略的"优化预算"看作一个增量，可以分配到两个正交维度上：

- **C（长）**：横截面维度——"ETF 之间的差距放大多少"
- **CS（宽）**：时间维度——"今天的信号值不值得放大"

**全部增量给 C（一维堆高）**：C=5，6Y +966%，MDD -33.3%
**分给 C+CS（二维展开）**：C=0.5, CS=10，6Y +873%，MDD -17.9%

同样的"优化力度"，二维展开用更少的 MDD 换来了几乎同等的收益。就像矩形面积 = 长 × 宽——把全部增量给长度，不如分一半给宽度。乘积更大。

一维堆高 = 摩天楼，风一吹就晃。二维展开 = 金字塔，基座宽了自然更稳。

### 3.3 0.5 的哲学：压缩的历史

CS 公式中的 `σ − 0.5`，0.5 看似随意，实则是对 6 年市场状态的一种**无损压缩**。它等价于在问："今天的 σ 是否超过了历史中位数？"

这个减法让 CS 拥有了**隐式的纵向感知能力**——它不需要显式地存储任何历史数据，却能判断今天处于历史的什么位置。这就是你所说的"二维信息"涌现的机制：0.5 作为压缩表示，让一个纯横截面统计量（σ_top6）携带了时间维度的信息。

## 四、网格搜索与最优解

### 4.1 MDD<20% 约束下的最佳组合

| C | CS | 1Y MDD | 3Y MDD | 6Y MDD | 6Y 总收益 |
|---|-----|------|------|------|------|
| 0.5 | 0 | -16.3% | -16.3% | -16.3% | +528% |
| **0.5** | **10** | **-17.9%** | **-17.9%** | **-17.9%** | **+910%** |
| 1.0 | 5 | -19.0% | -19.1% | -19.1% | +1030% |
| 0.5 | 20 | -20.2% | -20.2% | -20.2% | +1100% |

**C=0.5, CS=10 被选为 preset1**：在所有窗口（1Y/3Y/6Y）下 MDD 稳定在 -17.9%，比 C=1.0 低 1.2%，比 CS=20 低 2.3%。

### 4.2 CS 的渐近线

```
CS=5:  +688%
CS=10: +873%   (+185)
CS=20: +1100%  (+227)  
CS=30: +1247%  (+147)
CS=50: +1366%  (+119)
CS=100:+1367%  (+1)     ← 天花板
```

CS 在 50 附近触及渐近线 ~1400%。线性公式在极端值下趋向二元开关——σ 大时全仓第一名，σ 小时等权分散。突破 1400% 需要公式升级（如 sigmoid 替代线性）。

### 4.3 为什么 C=0.5 而非 C=1.0

用户主观偏好 MDD<20%。C=1.0 在 1Y 窗口 MDD=-19.0%，虽然仍在范围内，但 C=0.5 更保守且各窗口 MDD 一致性更好。最终选择 C=0.5 保持原 preset1 的风险特征，CS 负责提升收益。

## 五、信号稳定性验证

CS=10 是否导致过度换手？对比相邻交易日 Top-6 持仓的 Jaccard 相似度和加权仓位重叠：

| | Dynamic CS=10 | Static CS=0 |
|------|------|------|
| Jaccard (选股重叠) | 0.83 | 0.87 |
| 加权重叠 | 0.75 | 0.82 |
| 换仓次数 (6Y) | 916 | 746 |
| 高换手日 (J<0.5) | 9.7% | 8.9% |

CS 仅增加 0.8% 高换手率，选股一致性保持 83%。额外 170 次换仓分散在 1452 天中，是信号驱动的有意义的仓位调整而非噪声交易。

## 六、交易行为层面的观察

动态 CS 最令人印象深刻的表现发生在 2023-03-16 至 2023-05-04：

- 策略重仓传媒和游戏（AI 应用端启动 + 游戏版号重启）
- σ_top6 在该区间飙升，C_eff 被推到 ~5，近乎满仓押注
- 净值在该区间"跃迁式"拉开与所有基准的差距
- 随后又精准逃顶——因子分数分化回落，仓位自动分散

这不是"选对了方向"——静态 C=0.5 也选了传媒和游戏，但它只给了 ~18% 和 ~15% 的仓位。**CS 在"该贪的时候贪了"，这是核心差异。**

## 七、未来方向

1. **0.5 基准值改为滚动窗口中位数**：当前固定 0.5 对全历史均值校准，滚动窗口（如 252 日）能让 CS 感知近期市场状态——熊市中 σ 普遍低，σ=0.6 也算"熊市中难得的强信号"。
2. **公式升级为 sigmoid**：线性 CS 在极端值趋向二元开关，sigmoid 可提供更平滑的映射。
3. **牛熊状态感知 CS**（🟢 低优先级研究项）：对比数据显示静态 C 在熊市月份偶尔回赢动态 C（差距很小，<3%）。暗示熊市中"共识"的可靠性天然更低——可以考虑让 CS 随市场状态连续调节（而非 MA Trend 式的二元跳变）。但当前不考虑在 MA Trend 的二值信号上堆叠新逻辑，避免硬编码耦合。等未来有更优雅的连续型牛熊判断机制后再议。
4. **CS 与 MA Trend 的交互**：当前总仓位由 MA Trend 控制（二值），CS 控制内部分配。两者独立运行，理论上可联动。
