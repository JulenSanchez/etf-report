# Walk-Forward 优化 — REQ-223

> **日期**: 2026-05-27
> **脚本**: `scripts/quant_walkforward.py`
> **数据**: `results.json`

---

## 一、方法

模拟实盘场景：每 6 个月用过去 2 年数据重选最优 (C, CS)，应用到接下来 6 个月。

| 参数 | 值 |
|------|-----|
| Anchor（训练窗） | 2Y |
| Step（步进） | 6M |
| OOS（测试窗） | 6M |
| 搜索空间 | C ∈ [0.1, 0.2, ..., 1.0] (10 点) × CS ∈ [0, 0.5, ..., 3.0] (7 点) |
| 优化目标 | Anchor 窗口 Sharpe 最大化 |
| 对照 | Static preset1 (C=0.5, CS=10) |

共 8 步 × 70 组合 = 560 次回测。

## 三、关键发现

### 1. CS 持续顶到上限

8 步中 5 步 CS=3.0（搜索上限），说明 2Y anchor 优化偏好 CS ≥ 3.0。这与用户独立发现（1Y Sharpe 最优在 CS≈0.6）存在窗口长度差异——更长的训练窗可能偏好更激进的集中度。

### 2. WF00 翻车

第一步训练窗（2020-2022，COVID 复苏牛市）选出的最优参数（C=0.2, CS=2.5），在接下来的 6M OOS 直接亏损 -5.2%。训练窗口的环境与 OOS 窗口（2022H2 中概股崩盘 + 清零）完全不同。

### 3. C 极不稳定

C 从 0.2 到 1.0 来回跳，无收敛趋势。说明在 2Y 尺度上，最优 C 高度依赖锚定窗口的具体市场环境。

### 4. 锚定 Sharpe 很低

多数 anchor 的 Sharpe < 1.0（最低 0.36），远低于全期 preset1 的 1.93。2Y 短窗口本身 Sharpe 波动就大，用它作为优化目标未必可靠。

## 四、结论

**"每半年重优化 C 和 CS"不靠谱。** 2Y anchor 选出的最优参数对 6M 未来的预测力很弱（WF00 直接亏钱），参数本身也极不稳定。

这不等于"静态参数就是最优"——它只说明**短期 WF 优化无法改善长期表现**。静态 preset1 之所以成立，正是因为它的参数是在 6 年全史上校准的，而短期锚定窗口缺乏这种跨周期稳定性。

## 五、对后续的影响

- WF 方法本身有价值（模拟实盘不偷看未来），但优化目标（Sharpe）和锚定窗口（2Y）可能需要调整
- 用户提出的 Sortino + 总收益复合指标可能比 Sharpe 更适合作为优化目标
- REQ-233（因子归因）更适合下一步：理解因子在不同市场环境下的驱动力，比盲目优化参数更有意义
