# ETF 报告生成工作流 (etf-report)

## 概述

自动分析与生成 6 支 ETF 的投资分析报告，包括：
- K线数据（60日日线 + 52周周线）
- 实时行情数据
- 技术分析（MA均线）
- 成分股配置
- 宏观环境分析

生成的报告采用**数据+模板分离架构**，100% 保持原始 HTML 样式一致。

## ⚙️ 自动化任务规则

> **重要**：此规则适用于 CodeBuddy 定时任务/自动化执行场景

当此 Skill 被**定时任务**或**自动化系统**触发时：
1. **必须使用发布模式** (`--publish` 参数)
2. 执行命令：`python scripts/update_report.py --publish`
3. 完整执行所有步骤，包括：
   - ✅ 企微通知推送
   - ✅ GitHub Pages 部署

当由**用户手动触发**（如说"更新ETF报告"）时：
1. **默认使用开发模式**（不带 `--publish`）
2. 只更新本地 HTML 文件（根目录 `index.html`）

**判断规则**：
- 如果是定时任务/自动化运行 → 发布模式
- 如果是用户对话触发 → 开发模式（除非用户明确要求发布）

## 架构设计

### 核心原则

**数据+模板分离**：
- 根目录 `index.html` 作为展示入口（GitHub Pages + 本地预览）
- 脚本自动更新 HTML 中的动态数据部分
- 保证 100% 样式一致性

### 文件结构

```
etf-report/
├── SKILL.md                         # 本文件（工作流说明）
├── index.html                       # 根目录主报告（发布入口）
│
├── scripts/                         # 执行脚本
│   ├── update_report.py            # 主控脚本（入口）
│   ├── fix_ma_and_benchmark.py     # K线数据获取
│   └── realtime_data_updater.py    # 实时行情获取
│
├── data/                           # 数据文件
│   ├── etf_full_kline_data.json
│   └── etf_realtime_data.json
│
├── reports/                        # 分析报告（Markdown）
│   ├── 宏观环境与风险偏好建议.md
│   └── {code}_{name}_分析报告.md
│
└── config/                         # 配置文件
    ├── config.yaml
    └── holdings.yaml
```

## 注意事项

1. 建议在**交易日收盘后（15:00之后）**执行更新
2. 非交易日无需更新
3. MA均线从第一天即有完整数据（已预热19天）
4. 基准指数线默认隐藏，可点击图例显示
5. **预览报告时使用 `file://` 协议 + 绝对路径，不要启动 localhost 服务器**

## 成分股配置

成分股代码在 `scripts/realtime_data_updater.py` 的 `ETF_CONFIG` 中维护，需每季度根据ETF季报更新。
