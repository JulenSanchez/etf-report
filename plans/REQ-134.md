## 动机

当前仓库对外目标不是暴露内部开发治理，而是让他人 clone 后可以从 0 跑完整更新流程。

现状存在两个阻塞：
- `README.md` 指向 `requirements.txt`，但仓库中缺失依赖声明文件。
- `config/config.yaml` 仍承载个人机器路径与发布环境，直接上传会泄露私有配置，也不利于他人迁移为自己的配置。

## 实施步骤

### Step 1: 依赖声明补齐
- 新增 `requirements.txt`
- 明确主流程与测试所需依赖，保证首次安装路径自洽

### Step 2: 配置模板化
- 让 `config/config.example.yaml` 成为公开模板
- `config/config.yaml` 改为本地覆盖配置，不再作为公开默认事实源
- 新增 `config/secrets.example.yaml`，把 webhook 等敏感配置迁移到私有文件

### Step 3: 文档与忽略规则同步
- 更新 `README.md` 和必要导航文档
- 更新 `.gitignore`
- 从 Git 跟踪中移除 `config/config.yaml`

## 文件影响分析

| 文件 | 改动类型 | 改动说明 |
|------|---------|---------|
| `requirements.txt` | 新增 | 补充运行与测试依赖 |
| `scripts/config_manager.py` | 修改 | 支持 example 模板 + 本地覆盖 |
| `config/config.example.yaml` | 修改 | 补充可公开默认配置和发布占位字段 |
| `config/secrets.example.yaml` | 新增 | 提供敏感信息模板 |
| `.gitignore` | 修改 | 忽略本地私有配置 |
| `README.md` | 修改 | 补充首次配置与运行步骤 |
