# REQ-369: F7 Checkpoint 胜率优化

**优先级**: 🟡 P1 | **状态**: 讨论中 | **目标版本**: v3.13.0

## 背景

REQ-367（CAPM 残差 F7）被证伪，REQ-368（共识门控 damp）也被证伪。但 F7 在个股异动检测中表现优秀（纳指抄底、卫星逃顶、有色逃顶、通信逃顶），核心能力已验证。

当前讨论方向：F7 归一化后的 Z 值分布可能存在关键阈值（checkpoint），在这些阈值上的调仓决策胜率更高。如果能系统性地发现和验证这些 checkpoint，可以作为 F7 参数的辅助调优指标，而非取代 F7 本身。

## 探索方向

1. 对 F7 Z-score 做分段胜率统计（Z ∈ [0,1), [1,2), [2,3), ...），找出胜率跳变的阈值
2. 验证 checkpoint 在不同市场环境（牛/熊/震荡）下的稳定性
3. 如果 checkpoint 存在且稳定，考虑将其作为 f7_t / f7_k 的调参约束

## 备注

- 不改 F7 公式，不新增参数
- 纯研究型需求，产出为数据分析报告
- 如果 checkpoint 不存在或不稳定，可快速关闭
