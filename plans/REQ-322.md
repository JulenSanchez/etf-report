# REQ-322: 回测性能优化

**优先级**: 🟡 Normal | **目标版本**: v4.0.0 | **依赖**: 无 | **创建**: 2026-06-26

## 现状

cProfile 实测（6天回测窗口，CSV 含 6Y 数据）：

```
总耗时: 50.5s
  _precompute_factors: 40.9s (80%)  ← Python for loop × 54 ETFs × ~1500 rows
  price_lookup iterrows: 6.7s (13%) ← pandas 反模式
  主循环:                2.8s ( 6%) ← 已经很轻
```

53M 函数调用，其中 pandas `Series.__init__`、`iterrows`、`__eq__` 占大头。

## 优化项（按风险排序）

| # | 优化 | 方法 | 预计 | 风险 |
|---|------|------|------|------|
| 1 | 缓存因子 | pickle 到 `data/quant/.factor_cache`，CSV hash 校验 | 40s→0.5s | 低 |
| 2 | 去 iterrows | `zip` + `itertuples` 替代 | 6.7s→0.7s | 低 |
| 3 | 向量化 | numpy `where`/`diff` 替代 Python for loop | 40s→5s | 中 |
| 4 | 3Y 窗口 | 优化时用 3Y 而非 6Y | 全减半 | 无（规范允许）|

## 验证规则

每条优化独立验证，前后对比相同输入的回测耗时：

```
验证命令:
  python -c "
  import time, sys; sys.path.insert(0,'scripts')
  from quant_backtest import run_backtest
  t0=time.time(); run_backtest(start_date='2025-06-26',preset='gam-2'); print(f'{time.time()-t0:.1f}s')
  "

验证条件:
  - 相同 preset (gam-2)
  - 相同 start_date (2025-06-26)
  - 相同数据（CSV 未变）
  - 结果 TR/MDD 与优化前一致（数值回归）

通过标准:
  #1: 总耗时 < 15s (从 50s)
  #1+#2: 总耗时 < 10s
  #1+#2+#3: 总耗时 < 5s
```
