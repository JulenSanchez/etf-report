# REQ-277: F1 检查点/冻结点机制

## 需求来源

前序 f1_active_days 重构引入了 bitmask 参数化，但多 bit 场景下存在逻辑 bug：

- **`w_end` 查找冗余**：Base 模式用"本周一"做参考点、其他模式用"今日"，本质相同，应统一为"本周一"——永远取上周最后一个完整周 bar 作为 base
- **`>=` 应为 `==`**：当前代码 `days_in_week >= threshold` 导致门槛日之后的每一天都重新滚 EMA。正确行为：**只有门槛日当天**创建检查点，之后的中间日冻结在上一个检查点
- **缺少跨日冻结状态**：没有 `checkpoint_f1` 变量追踪本周当前活跃的检查点

## 现状 vs 目标

```
场景: f1_active_days=9 (周二+周五)

当前（错误）:
  周一hold  周二compute  周三compute  周四compute  周五compute
  上周F1   周二价→EMA   周三价→EMA   周四价→EMA   周五价→EMA
           ↑            ↑            ↑
           周三周四本应冻结，却在自由移动

目标（正确）:
  周一hold  周二检查点✓  周三冻结     周四冻结     周五检查点✓
  上周F1   周二价→EMA   复用周二     复用周二     周五价→EMA
           F1冻结在此   F1不变      F1不变      新F1覆盖
```

### Test 2: 逐日冻结行为——单 ETF 短窗口

**目的**：肉眼验证周二+周五模式下，周三周四的 F1 值是冻结的

| 项目 | 值 |
|------|-----|
| ETF | 515220（煤炭ETF，有完整数据） |
| 窗口 | 2026-03-30 ~ 2026-04-10（10 个交易日，跨度 2 周） |
| 模式 | f1_active_days=9（周二+周五） |

**通过标准**：

假设第 1 周（3/30 周一 ~ 4/3 周五）：

```
日期      星期   days_in_week  应输出
3/30     一     1             base_F1（上周五的完整周 F1）
3/31     二     2             检查点★（周二价滚 EMA 后的 F1，值 X）
4/01     三     3             冻结 = X（与周二完全相同）
4/02     四     4             冻结 = X（与周二完全相同）
4/03     五     5             检查点★（周五价滚 EMA 后的 F1，值 Y，Y≠X）
```

- [ ] 3/31 (周二) F1 值 X ≠ base_F1（证明抢跑生效）
- [ ] 4/01 (周三) F1 值 == X（冻结，与周二完全相同）
- [ ] 4/02 (周四) F1 值 == X（冻结，与周二完全相同）
- [ ] 4/03 (周五) F1 值 Y ≠ X（新检查点覆盖旧检查点）
- [ ] 3/30 (周一) F1 值 == base_F1（未到任何检查点，hold 上周）
