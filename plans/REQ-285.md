# REQ-285: GitHub Pages 发布面二期迁移

**优先级**: 🟡 Normal  
**估时**: 4-8h  
**状态**: backlog  
**目标版本**: v4.0.0  
**依赖**: REQ-281

---

## 动机

REQ-281 为了避免线上报告断站，保留了根目录 `index.html` / `assets/` 作为 GitHub Pages 发布面。这是安全过渡方案，但长期看源码目录与发布产物仍混在一起。

## 实施范围

- 评估 GitHub Pages source 改为 `/docs`、GitHub Actions Pages 发布 `web/` 或 `dist/` 的可行性。
- 将源码目录与发布产物目录彻底分离。
- 改造 `deployer.py` 为显式 build → deploy 流程。
- 迁移前确认远端 Pages 设置，避免线上报告断站。

## 验证方式

| 验证项 | 方法 | 通过标准 |
|---|---|---|
| 本地构建 | 执行 build/deploy dry-run | 产物完整 |
| Pages 设置 | 人工确认 GitHub Pages source | 指向新发布面 |
| 线上验证 | 打开 GitHub Pages | 报告可访问且 payload 正常 |
