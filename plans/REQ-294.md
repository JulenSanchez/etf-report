## 动机

全链路审计发现 `optimization_analyzer.py` 存在硬编码参数列表 bug（新增参数不会被分析），`optimization.md` 存在 section 编号重复、单位转换未文档化、缺少 `--lock` 机制。分层优化（信号层/执行层）落地前必须先修这些。

## 问题清单

### 🔴 Bug: analyzer 硬编码参数列表

`scripts/optimization_analyzer.py`:

- **Line 161** (`top_bottom_divergence`): `for param in ['w1', 'w3', 'max_holdings', 'disc_step', 'vol_window', 'dead_zone', 'f7_t', 'ma_bear_pos']:`
  - `vol_window` 在 PARAM_BOUNDS 中不存在 → 正确名字是 `f3_vol_window`
  - 缺失 9 个参数：`f7_k`, `f7_window`, `concentration`, `c_sensitivity`, `score_band`, `ma_trend_period`, `f1_sensitivity`, `f3_sensitivity`, `f1_ema_period`
- **Lines 147-153** (`convergence`): 同样只记录 8 个固定参数

**后果**：只要搜索了上述参数中的任何一个，analysis.json 的 top_bottom_divergence 和 convergence 就不会包含它的数据 → report 永远写不全 §2 和 §3。

**修复**：改为从 trial.params 动态提取所有键，而非硬编码列表。

### 🟡 Doc: optimization.md section 编号重复

两个"一、二、三..."序列（用户发起 → 环境约定 vs 分析哲学 → 常见陷阱），引用时无法唯一定位。

**修复**：将后半部分重新编号为 七～十二。

### 🟡 Doc: 单位转换未文档化

| 参数 | UI 值（optimizer/Tuner） | Config 值（yaml） | 转换 |
|------|------------------------|-------------------|------|
| concentration | 0-30 | 0-3.0 | UI ÷10 |
| c_sensitivity | 0-200 | 0-20.0 | UI ÷10 |
| score_band | 0-20 | 0-0.2 | UI ÷100 |

optimization.md 搜索参数表写的是 UI 值范围，但没标注单位转换。用户手写 `--params` 时容易填入 config 值。

**修复**：在搜索参数表加一列注明单位。

### 🟢 Enhancement: 无 `--lock` 机制

分层优化要求第一轮锁死执行层参数。当前只能通过 `--params "max_holdings=4 disc_step=0.11 ..."` 逐个枚举——漏一个就会被 TPE 搜索。

**修复**：在 `quant_optimizer.py` 加 `--lock` 参数，自动从基线 preset 读取指定参数的当前值并固定。
