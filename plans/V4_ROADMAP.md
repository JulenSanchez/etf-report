## 2. v4.0.0 待建设

### 2.1 新页面（第三大主题）

当前两大主题：正式报告页 `index.html` + Tuner 调试页 `tuner.html`。

v4 新增：**人设仪表盘（Persona Dashboard）** —— 三派对比视图。

**信息架构草案**：
```
┌─ 人设切换器（三 Tab）
├─ 各人设核心指标卡（目标函数实时值）
├─ 三派对比雷达图（因子暴露 / 扇区偏好 / 风险指标）
├─ 选股分歧日历（哪些天三派选了不同的 ETF）
├─ 人设演化时间线（参数变更历史 / 策略升级记录）
└─ 版本演化协议面板（新策略升级时三派的投票状态）
```

待讨论的具体问题：
- 是否需要独立 HTML 页面，还是作为 Tuner 的扩展 Tab？
- 数据来源：实时回测 vs 预计算 payload？
- 存储方案：人设参数历史如何持久化？

### 2.2 版本演化协议（代码层）

人设宣言中已定义基础规则（见 `persona-declarations-20260528.md` §版本演化协议），v4 需要实现为：

1. **策略升级提案**：新增因子 / 改执行逻辑 / 改置信函数 → 自动跑三派回测
2. **独立投票**：各自用目标函数判断接受/拒绝
3. **分歧处理**：接受方升级，拒绝方保留当前版本（策略分叉）
4. **分叉记录**：记录谁在用哪个策略版本的哪组参数

### 2.3 回测工作台（Backtest Workbench）

解决当前性能瓶颈的基础设施升级：

- **共享数据层**：启动时一次性加载 44 支 ETF 全史数据到进程内存
- **多进程调度**：`ProcessPoolExecutor` 并行跑多窗口/多参数回测
- **结果缓存**：已有 `last_backtest.json` → 扩展为键值缓存 `{preset}_{window}_{param_hash}`
- **Queue 调度**：TPE 优化 → 采样队列 → 多 worker 并行跑 → 结果批量回传
- **性能目标**：40 trial × 2Y TPE 从 30min → 10min（3x 加速）

### 2.4 扩展参数自由度

每个人设可独立探索的参数维度：

- 精算师：w1/w3/w7、C、CS、f1s/f3s/f7t/f7k/f7w（🔬 TPE 运行中）
- 禅修者：同上，CS=0 锁定（🔬 TPE 运行中）
- 赌徒：✅ 已完成

远期：每个人设可以跑自己的**持续优化管线**（月度重跑 TPE，自动检测退化，提案升级）。

## 2.5 多 Agent 自发优化（远期愿景）

每个 Persona 作为独立 Agent：
- **自主监控**：定期检测自己的等权目标函数是否退化
- **自主提案**：退化时自动触发 TPE 参数搜索
- **自主投票**：框架级变更（新因子/新执行逻辑）时各自用目标函数投票
- **自主分叉**：策略版本树，不同人设可运行不同版本的回测引擎

这本质上是把当前手工的"发现问题→跑 TPE→人工判断→落地"流程自动化。
