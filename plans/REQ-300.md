# REQ-300: 正式页合成杠杆摘要展示

**创建**: 2026-06-18 | **状态**: backlog | **优先级**: 🟡 Normal | **目标版本**: v3.9.0

## 背景

Stage A 的合成杠杆结果如果进入正式页，必须以风险摘要形式展示，不能只展示更高收益。正式页的目标是解释“杠杆贡献与代价”，不是鼓励直接加杠杆。

## 目标

正式页 payload / UI 支持展示：

```text
accountMode
maxGrossExposure
avgExposure
leverageContribution
interestDragEstimate
maxDailyLoss
worstMonth
daysAbove100
daysAbove150
daysAbove180
```

必须标注：

```text
Stage A 为零融资成本合成杠杆模型，不代表真实两融账户收益。
```

## 非目标

- 不展示真实维持担保比例。
- 不展示追保/强平。
- 不把未 promotion 的研究候选放进正式页主指标。

## 预计改动区域

- `scripts/quant_build_payload.py`
- `scripts/update_report.py`
- `assets/js/quant-main.js`
- `assets/js/quant_payload.js`
- `index.html`

## 验收标准

1. 正式页能展示已 promotion 的 synthetic_leverage 摘要。
2. 收益提升、融资成本压力估算和尾部风险同屏。
3. 若没有杠杆结果，正式页保持现有现金账户展示。
4. payload 缺字段时前端显示 `-` 或隐藏模块，不报错。
5. `python scripts/quant_build_payload.py` 通过。
