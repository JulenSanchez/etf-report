## 动机

> 回测结果只展示 NAV 曲线和持仓快照，缺少对全池 ETF 短期涨跌结构的直觉感知。热力图用一屏展示所有 ETF 的近期涨跌分布，快速发现"谁在涨、谁在跌、涨跌是否集中"。

**现状问题**：
- 回测结果页只看组合级别指标，无法快速扫视底层标的的近期表现
- 调参时不知道当前哪些 ETF 正在走强/走弱，调仓信号缺乏直觉支撑

**不做会怎样**：
- 调参仍然依赖单一回测指标，缺少对池子结构的感知

## 设计决策

### 前端交互

1. **方案 A — 新增右侧第三 tab**：热力图 tab 与"参数原理""回测结果"并列
2. **固定正方形格子**：40×40px，格内 10px 等宽字体显示 1 位小数（如 `3.2`、`-12.3`），不加百分号
3. **chart DOM 全量渲染**：宽度/高度 = 列数/行数 × 40px，由 ECharts 按固定尺寸渲染，不做自适应缩放
4. **纵向滚动**：chart 全高渲染（40 行 → ~1658px），外层 viewport `overflow-y: auto`，浏览器原生滚动条
5. **横向滑块**：自定义单一整体滑块（16px 高蓝色条），控制 `viewport.scrollLeft`；可拖动平移 + 点击轨道跳转 + 滚轮联动
6. **点击排序列**：点击任意格子 → 按该日升序排列 → 再点降序 → 再点恢复扇区默认。右上角有状态指示
7. **扇区分组**：Y 轴色点标注扇区，扇区间虚线分隔，顶部扇区图例
8. **颜色**：绿涨红跌，9 档 pieces 色阶
9. **周期切换**：5 日 / 20 日按钮切换
10. **数据范围**：底层预计算最远 6 年（与回测窗口一致），默认显示最近约 100 个交易日

### 后端预计算

**接入 Tuner preload() 流程**，在已有 CSV 加载完成后、Flask 就绪前执行：

```
preload()
  ├─ 1. load_etf_data()           → CACHE["all_daily"]    ← 已有
  ├─ 2. build_hs300_weekly()...   → CACHE["hs300_*"]      ← 已有
  ├─ 3. precompute_heatmap()      → CACHE["heatmap"]      ← 新增
  └─ 4. Flask 就绪
```

**计算逻辑**（纯 pandas，向量化）：

```python
def _precompute_heatmap_returns(all_daily, lookbacks=(5, 20)):
    returns = {}
    for lb in lookbacks:
        returns[str(lb)] = {}
        for code, df in all_daily.items():
            close = df["close"].astype(float)
            ret = close.pct_change(lb)   # 40 支 ETF 全算 < 0.1s
            returns[str(lb)][code] = ret.dropna()
    return returns
```

**复用关系**：与 `quant_backtest.py` 共同消费 `CACHE["all_daily"]`，不重复加载 CSV，不重复 parse OHLCV。各自做独立派生计算。

**API 端点**：

```
GET /api/heatmap_data?lookback=20&force=0
```

返回：
```json
{
  "lookback": 20,
  "dates": ["2024-01-02", ...],
  "etfs": [
    {"code": "512400", "name": "有色金属ETF", "sector": "资源周期",
     "returns": [0.032, -0.015, ...]}
  ]
}
```

- `force=1`：丢弃缓存，从 `CACHE["all_daily"]` 重新计算
- 默认走缓存，Tuner 运行期间只算一次
