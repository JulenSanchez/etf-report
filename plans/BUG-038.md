# BUG-038 🔴 CRITICAL: 合成杠杆从未生效

**严重度**: 🔴 Critical | **状态**: open | **发现**: 2026-06-25 | **引入**: v3.0.0（合成杠杆首次实现）

## 根因

`scripts/quant_backtest.py:190` — `_execute_rebalance()`

```python
buy_value = min(diff, cash)   # 买入金额不得超过现金余额
```

合成杠杆模式下 `mbull=1.58` → `target_value = tv × 1.58` → `diff = 1,580,000`。但 `cash = 1,000,000` → `buy_value = min(1,580,000, 1,000,000) = 1,000,000`。杠杆买入被现金上限截断。

**现象**：前端显示 158% 仓位（信号层的目标仓位），但实际买入量 = 现金余额，有效敞口 ≤1.0。日收益 1:1 跟随标的，没有杠杆放大。

**两层分离**：
- 信号层正常：bull 改变了选股+调仓行为（策略"以为"有杠杆所以更激进）
- 执行层有 bug：买入被现金截断，杠杆从未生效

## 修复方案

`_execute_rebalance` 第 190 行：synthetic_leverage 模式下，`buy_value` 上限从 `cash` 改为 `max_gross_exposure * tradable_tv`，允许现金为负（借款）。

```python
# 当前（bug）
buy_value = min(diff, cash)

# 修复后
if account_mode == "synthetic_leverage":
    max_buy = max_gross_exposure * tradable_tv - current_holdings_value
    buy_value = min(diff, max(max_buy, 0))
else:
    buy_value = min(diff, cash)
```
