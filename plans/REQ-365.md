## 完整历程

### Phase 1: 拆分（7/9）

BUG-051 核查中发现 `discretize_step` 有双重身份——信号步长（离散化精度）和执行步长（调仓阈值）。拆分为两个独立参数 `discretize_step` + `execution_step`，Tuner 前端分别暴露。

### Phase 2: 研究推翻（7/10）

在拆分后的实验中，两个参数**从未被独立变化过**——所有实验都保持 `exec = sig`，等价于一个参数控制两个锁定的维度。

REQ-366 的系统性研究发现：
- 离散化的 alpha 来自"粗粒度集中 + 信号延迟"的混合效应
- C 和 CS 无法替代这个效应（操作的是幅度维度，不是频率维度）
- 分离后没有产生两个独立可调的干净参数

### Phase 3: 合并为整数 N（7/10 最终态）

将 `discretize_step` + `execution_step` 合并为单一整数 `signal_steps`（N）：

- `step = total_target / N`（牛熊自适应，bull=1.58/N, bear=0.60/N）
- 同一 `step` 同时用于离散化和调仓执行
- 默认 N=17（接近连续 softmax，纯摩擦参数）

## 后续

N 现在是纯摩擦参数（N=17 时步长 ~0.09，接近连续）。N 减小产生的 alpha（约 1100 万 / 6Y）已被确认为离散化的粗粒度效应，后续应通过死区 + C/CS 重新表达。详见 REQ-366。
