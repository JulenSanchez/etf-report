# REQ-364: 波动率归一化信心函数（Z_vol）

**优先级**: 🟡 P1 | **状态**: 研究完成，不推进 | **目标版本**: —

## 1. 理论论证

### 1.1 当前 `ma_trend` 的设计逻辑

`ma_trend` 回答的问题是"HS300 在趋势线上吗"：

```
输入：HS300 周线收盘价 vs 周线 MA20
输出：二值 — 线上 = bull_pos，线下 = bear_pos
```

这等价于一个固定阈值判定：HS300 从 MA 上方跌到下方 ≈ 趋势反转。但这个阈值完全不感知波动率——在年化波动率 15% 的 2017 年和年化波动率 40% 的 2025 年，MA 穿线触发防守的时间点质量截然不同。高波期 MA 穿线频繁，信号噪声比恶化。

### 1.2 Z_vol 的理论基础

来自豆包对话（2026-07-09）的核心论点：

> 传统"回撤 20% = 技术性熊市"是美股低波巧合。标普 σ_ann ≈ 20% → 20% 回撤 Z=1.0。高波资产（KOSPI σ_ann ≈ 40%）同样回撤 20% 仅 Z=0.5，只是常规噪音。跨资产牛熊判定必须使用波动率归一化后的回撤倍数。

**Z_vol = |回撤| / 年化波动率** 的本质是统计标准化：将绝对回撤转换为"相对于该资产正常波动水平的倍数"。这解决了固定阈值跨波动率环境不可比的问题。

### 1.3 为什么 Z_vol 作用于 HS300 而非策略净值

策略净值的回撤是仓位决策的结果——我们自己决定半仓还是满仓，回撤大小是我们选择的风险敞口。用它来算 Z_vol 再反过来调节仓位，形成循环论证：减仓 → 回撤变小 → Z_vol 变小 → 加仓 → 回撤变大 → ...

HS300 的回撤是独立于我们持仓的外部事实。它和 `ma_trend` 回答同一个问题——"市场环境怎么样"——只是用了不同的测量方式：

| | ma_trend | Z_vol |
|------|----------|-------|
| 数据源 | HS300 日线 | HS300 日线（同源） |
| 输入 | 价格 vs MA 均线 | 回撤 ÷ 滚动年化波动率 |
| 输出 | 二值（线上/线下） | 连续（0 ~ 1.65+） |
| 波动率感知 | 无 | 有——高波时回撤阈值自动放大 |
| 仓位映射 | bull_pos or bear_pos | bull_pos ~ bear_pos 线性插值 |
| 滞后性 | MA 周期越长滞后越大 | 回撤从峰算，无均线滞后 |

### 1.4 在引擎中的位置

引擎每一日调仓的决策链：

```
┌─ Layer 1: 选股 ─────────────────────┐
│  F1(EMA偏离) + F3(量比) + F7(对数收益偏离)  │
│  → composite score → softmax → 每 ETF 权重   │
└──────────────────────────────────────┘
                    ↓
┌─ Layer 2: 组合仓位（信心函数）─────────┐
│  ma_trend 或 z_vol                       │
│  → total_target → 统一缩放 Layer 1 产出    │
└──────────────────────────────────────┘
```

Z_vol 在 Layer 2，和 `ma_trend` 是同等地位的替代方案——同一个 `conf_type` 枚举下的新选项。两者共用 `ma_bull_pos` / `ma_bear_pos` 仓位参数。

### 1.5 和 gam-0 双时钟结构的兼容性

gam-0 的 71/13/16 权重分配在 Layer 1 实现了一个双时钟：F1 周频管方向（主心骨，低噪声），F7 日频管紧急刹车（保险丝，高灵敏）。Z_vol 在 Layer 2 提供了一个互补的市场环境判断——不干扰 Layer 1 的选股逻辑，只改变"子弹打多少"。

在高波市场中（如 2024-2025），HS300 频繁 MA 穿线会导致 `ma_trend` 反复切换满仓/防守，而 Z_vol 因为分母（σ_ann）同步放大，阈值自动抬高，信号更稳定。这恰好契合 gam-0 的设计意图：Layer 1 已经在处理个股波动（F7），Layer 2 不应再被市场波动误导。

## 2. 算法

### 2.1 HS300 滚动波动率

HS300 日线数据已在引擎中加载（`benchmark_data.py` → `load_hs300_daily_cached`）。在回测主循环的 z_vol 分支中：

```
1. 每个 rebalance 日，从 HS300 日线取 rb_date 之前 250 个交易日的收盘价
2. 计算对数日收益率序列
3. σ_day = std(日收益)  // 样本标准差, ddof=1
4. σ_ann = σ_day × √250
5. σ_ann = max(σ_ann, 0.02)  // floor 防除零
```

HS300 日线列："date", "close"——直接从已加载的 `hs300_daily` DataFrame 取。

### 2.2 HS300 回撤（滚动峰值）

**为什么不用历史峰值**：这是一个被美股长期牛市遮蔽的观念盲区。标普大部分时间在创新高，"历史峰值回撤"和"窗口内回撤"近似等价。但 A 股存在结构性熊市（HS300 2021 年见顶后数年未回），锚定 5 年前的历史高点毫无意义——就像豆包对话中指出的"20% 回撤 = 技术性熊市"一样，这个惯例在美股成立，搬到 A 股就失效。

**Phase 1 实证验证**（`scripts/quant_zvol_study.py`）：滚动 500 日峰值 vs 历史峰值，Z_vol 与 ma_trend 一致率从 36.2% 提升至 58.3%，Z ≥ 1.65 的熊市天数从 444 降至 271。滚动峰值消除了结构性熊市带来的信号钝化。

**算法**：

```
1. peak_window = 500（默认，≈2 年交易日，与波动率窗口独立可配置）
2. hs300_peak[t] = max(hs300_close[t - peak_window + 1 : t])
3. DD[t] = hs300_close[t] / hs300_peak[t] - 1  // 负数
4. Z_vol[t] = |DD[t]| / σ_ann[t]
```

滚动峰值使得——在美股长期牛市中行为等同于历史峰值；在 A 股结构性熊市中峰值随窗口滚动重置，Z_vol 对中期趋势反转保持敏感。这与 F7 的 250 日滚动 lookback 共享同一设计哲学。

### 2.3 Z_vol → total_target 映射

```
if Z_vol < z_low:                    total_target = ma_bull_pos
elif Z_vol >= z_high:                total_target = ma_bear_pos
else:
    t = (Z_vol - z_low) / (z_high - z_low)
    total_target = ma_bull_pos + t × (ma_bear_pos - ma_bull_pos)
```

| 参数 | 默认 | 范围 | 含义 |
|------|------|------|------|
| `z_low` | 0.8 | 0.3~1.5 | Z < 此值 = 正常回调，满仓 |
| `z_high` | 1.65 | 1.2~3.0 | Z ≥ 此值 = 统计显著下跌，防守 |
| `z_vol_window` | 250 | 60~500 | 波动率滚动窗口（日） |
| `z_vol_peak_window` | 500 | 250~1000 | 回撤峰值滚动窗口（日），≈2 年 |

### 2.4 冷启动

前 250 日 HS300 数据不足时：σ_ann 用已有窗口计算（最少 20 日），Z_vol 可能不稳定，但这是任何滚动统计的固有属性。首年默认行为偏向满仓（Z_vol 被低估时映射到 bull_pos）。

## 4. 待解决的问题（回测验证后回答）

1. **z_low=0.8 / z_high=1.65 的理论值适合 HS300 吗？** HS300 年化波动率历史上在 15%~35% 之间摆动。正常的 20% 回撤在低波期 Z=1.3（接近趋势反转），在高波期 Z=0.57（噪音）。1.65 来自正态分布单尾 5% 显著性，但市场收益非正态——需要在历史回撤分布上做实证标定。

2. **急跌时分子分母同涨的钝化效应**：新冠崩盘（2020-03）DD 飙升 + σ_ann 同步飙升 → Z_vol 可能被低估。这是所有波动率归一化方法的已知局限。对策：σ_ann 窗口可以比 250 日更短（如 60 日），让分母对急跌更敏感。

3. **HS300 回撤 vs 持仓回撤的信号分歧**：HS300 涨但我们持仓跌（扇区轮动导致偏离基准）→ Z_vol 满仓但实际在亏钱。这不是 Z_vol 的问题——它是"市场环境"信号，不是"策略健康"信号。策略健康由 Layer 1 的 F7 和 MDD 监控来覆盖。

4. **滚动峰值窗口的最佳值**：Phase 1 用 500 日得到了一致率 58.3%。更短（250 日）会使 Z_vol 更灵敏但可能过于接近 ma_trend 而失去差异化价值；更长（1000 日）会趋近历史峰值的钝化问题。最佳窗口应在 Phase 3 参数搜索中确定。
