# REQ-354: 拆股自动检测 + 内存清洗 + 盘后全量重拉

**优先级**: 🔴 P0 | **状态**: done | **目标版本**: v3.11.0 | **完成日期**: 2026-07-06

---

## 动机

ETF 份额拆分（如 515880 通信 ETF 1:2）当天，qfq 前复权数据未调整历史价格。CSV 为拆前价，Sina 实时为拆后价，回测出现虚假 50% 暴跌。此前依赖手动全量重拉 + 手改 CSV，需要自动化。

## 事实基础

- AKShare `fund_cf_em` 可检出池内任意比例拆股（已实测 54 支出 3 条记录）
- 拆股当天 qfq 不调整历史（7/6 实测：7/3 close=1.579 拆前、7/6 close=0.76 拆后）
- 但 qfq 最终会调整（上一个 3:1 拆分已平滑，无跳变）

## 实现

### 拆股检测（`_ensure_splits_detected`）

Tuner 首次 `refresh_data` 时通过 AKShare `fund_cf_em` 拉取当年截至今日的拆股事件，与 `corporate_action_events.json` 已注册事件合并去重。后续调用复用缓存。

### 盘中内存清洗（`_apply_split_memory_bridge`）

仅对最新一次拆股生效（避免旧拆分被重复清洗）。自愈判断：

```
csv_last_close / intraday_close ≈ split_ratio → 需要洗
csv_last_close / intraday_close ≈ 1.0        → 已调整，跳过
```

清洗输入包含 intraday bar，确保跨周末边界检测正确（如周五 ex_date → 周一生效）。

### 盘后全量重拉（`_full_refetch_split_etfs`）

仅 post-market 触发。跳过已拉过（CSV 末笔日期 ≥ 拆分日）的 ETF。等 qfq 调整生效后永久修复 CSV。

### 调用位置

`refresh_data()` 中，`_reload_csv_to_cache` 之后、`_precompute_benchmarks` 之前。

## 验证

写入独立测试脚本，覆盖两个场景均 PASS：
1. CSV 已调整 → 比值 ≈1.0 → 跳过清洗 → 数据不变
2. CSV 未调整 → 比值 ≈2.0 → 触发清洗 → 首笔正确 ÷2

## 行为矩阵

| 场景 | 检测 | 内存清洗 | 全量重拉 | 结果 |
|------|------|---------|---------|------|
| 拆股当天盘中 | AKShare | ✅ | 不触发 | 回测正确 |
| 拆股当天盘后 | 同上 | 跳过（非 intraday） | ✅（qfq 可能未调整→无害） | CSV 待 qfq 生效 |
| 次日盘中 | 同上 | 自愈跳过（CSV 已调） | 不触发 | 静默正常 |
| 次日盘后 | 同上 | 跳过 | ✅ qfq 已调整→永久修复 | CSV 修复 |

## 已知限制

- 只处理最新一次拆股（旧拆分已被 qfq 处理，无需重复洗）
- 拆股当天盘后全量重拉可能无效（qfq 同日未调整），需次日盘后再次重拉才永久修复
- AKShare `fund_cf_em` 按年查询，返回整年数据但仅消费 ≤ 今日的事件
