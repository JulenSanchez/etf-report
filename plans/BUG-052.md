# BUG-052: build_frontier_output MDD 范围错配 + 重验证后缺槽位去重

**严重度**: 🟡 Medium | **状态**: fixed | **发现日期**: 2026-07-10 | **关闭日期**: 2026-07-10

## 现象

`/optimize` 跑完后 Tuner 前沿选点图异常：

1. **actuary**：文件有 16 个前沿点，图表只显示 7 个——大量点 MDD < -40 被 x 轴硬编码 `[-40, -20]` 裁掉
2. **gambler**：8 个前沿点中 3 个 MDD 集中在 -35.3 ~ -35.5（同一 1% 槽位），散点图堆叠

## 根因

两个问题均出在 `src/etf_report/core/quant_contract.py::build_frontier_output`：

### 1. mdd_range 默认值与前端不一致（L511）

```python
def build_frontier_output(school="gambler", data_dir="research/params",
                          output_path=None, mdd_range=(-50, -15), ...):
```

前端 x 轴硬编码 `[-40, -20]`（`templates/tuner.html:2431`），后端默认 `(-50, -15)`。actuary pool 25 个 trial 的 MDD 范围 [-44, -20]，全部通过后端过滤，16 个通过重验证。其中 9 个 MDD < -40 在前端不可见。

### 2. 重验证后无槽位去重（L613-634 之后）

当前流程：

```
pool trial → mdd_range 过滤 → 槽位去重(round(mdd)) → 重验证(回测) → mdd_range 过滤 → 输出
                                                                      ↑ gambler 专用 Pareto re-filter
                                                                      × zen/actuary 无 re-filter
```

重验证会改变 MDD 值（回测用最新数据重算）。原先在不同槽位的两个 trial，重验证后可能落到同一个 `round(mdd)` 槽位。gambler 有一条 post-validation Pareto re-filter（L641-649），但 zen/actuary 完全没有重验证后的去重。

且 gambler 的 post-validation re-filter 是 Pareto 支配去重，不是槽位去重。Pareto 允许两个非支配点落在同一 `round(mdd)` 槽位（如 gambler 的 -35.5 / -35.4 / -35.3 三个点均 `round` 到 -35）。

### 完整因果链

```
mdd_range=(-50,-15) 过宽
  → 大量 MDD < -40 的 trial 进入候选
  → 重验证后部分 MDD 仍 < -40
  → 写入 frontier JSON
  → 前端 [-40,-20] 裁掉，用户只看到 7/16

重验证后无槽位去重
  → 同一 round(mdd) 槽位有多个点
  → 前端散点图堆叠
```

## 修复方案

`build_frontier_output` 四处改动：

1. **默认值对齐**（L511）：`mdd_range=(-50, -15)` → `mdd_range=(-40, -20)`

2. **gambler 追加槽位去重**（L545 之后）：`compute_frontier`（Pareto 非支配筛选）保留，在其结果上再跑一次 `round(mdd)` 槽位去重。Pareto 保证曲线不"回退"，槽位去重保证不堆叠。是增强关系，不是替代。

3. **重验证后槽位去重**（L634 之后）：对所有 school，重验证后的 `validated` 再跑一次 `round(mdd)` 槽位去重。

4. **gambler 专用 Pareto re-filter 保留**（L641-649）：Pareto re-filter 确保重验证改变 AR/MDD 后曲线仍满足非支配性，与步骤 2 一致——Pareto 在前、槽位去重在后。

### 最终流程

```
pool trial
  → mdd_range(-40,-20) 过滤
  → [gambler] Pareto 非支配 → 槽位去重
  → [zen/actuary] 槽位去重
  → 重验证（回测重算 MDD/AR/Calmar/Sortino）
  → mdd_range(-40,-20) 再过滤
  → [gambler] Pareto re-filter（保证非支配曲线）
  → [全部] 槽位去重（保证每 1% MDD 最多 1 点）
  → 写入 frontier JSON
```

## 影响面

- **仅改** `quant_contract.py::build_frontier_output`
- 不涉及 `iterative_optimizer.py` / `quant_backtest.py`
- 不涉及 Tuner 前端
- 修复后需重新刷新前沿文件（手动跑 `build_frontier_output` 或重跑 `/optimize`）

## 验证

修复后运行：
```bash
python -c "from src.etf_report.core.quant_contract import build_frontier_output; build_frontier_output('actuary'); build_frontier_output('gambler'); build_frontier_output('zen')"
```

检查：
- 所有前沿点 MDD ∈ [-40, -20]
- 同一 school 下无重复 `round(mdd)` 槽位
