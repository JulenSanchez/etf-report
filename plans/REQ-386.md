## 背景

Walk-Forward Phase 1（REQ-323）跑 3 步独立 in-sample 优化（3Y 窗 → TPE → 最优参数）。核心假设是：参数空间足够稳定，TPE 能在 3Y 窗口内找到可泛化的参数组合。

但这个假设未经检验。如果 3 步优化出 3 组完全不同的参数、样本外表现崩塌，根因有两种可能：

| 根因 | 含义 | 对策 |
|------|------|------|
| **参数框架不稳定** | 17D 空间中 3Y 窗口的信噪比太低，TPE 在噪音里过拟合 | 简化参数空间后再做 WF |
| **市场结构确实在变** | 参数应该每年不同，这正是 WF 的价值 | 推进 WF，接受参数波动 |

## 核心问题

**WF 是用来验证方法的，还是用来诊断框架的？**

如果能提前确认框架足够稳定，Phase 1 就可以聚焦"验证 walk-forward 方法"。如果框架本身不稳定，Phase 1 就要切换目标为"诊断哪个参数维度在漂移"。

## 参数优化框架三代演变

### Phase 1: TPE 裸跑（v3.10 前）

```
工具: optuna TPESampler, Sobol 50 + TPE 50
参数: 全维独立采样，无分组约束
目标: 6Y COMP（等权均值）
```

**失败模式**: 280 trials 无一超越现有预设。根因——16D 无约束，TPE 被高 MDD 端霸占（bull→2.0 的 trial AR 天然更高），实用 MDD 区间（-20%~-40%）完全空白。随机搜索几乎碰不到 gam-2 所在的尖峰。

**教训**: 参数不是独立变量，全维 TPE 在弱信噪比下发散。

### Phase 2: 复杂框架（v3.10~v3.12）

```
工具: 迭代缩界 TPE + 分栏取样 + 四区并行（pareto_optimizer.py + iterative_optimizer.py）
流程: Sobol 种子 → narrow_bounds（按 MDD 分栏）→ 多区独立 TPE 并行
      → 合并去重 → 缩界 → 下一轮 → 直到收敛
```

| REQ | 能力 | 核心逻辑 |
|-----|------|---------|
| 327 | 迭代缩界 TPE | 多轮缩界：top-15 窄界 → TPE → 合并 → 下一轮缩界 |
| 336 | 分栏取样 | 按 5% MDD 栏分 trial，栏内样本 <3 自动合并到邻栏 |
| 344 | 四区并行 | 4 个 MDD 区间各跑独立 TPE，ProcessPoolExecutor 并行 |

**成果**: 376 trials → 11 前沿点。R3 找到甜点 MDD=-29.3% / COMP=+92.8%。

**问题**: 框架太重。两套脚本、多轮迭代、并行调度、缩界逻辑——每个环节都是独立维护点。保守端（-20%~-24%）COMP 偏低，需要额外 target 补跑。

### Phase 3: 简化 + 联合优化（v3.13，当前态）

```
工具: research_utils.py  backtest() + grid_sweep() + optimize_group()
理念: 分组约束 → TPE 在组内找代偿路径；其余参数锁 DEFAULT_LOCK
```

核心变化：

| 维度 | Phase 2 | Phase 3 |
|------|---------|---------|
| 入口 | 两套独立脚本（已删） | `research_utils.py` 三个函数 |
| 参数空间 | 全维 TPE | 分组约束——只变组内参数，其余锁死 |
| MDD 约束 | 多区分栏 + 区间独立 TPE | 硬约束 `mdd_bound=-40`，不满足 → trial pruned |
| 多区并行 | ProcessPoolExecutor × 4 | 不需要——单组 TPE 就够 |
| 默认值 | 散落硬编码 | `DEFAULT_LOCK` 单源（BUG-058 治理） |
| 降维 | 无 | N 合并（REQ-365/366）：signal_steps + top_boost → 单整数 |

关键概念：

- **分组约束优化**（REQ-374）: C/CS/TB 作为集中度组一起 TPE，而非分别扫。组内参数互相代偿——TPE 在等风险面上找最优路径。
- **等风险面**: bull/bear/MH/C/CS/TB 沿风险光谱同向施力。激进端：高 bull + 高 MH + 高 C；保守端：低 bull + 低 MH + 低 C。这两个方向不是独立维度，而是同一风险轴的两端。

**Phase 2→3 的简化本身就是成熟信号**: 你知道哪些维度需要联合优化、哪些可以锁死、哪些已被证明不需要每轮重搜。

### 当前框架对 WF 的关键问题

Phase 3 的简化建立在 6Y 全史上。WF 训练窗只有 3Y：

- **分组假设是否仍然成立？** C/CS/TB 在 3Y 上的代偿关系可能偏移——短窗口里 C 的作用被放大（集中度直接决定少数 ETF 的表现），TB 的边际贡献缩小
- **等风险面是否仍然单调？** 3Y 窗口可能不是完整的牛熊周期，bull↑ 和 MDD↑ 的关系可能被一段单边行情扭曲
- **TPE 在 3Y 上的收敛质量？** 数据量减半，噪音更大，信噪比更低

**386 的审查本质：验证 Phase 3 的简化假设在 3Y 窗口上是否仍然成立。**

### 参数分层：哪些参与 WF、哪些锁定

17 个核心参数按"WF 必要性"分四类：

#### 🔒 锁定（已证伪或结构性锁定）

| 参数 | 值 | 证据 |
|------|-----|------|
| f7_lookback | 250 | REQ-382：当前值 6Y 最优，缩短无益 |
| — | — | （无新增锁定，f7_down 移入组 C） |
| band | 0.03 | 历史讨论低敏，分数带本质是摩擦参数 |
| band_sensitivity | 0.0 | 同上 |
| conf_type | ma_trend | 旧信心函数已退役 |
| dead_zone / full_zone | 默认 | 同上 |
| ma_direction_confirm | True | 结构性固定 |

→ 锁定 7 个，剩余 10 个参与 WF。

#### 🟢 WF 优化参数（分 4 组联合优化）

```
组 A「风险敞口」: bull, bear, MH, MA_period      ← 控制"下注多少"
组 B「集中度」:   C, CS, N                        ← 控制"怎么分"
组 C「信号灵敏度」: f1_sensitivity, f3_sensitivity,   ← 控制"感觉多敏锐"
                    f7_up_power, f7_up_span,
                    f7_down_power, f7_down_span
组 D「因子权重」:   w1, w3, w7                     ← 控制"听谁的"
                                                     （约束 w1+w3+w7=100%）
```

**分组逻辑**：

- 组内参数有代偿关系（Phase 3 已验证），适合联合 TPE
- 组间相对独立——风险敞口定了预算，集中度在预算内分配，灵敏度微调信号
- 4 组而非 1 组全开，避免 TPE 在 9D 空间发散（Phase 1 教训）

### 单窗口优化流程

对每个 3Y 训练窗，按优先级串行优化 4 组：

```
输入: gam-0 默认参数（DEFAULT_LOCK）
约束: MDD >= -40%（铁律）

Step 1 — 风险敞口组: {bull, bear, MH, MA_period}
  目标: max 3Y_AR
  原因: 先定风险预算——leverage 和 max positions 决定天花板
  预期: bull 可能在 1.3~1.8 间波动（牛市窗口偏高，熊市窗口偏低）

Step 2 — 集中度组: {C, CS, N}
  目标: max 3Y_AR
  原因: 在 Step 1 的风险预算内，优化仓位分配
  预期: C 和 N 协同调整——高 bull 时 C 可能偏低（分散风险）

Step 3 — 信号灵敏度组: {f1_s, f3_s, f7_up_power, f7_up_span, f7_down_power, f7_down_span}
  目标: max 3Y_AR
  原因: 风险预算和分配定好后，微调信号响应曲线
  预期: 变化幅度小（灵敏度是微调参数）

Step 4 — 因子权重组: {w1, w3, w7}（约束 sum=100%）
  目标: max 3Y_AR
  原因: 最后调整因子间的权重分配
  预期: gam-0 当前 w1=30/w3=60/w7=10 可能在 3Y 上偏移
```

每一步 TPE 约 30~50 trials。单窗口总计 ~150 trials，3 步 × 150 = 450 trials。

**为什么不迭代回去？** Phase 2 的迭代缩界证明了：第二轮缩界收益递减（R3 后 R4-R5 未超越甜蜜点）。在当前数据量下，一次 pass 四组就够了。如果 Phase 1 结果证明某个组的参数在步间剧烈跳变，再考虑"先粗后精"的两轮 pass。

### 预期效果

| 场景 | 预期 | 含义 |
|------|------|------|
| ✅ 理想 | 3 步参数在合理范围内波动（bull 1.3~1.8，C 0.5~0.7），OOS AR 不低于 IS-AR 的 70% | 框架成熟，WF 有效 |
| ⚠️ 可接受 | 某组参数稳定、某组跳变。比如风险敞口组步间一致，但权重组剧烈变化 | 锁定跳变组、放开稳定组，Phase 2 精简维度 |
| ❌ 失败 | 3 步参数无规律跳变，OOS AR 崩塌（< IS-AR 的 50%） | 框架不稳定，需进一步简化后才能 WF |

## 审查维度（386 自身产出）

### 1. 参数敏感度在 3Y vs 6Y 上的一致性

取 9 个 WF 参数，分别做 3Y 和 6Y 的 Sobol 采样（~30 trials），看每个参数的方向性偏好。

- 判定：方向一致 → OK；方向相反 → 窗口敏感，考虑锁定该参数

### 2. 3Y 窗口 TPE 的收敛质量

取一个 3Y 窗口（2023-06~2026-06），跑 2 次独立 TPE（不同 seed），看两组最优参数差异。

- 判定：AR 差 < 5pp → 信噪比可接受；> 10pp → 不稳定

### 3. 分组假设在 3Y 上的验证

风险敞口组 {bull, MH} 和集中度组 {C, N}，分别在 3Y 和 6Y 上做联合扰动（各 9 格点），对比响应面。

- 判定：3Y 响应面崎岖度 ≈ 6Y → 分组假设成立；3Y 明显更多局部极值 → 分组需调整

### 4. 锁定参数清单

确认 §锁定 的 8 个参数确实不应参与 WF。如有新证据，调整锁定/放开。
