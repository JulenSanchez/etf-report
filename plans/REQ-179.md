## 0. 背景

REQ-177（多因子量化打分系统 v1）交付后，主页面已经能展示 3 套写死的策略 preset 净值曲线。但**调参链路一片空白**：

- 每次想试新参数 → 改 YAML → 跑 `update_report.py` 等 30 秒 → 翻命令行看 metrics
- 想调因子算法（比如 RSI 分段映射）→ 改 Python → 重启 → 重跑 → 再翻命令行

REQ-179 的出发点：**把"作者侧"的实验工具单独建起来**，让"调参→看图→再调"形成秒级闭环。

随着工具越用越深，因子算法本身的缺陷逐渐暴露（RSI 分段映射不合理、F3 高位饱和锁死、截面标准化丢失绝对极性等），所以 REQ-179 自然演化成 **工具 + 算法升级** 的双线 Epic。

## 2. 已交付：REQ-179.B — 因子算法升级

### 2.1 截面标准化 → 连续映射（早期改造）

**动机**：原 `normalize_cross_section`（percentile_rank）让 25 支 ETF 强制均匀分布，丢失了"全市场都强"或"全市场都弱"这种**牛熊信号**。

**改造**：每个因子单独映射到 [0, 1]，保留绝对极性

- F1: `sigmoid(ema_dev / sensitivity)` —— 稳态 0.5，远离均线时往两端饱和
- F2: 死区 + 饱和（详见 2.2）
- F3: log+sigmoid（详见 2.3）
- F4: `val_score / 100` —— 估值百分位低 = 高分

**效果**：综合分均值不再恒为 ~52，能跟着市场状态自然漂移；信心函数能识别牛熊。

### 2.2 F2 重构：从"健康区"到"异动检测"

**讨论历程**（关键洞察）：
1. 原 F2 是"趋势自适应分段"——上行 RSI 40-65 最佳，下行 30-50 最佳。但**无法表达"博反弹/逃顶"** —— 极低 RSI 反而被打低分（顺势思维）
2. 用户洞察：**"RSI 是捕捉异动的，需要参考近期均值，大幅偏离才给出影响整体评估的判断"** —— 把 F2 从绝对值评分改成**相对偏离评分**
3. 最终设计：z-score（vs 自身近期 20 日基线）+ 死区（|z|<1.0 输出 0.5）+ 饱和函数（极端时趋向 1.0/0.0）

**算法**：

```python
# Step 1: factor_rsi_adaptive
rsi_series = calc_rsi(close, period=14)
recent = rsi_series.tail(rsi_window=20)
z = (current_rsi - recent.mean()) / max(recent.std(), 5.0)
return clip(z, -3, 3)

# Step 2: map_f2(z, dead_zone=1.0)
if abs(z) < dead_zone: return 0.5     # 死区, 让 F1/F4 主导
overshoot = abs(z) - dead_zone
magnitude = 1 - exp(-overshoot)
return 0.5 + 0.5 * magnitude if z < 0 else 0.5 - 0.5 * magnitude
```

**关键洞察 — "自身涌现"**：
- 大多数时候 |z| < 1，F2 输出 0.5（沉默）→ F1+F4 主导基调
- 只有当某只 ETF 出现极端 RSI 异动时，F2 才推到 0.85+/0.15- → 把综合分明显拉离基调

### 2.3 F3 改造：log+sigmoid 拉伸高位

**问题**：旧 `1 - exp(-ratio/sens)` 在 ratio>5 时基本饱和锁死，**真正爆量品种**（ratio=8）跟**一般强势**（ratio=3）拉不开差距。

**新设计**：先 log1p 让 ratio 的 0~10 范围更线性，再 sigmoid 映射

```python
log_r = log1p(ratio - 1) if ratio >= 1 else -log1p(1/ratio - 1)
return sigmoid(log_r / sensitivity)
```

**效果对比**（sensitivity=1.0）：

| ratio | 旧 F3 | 新 F3 |
|-------|-------|-------|
| 2.0   | 0.74  | 0.66  |
| 5.0   | 0.96  | 0.83  |
| 10.0  | 0.999 | 0.91  |

爆量品种的"涌现"能力恢复。

### 2.4 设计哲学（对话沉淀）

经过多轮讨论形成的因子分工：

| 因子 | 角色 | 节奏 | 作用 |
|------|------|------|------|
| **F1 EMA 偏离** | 地基 | 周/月 | 长期趋势判断 |
| **F4 估值百分位** | 地基 | 周/月 | 长期估值锚 |
| **F2 RSI 异动** | 操作 | 日/周 | 极端时博反弹 / 逃顶 |
| **F3 方向性量比** | 操作 | 日/周 | 验证主升 / 跟随热点 |

设计原则：
1. **F1+F4 决定基调**（长期持仓水平）
2. **F2+F3 在极端处涌现**（短期翻转 / 主升验证），通过自身 z-score 死区 + 高位拉伸自然实现，**不引入动态权重机制**（保留可调参性）
3. **自身涌现 ≠ 系统性热度**：每只 ETF 独立计算 z-score / 量比，热度是个体属性
4. 个人偏好 (bias) 暂时仍作为"标签 + 统一加成"实现，未来考虑升级为 F5 因子（可调权重表达"长期看好但只优先在分数接近时考虑"的语义）
