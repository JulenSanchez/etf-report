## 动机

`index.html` 当前体积 ~704 KB / 15100 行，其中 `<script>` 内联主体占据 1935-14129 行（约 12200 行 JS），是单文件最大的臃肿源头。抽离到独立 `assets/js/report.js`（或按模块拆多文件）后：

- 浏览器可并行下载 HTML 与 JS
- 便于浏览器缓存（数据变化不触发 JS 重拉）
- 可读性 / 调试性 / 代码审查友好度大幅提升

## 风险约束（重要）

REQ-109「资源优化（压缩 CSS/JS）」已被废弃，历史根因是**压缩/单行化**导致 index.html 挂掉。本需求**只做抽离，不做压缩/混淆/合并/单行化**，保持源码完全可读；每步改动都要有可对比的本地 `file://` 预览验证。

## 实施步骤（粗）

### Step 1: 拆分策略评审
- 确定一次性抽成 `report.js` 单文件，还是按模块拆（数据装配 / 图表渲染 / 交互 / 导出）
- 评估是否保留 `runtime_payload.js` 的现有加载顺序

### Step 2: 抽离与引用
- 将 1935-14129 行 JS 按评审结果迁出
- `index.html` 用 `<script src="./assets/js/report.js" defer>` 引用

- 9 个图表功能、调试工具栏、热区/显形/聚焦三模式全部回归
