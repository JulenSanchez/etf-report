# REQ-359: 因子缓存 — 多预设视图

**优先级**: 🟢 P2 | **状态**: backlog | **目标版本**: v3.13.0

## 背景

REQ-357 实现了因子缓存管理（固定的 gam-0 生产缓存）。当用户切换预设（如从 gam-0 换到 zen-1），回测会生成不同参数的因子缓存，但 DM 面板只展示 gam-0 的缓存状态。

## 需求

因子缓存视图扩展为多行：

```
515880 通信  gam-0: OK   gam-1: --   zen-1: --
512400 有色  gam-0: OK   gam-1: OK   zen-1: --
```

- 每行 = ETF × 预设
- 状态：OK（有缓存）/ --（无缓存）/ OLD（参数已变）
- 支持按预设筛选
- 操作：删除特定预设的缓存 / 重建特定预设的缓存

## 依赖

- REQ-357（因子缓存管理面板基础）
- REQ-358（份额变动后自动清理）
