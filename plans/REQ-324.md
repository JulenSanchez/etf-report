# REQ-324: Walk-Forward Phase 2 — 全量前沿曲线

**优先级**: 🔴 P0 | **目标版本**: v3.11.0 | **依赖**: REQ-323 (方法验证通过) | **状态**: 规划中

## 背景

REQ-323 验证了单条 walk-forward 轨迹可行。现在扩展到多个 MDD 约束值，产出第一条完整的 walk-forward 前沿曲线。

## 目标

对 gambler 的 **5 个 MDD 约束值**（-20%, -25%, -30%, -35%, -40%），每个独立走完层 1 + 层 2，产出 walk-forward 前沿曲线。

```
每个 MDD 约束:
  层 1: 3 步 in-sample 优化 → P1, P2, P3
  层 2: 3 步样本外测试 → 3 个 (实际MDD, AR) 点
        取 3 段 AR 的几何均值 → 一个 walk-forward (MDD, AR) 点

5 个 MDD 约束 → 5 个点 → walk-forward 前沿曲线
```

```
in-sample 前沿:   (MDD约束，  训练窗内AR)  ─ 偏乐观
walk-forward 前沿: (实际MDD， 样本外AR)   ─ 生产应发布

两条线之间的 gap → 反映策略稳定性
```

## 前沿曲线的 MDD 轴

关键设计：前沿的 x 轴是**样本外实际 MDD**，不是优化时的 MDD 约束值。

```
约束 MDD=-25% → 实际跑出来可能是 MDD=-30%
约束 MDD=-20% → 实际跑出来可能是 MDD=-22%

前沿上的点标注的是真实经历的回撤，不是纸面约束。
```

## 三派扩展

| 派别 | 风险轴 | 层 1 | 层 2 产出 |
|------|--------|------|----------|
| gambler | MDD | 含约束 AR 优化 | (实际MDD, AR) 曲线 |
| zen | 无 | 无约束 Sortino | 单点 (样本外 Sortino) |
| actuary | bear | 含 bear 约束 | (实际bear, AR) 窄曲线 |

zen/act 的层 2 验证先做最小版本（1 步样本外），不阻塞 gambler 主流程。

## 并行策略

5 个 MDD 约束可以**完全并行**——每个是独立优化，无共享状态。层 1 完成后立即启动层 2。

## 验收

```bash
# 1. 5 个 MDD 约束各产出 walk-forward 点
python -c "
import json, pathlib
for mdd in [-20,-25,-30,-35,-40]:
    r = json.loads(pathlib.Path(f'research/params/wf_gam_m{mdd}/result.json').read_text('utf-8'))
    assert r['wf_ar'] is not None and r['wf_mdd'] is not None
    print(f'MDD={mdd}%: wf_AR={r[\"wf_ar\"]:+.1f}% wf_MDD={r[\"wf_mdd\"]:+.1f}%')
print('PASS: 5/5 constraints')
"

# 2. 前沿单调递增（更高风险→更高收益）
python -c "
import json, pathlib
pts = []
for mdd in [-20,-25,-30,-35,-40]:
    r = json.loads(pathlib.Path(f'research/params/wf_gam_m{mdd}/result.json').read_text('utf-8'))
    pts.append((r['wf_mdd'], r['wf_ar']))
pts.sort()
for i in range(1, len(pts)):
    assert pts[i][1] >= pts[i-1][1] * 0.5, f'AR drops too much: {pts[i-1]} -> {pts[i]}'
print('Frontier monotonic:', pts)
print('PASS')
"

# 3. in-sample vs walk-forward gap
python -c "
import json, pathlib
for mdd in [-20,-25,-30]:
    r = json.loads(pathlib.Path(f'research/params/wf_gam_m{mdd}/result.json').read_text('utf-8'))
    gap = r.get('is_ar',0) - r.get('wf_ar',0)
    print(f'MDD={mdd}%: IS={r.get(\"is_ar\",0):+.1f}% WF={r.get(\"wf_ar\",0):+.1f}% gap={gap:+.1f}%')
    assert gap >= 0, 'walk-forward should not exceed in-sample'
print('PASS: gap exists and is non-negative')
"
```

### 通过标准

1. ✅ 5/5 MDD 约束产出有效 walk-forward 点
2. ✅ 前沿单调：更高风险 → 更高或持平收益（允许 ±50% 噪声）
3. ✅ in-sample vs walk-forward gap ≥ 0（样本外不优于样本内）
