# REQ-333: bull 隐式推导 account_mode

**优先级**: 🟡 P2 | **目标版本**: v3.10.0 | **状态**: 规划中

## 动机

`account_mode` 是显式字段，多处冗余。实际规则很简单：bull>1.0 → 杠杆，bull<=1.0 → 现金。无需单独维护。

## 方案

**推导规则**：`account_mode = "synthetic_leverage" if ma_bull_pos > 1.0 else "cash"`

| 文件 | 改动 |
|------|------|
| `quant_contract.py` | `tuner_params_to_config_override` 自动推导，删除显式 account_mode 读取 |
| `quant_contract.py` | `create_optuna_objective` 删除 `account_mode` 参数 |
| `quant_contract.py` | `build_frontier_output` 删除 `_FIXED_PARAMS` 中的 account_mode |
| `iterative_optimizer.py` | 删除 `--account-mode` 参数 |
| `tuner.html` | 删除账户模式选择器 |
| `config/quant_universe.yaml` | 删除各 preset 的 `account:` 块 |

## 验收

```bash
# bull=1.58 → synthetic_leverage
python -c "from etf_report.core.quant_contract import tuner_params_to_config_override; ov=tuner_params_to_config_override({'ma_bull_pos':1.58}); assert ov['account']['mode']=='synthetic_leverage'"

# bull=0.80 → cash
python -c "from etf_report.core.quant_contract import tuner_params_to_config_override; ov=tuner_params_to_config_override({'ma_bull_pos':0.80}); assert ov['account']['mode']=='cash'"
```
