# REQ-384: 基准指数数据源切换到宽基 ETF

**优先级**: 🟡 P1 | **状态**: ✅ 已交付（2026-07-20，含全量清理） | **目标版本**: v3.14.0 | **日期**: 2026-07-20

## 背景

四个投票指数（000016/000300/000905/399006）依赖 akshare `stock_zh_index_daily` 拉取日线。akshare 近期的数据质量出问题——成交量单位突变（从数百亿 → 3.3亿，差 100 倍）、成交额字段大面积为空。BUG-057 虽已修（磁盘刷新），但上游数据源质量不归我们控制。

替代方案：宽基 ETF（510050 上证50 / 510300 沪深300 / 510500 中证500 / 159915 创业板）。基金公司严格追踪指数，日线数据走 Sina 管线，成交量/成交额完整。不入策略选股池，仅作 MA 信号源。

## 初始交付（Phase 1）

`benchmark_data.py` + `quant_data_fetcher.py`：ETF CSV 优先，aksure fallback。数据管道切换，`000300_daily.csv` 实际内容来自 ETF。

## 全量清理（Phase 2，2026-07-20）

**全项目 index code → ETF code**。000300/000016/000905/399006 从代码库彻底消失：

| 文件 | 改动 |
|------|------|
| `benchmark_data.py` | 重写：ETF code 为主键；`BENCHMARK_ETF_TO_AKSHARE` / `BENCHMARK_ETF_NAMES` / `BENCHMARK_ETF_CODES` 替代旧映射；aksure fallback 隔离到 `_daily_ak.csv` |
| `quant_data_fetcher.py` | 合并为 `BENCHMARK_ETF_AKSHARE`；`fetch_benchmark_indices` ETF 优先、缺失时自拉 kline |
| `quant_tuner.py` | Sina `sh000300` → `sh510300`；2 处 BM_NAMES + 基准循环 + 数据面板标签 → ETF code；3 处重复映射去重 |
| `tuner.js` | `BENCHMARK_CHOICES` + 默认值 → ETF code |
| `quant_backtest.py` / `update_report.py` / `quant_contract.py` | 默认值 `"000300"` → `"510300"` |
| `config/defaults.yaml` / `config.yaml` / `config.example.yaml` | benchmark code → ETF |
| `tests/test_fix_ma.py` / `conftest.py` | `sh000300` → `sh510300` |

旧 index CSV 已删除（8 个文件），四支 ETF CSV 从 Sina 管线全量重拉。aksure 代码保留不删（`_load_index_daily_cached` 内部函数，`_daily_ak.csv` 路径隔离）。
