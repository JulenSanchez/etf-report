# REQ-193: 开发环境便携化与自动化

**优先级**: 🟡 Medium
**状态**: done
**完成日期**: 2026-05-08

---

## 动机

用户需要将开发环境从公司迁移到家里。当前项目有大量私有文件（plans/、runbooks/、config/secrets.yaml 等）被 .gitignore 排除，无法通过 git clone 带走，且无备份。

## 交付内容

### 1. 仓库改 Public + 全部文件入库

- .gitignore 精简为只忽略运行产物（data/, logs/, __pycache__/）
- plans/(70), research/(10), runbooks/(3), config/config.yaml 等全部 git add
- secrets.yaml 加回 .gitignore，并用 git filter-branch 从历史中彻底清除
- config.yaml 中 repo_root 本地绝对路径改为空字符串（fallback 到 skill_dir）

### 2. BatchFiles/ (6 个 bat)

| 脚本 | 功能 |
|------|------|
| UpdateReport.bat | --publish 一键更新+发布 |
| StartTuner.bat | 增量更新数据+启动 Tuner+打开 Edge |
| FetchData.bat | 单独拉取 ETF 数据 |
| GitPull.bat | git fetch + reset --hard origin/main |
| GitPush.bat | git add -A + auto commit + force push |
| Release.bat | 半自动化 RELEASE_RUNBOOK 7 阶段 |
| SetupRules.bat | 安装 rules 到 .codebuddy 或 .claude |

所有 bat 使用 `%~dp0..` 相对路径，跨机器可用。

### 3. Rules bundle + setup_rules.py

- `rules/` 目录存 2 个 project-level rule（etf-report, statusbar-protocol），各 .mdc+.md
- `scripts/setup_rules.py`：自动检测 IDE，安装 rule 到正确目录
- 用户级 rule（karpathy, command-line 等）不打包，由用户自行配置

### 4. quant 数据自动拉取

- update_report.py 新增 Step 2.6：调用 quant_data_fetcher.main() 增量拉取 25 支 ETF 日线/周线
- 不阻断主流程，失败时保留上一版 CSV
- fallback 参数同步为 MA26/B30

## 涉及文件

- .gitignore, config/config.yaml, config/secrets.yaml
- BatchFiles/*.bat (7)
- rules/*.mdc + rules/*.md (4)
- scripts/setup_rules.py
- scripts/update_report.py
