# REQ-348: 盘后定价交易 — 信号推送时间迁移

**优先级**: 🔴 P0 | **目标版本**: v3.11.0 | **截止**: 2026-07-06 | **状态**: 设计中

## 背景

7 月 6 日起盘后固定价格交易扩展到全部 A 股 + ETF（15:05-15:30，收盘价成交）。当前 pre-close push 在 14:50 用盘中实时价估算信号，存在两个问题：

1. 信号基于盘中价，与收盘成交价有偏差
2. 依赖 Sina 实时 API 和成交量估算，复杂且脆弱（BUG-041 就是盘中路径的 bug）

改用盘后定价交易后，可以直接用已确认的收盘数据跑回测，信号更准，逻辑更简单。

参考：`docs/knowledge/after-hours-fixed-price.md`、`plans/REQ-196.md`

## 核心变更

| 项目 | 当前 | 新 |
|------|------|-----|
| 推送时间 | 14:50（盘中） | 15:05（盘后） |
| 数据来源 | Sina 实时 API + intraday_cache | CSV 收盘确认数据 |
| 成交量 | 盘中按时间比例估算 | 实际全天成交量 |
| 因子计算 | 盘中 bar 合并 | 标准日线 |
| 信号准确性 | 近似（盘中价） | 精确（收盘价） |

## 实现

### 1. `preclose_push.py` 简化

- 去除 `14:50` 盘中特殊逻辑
- 推送时间改为 `15:05`
- 刷新数据走 post-market 路径（`refresh_data` → CSV 确认 → `_reload_csv_to_cache`）
- 不需要 `_get_daily_with_cache` intraday 合并（CSV 已有当日确认数据）

### 2. Windows 计划任务

- `etf午盘报告`：触发时间 **14:50 → 15:05**
- 重新注册：`batchfiles/setup_quant_tasks.ps1`

### 3. 运维文档更新

| 文件 | 改动 |
|------|------|
| `docs/runbook/stable.md` | 计划任务表 14:50 → 15:05 |
| `docs/runbook/daily-automation.md` | 检查点 14:50 → 15:05 |

### 不改的

- `COOL_OFF_TIME = 15:10` — 已覆盖 15:05 时间窗口
- `refresh_data()` post-market 路径 — 15:10 后的 Sina fast path 逻辑不变
- Tuner 盘中刷新功能 — 研究用途保留

## 验收

1. 7 月 3 日盘后手动测试：`python preclose_push.py` → 推送基于 7/3 收盘价的信号
2. 7 月 4-5 日（非交易日）计划任务不触发
3. 7 月 6 日 `etf午盘报告` 在 15:05 自动触发 → 推送基于 7/6 收盘价信号 → 用户可在 15:05-15:30 以收盘价成交
