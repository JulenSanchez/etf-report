# REQ-348: 盘后定价交易 — 信号推送时间迁移

**优先级**: 🔴 P0 | **目标版本**: v3.11.0 | **截止**: 2026-07-06 | **状态**: done（2026-07-06 补充两融限制）

## 背景

7 月 6 日起盘后固定价格交易扩展到全部 A 股 + ETF（15:05-15:30，收盘价成交）。当前 pre-close push 在 14:50 用盘中实时价估算信号，存在两个问题：

1. 信号基于盘中价，与收盘成交价有偏差
2. 依赖 Sina 实时 API 和成交量估算，复杂且脆弱（BUG-041 就是盘中路径的 bug）

改用盘后定价交易后，可以直接用已确认的收盘数据跑回测，信号更准，逻辑更简单。

参考：`docs/knowledge/after-hours-fixed-price.md`、`plans/REQ-196.md`

## 核心变更

| 项目 | 当前 | 新 |
|------|------|-----|
| 推送时间 | 14:50（盘中） | 15:10（盘后，COOL_OFF_TIME 之后） |
| 数据来源 | Sina 实时 API + intraday_cache | refresh_data post-market → CSV 收盘确认数据 |
| 成交量 | 盘中按时间比例估算 | 实际全天成交量 |
| 因子计算 | 盘中 bar 合并 | 标准日线 |
| 信号准确性 | 近似（盘中价） | 精确（收盘价） |

## 设计决策

**preclose_push.py 不做时间判断，完全复用 `refresh_data` 的 intraday/post-market 自动分流。** `COOL_OFF_TIME=910`（15:10）为界：<15:10 intraday 路径，≥15:10 post-market 写 CSV。行为由计划任务触发时间控制，用户可随时调整。

## 实现

### 1. `preclose_push.py` 去时间化

- docstring 去掉"Pre-close"/"intraday"措辞 → 通用 "Signal push"
- Stage 2 注释/日志：`intraday data` → `market data`
- 推送标题：`收盘前 | 赌徒` → `赌徒 调仓信号`
- 逻辑零改动（refresh_data + backtest 调用不变）

### 2. Windows 计划任务

- `etf午盘报告`：触发时间 **14:50 → 15:10**
- 模板：`batchfiles/setup_quant_tasks.ps1`

### 3. 运维文档更新

| 文件 | 改动 |
|------|------|
| `docs/runbook/stable.md` | 计划任务表 14:50 → 15:10 |

### 不改的

- `COOL_OFF_TIME = 15:10` — 保持分界
- `COOL_OFF_MINUTES = 10` — 同上
- `preclose_push.bat` — 纯启动器
- Tuner 盘中刷新功能 — 保留
- `refresh_data()` 全部逻辑 — 零改动

## 验收

1. 清除当日 CSV 数据，7/3 盘后手动测试：`python scripts/preclose_push.py --output-md` → refresh_data 走 post-market 路径，推送基于 7/3 收盘价的信号
2. 无论成功/失败，恢复数据：`python scripts/quant_data_fetcher.py --start 2026-07-03 --end 2026-07-03`
3. 7/6 `etf午盘报告` 在 15:10 自动触发 → 推送基于 7/6 收盘价信号 → 用户可在 15:10-15:30 以收盘价成交

## 2026-07-06 补充：两融账户盘后定价交易限制

华泰证券两融账户实测：15:05 后担保品买卖（非融资买入/融券卖出）下单显示"该时间段不支持此类交易"。具体原因待与客户经理确认。

### 设计影响

盘后定价交易路径根据策略杠杆分为两条：

| 策略类型 | bull 范围 | 信号时间 | 成交价 | 跟踪误差 |
|---------|-----------|---------|--------|---------|
| 现金账户 | ≤100% | 15:10 盘后 | 收盘价 | 零误差 |
| 两融账户（合成杠杆） | >100% | 14:50 收盘前 | 盘中价 | 有误差 |

`preclose_push.py` 当前双路径已就绪（盘中路径 + 盘后 post-market 路径），无需代码改动。计划任务时间根据目标策略选择 14:50 或 15:10。

参考：`docs/knowledge/after-hours-fixed-price.md`（纯知识，规则与限制）
