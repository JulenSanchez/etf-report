# REQ-310: 动态分数带

**优先级**: 🟠 P1 | **目标版本**: v3.11.0 | **状态**: 设计中 | **更新**: 2026-07-02

## 实测数据

gam-0 6Y，仅改变 `score_band`：

```
Band     AR%       MDD%     Sortino   Calmar   换仓周%
0.00     90.83    -32.6     2.288     2.79     89.0
0.01     95.41    -32.0     2.373     2.98     82.8
0.02     94.89    -32.7     2.360     2.90     75.6
0.03    102.63    -30.9     2.475     3.32     68.8    ← 最优
0.05     82.14    -36.5     2.107     2.25     58.4
0.08     81.55    -35.5     2.139     2.29     46.4
```

**结论**：band 对策略表现的影响是明确的倒 U 型。0.03 是 gam-0 的经验最优值。

此前 forward-return 分析（被阻替换的后续表现）在单事件层面被噪声淹没（~50% 胜率），但累积到组合层面后效果清晰：适度的替换门槛降低了无效轮动带来的摩擦损耗。

## 设计方向修正

原方案想把 band 从 0.03 整体收窄——实测数据否定了这个方向。

正确的问题是：**0.03 是 6Y 均值下的最优值，但不同市场阶段的最优 band 可能不同。** 动态分数的价值在于在不同阶段逼近不同的最优值，而不是把全局均值往上或往下调。

## 设计方案

### 与 `c_sensitivity` 的对称设计

当前已有先例：`concentration` 是静态仓位集中度，`c_sensitivity` 控制它对分数分散度的响应。两参数均在 `PARAM_BOUNDS` 中独立可优化。

动态分数带采用完全对称的模式：

```
concentration         →  score_band          （静态基准）
c_sensitivity         →  band_sensitivity    （动态响应度）
有效 C = C × c_mult   →  有效 band = band + Δ(trend)
```

`band_sensitivity = 0` 时退化为当前静态行为，`> 0` 时启用动态调整。

### 术语：C / CS / B / BS

四个参数形成两组对称体系：

| 缩写 | 参数名 | 旧名 | 含义 |
|------|--------|------|------|
| **C** | `concentration` | — | 仓位集中度（静态基准） |
| **CS** | `c_sensitivity` | — | 集中度对分散度的响应 |
| **B** | `band` | `score_band` | 分数带（静态基准） |
| **BS** | `band_sensitivity` | — | 分数带对 gap 趋势的响应 |

两者公式对称：

```
有效 C  = C × (1 + CS × (dispersion − 0.5))
有效 B  = clamp(B − BS × trend, floor, ceiling)
```

BS = 0 时退化为当前静态行为。B/BS 与 C/CS 一样，均在 `PARAM_BOUNDS` 中独立可优化。

### 公式

BS 复用 CS 的输入信号——Top-N z-score 的标准差 `dispersion`：

```
dispersion = z_scores.std()                    # CS 已经在计算的值
dispersion_ma_slow = EMA(dispersion, span=12)   # 长期趋势（季度）
dispersion_ma_fast = EMA(dispersion, span=4)    # 短期变化（月）

trend = (fast - slow) / max(slow, 0.01)
       # >0: 整体在分化，alpha 机会扩大 → band 收窄（加速轮动）
       # <0: 整体在收敛，无新信息 → band 放宽（保持黏着）

有效 B = clamp(B − BS × trend, floor, ceiling)
```

### 为什么用分散度而不是局部边际差

1. **学术依据**：分散度是 factor timing 的标准尺度（E(α) = IC × σ_CS）。分散度扩张 = alpha 机会扩大 = 策略应该更主动。
2. **逻辑自洽**：CS 在分散度扩张时让仓位更集中，BS 在同一信号下让分数带收窄——同一个"机会扩大"的判断驱动两个参数往同一方向调整。
3. **对 MH 无依赖**：不管持仓 2 支还是 7 支，全体 z-score 的标准差稳定可比，适合作为各预设共享的信号源。

### 参数分层

**可优化参数**（进 `PARAM_BOUNDS` + Tuner 控件）：

| 缩写 | 参数名 | UI 范围 | 引擎单位 | 默认值 | 含义 |
|------|--------|---------|---------|--------|------|
| **B** | `band` | [0, 20], step 0.5 | ÷100 → [0, 0.20] | 3.0 (→0.03) | 静态基准分数带 |
| **BS** | `band_sensitivity` | [0, 100], step 1 | ÷1000 → [0, 0.10] | 30 (→0.03) | 对分散度趋势的响应度 |

**固定配置参数**（`defaults.yaml`，不进优化器）：

| 参数 | 值 | 含义 |
|------|------|------|
| `fast_span` | 4 | 短期 EMA 周期 |
| `slow_span` | 12 | 长期 EMA 周期 |
| `floor` | 0.015 | band 下限 |
| `ceiling` | 0.045 | band 上限 |

### 为什么 BS 需要可优化

实测显示 gam-0 最优 B=0.03，但不同预设对"动态响应度"的需求不同。CS 的经验：三派默认值 gam=30 / zen=15 / act=25，说明不同风险偏好对同一信号的响应度差异显著。BS 应该让优化器去发现各派的最优值。

### 预期行为

以 gam-0 (B=0.03, BS=0.03) 为例：

| 市场阶段 | 分散度趋势 | band 调整 | 效果 |
|---------|-----------|-----------|------|
| 热点扩散/轮动加速 | trend +0.3 | 0.03 − 0.009 = **0.021** | 收窄，允许更快轮动 |
| 市场平静/分数稳定 | trend ~0 | **0.03** | 等于静态值 |
| 分化消退/回归共振 | trend −0.3 | 0.03 + 0.009 = **0.039** | 放宽，增加黏着 |

band 在 [0.021, 0.039] 内浮动，不偏离静态锚点太远。

### 不做

- 不基于 CV 调节——CV 与替换质量无关
- 不在 Tuner 加控件——动态 band 是引擎行为
- 不把 band 参数加入优化搜索——base_band 已校准
