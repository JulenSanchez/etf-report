# REQ-325: 滑动条 + 前沿驱动 — 替换离散预设按钮

**优先级**: 🟠 P1 | **目标版本**: v3.10.0 | **依赖**: REQ-327 (已完成) | **状态**: 规划中

## 动机

当前赌徒派 4 个离散按钮（gam-0/1/2/3）来自 YAML 硬编码。REQ-327 产出了 376 个 trial 和 11 个帕累托前沿点——远超过 4 个固定档位。用户应能通过连续滑动条在 MDD 轴上自由探索最优参数。

## 设计

### 查找规则（方案 B：保守查找）

```
用户滑块值 = target_MDD（例：-25%）

SELECT point 
FROM frontier_points
WHERE point.mdd >= target_MDD    -- 回撤不超过目标
ORDER BY point.comp DESC
LIMIT 1
```

- 滑块=-25% → 选 MDD≥-25% 中 COMP 最高的点（MDD=-24.4%, COMP=53.1%）
- 滑块=-30% → 选 MDD≥-30% 中 COMP 最高的点（MDD=-29.3%, COMP=92.8%）
- 滑块=-20% → 选 MDD≥-20% 中 COMP 最高的点（MDD=-20.0%, COMP=28.7%）

#### 新函数: `compute_frontier()`

位于 `quant_contract.py`：

```python
def compute_frontier(trials, mdd_range=(-40, -20)):
    """从 trial 列表计算帕累托前沿。
    
    Returns:
        list[dict]: 非支配前沿点，按 MDD 升序排列
    """
    in_range = [r for r in trials 
                if mdd_range[0] <= r['mdd'] <= mdd_range[1] 
                and r.get('composite', -999) > -900]
    sorted_by_mdd = sorted(in_range, key=lambda r: -r['mdd'])
    frontier = []
    max_comp = -999
    for r in sorted_by_mdd:
        if r['composite'] > max_comp:
            max_comp = r['composite']
            frontier.append(r)
    frontier.sort(key=lambda r: r['mdd'])
    return frontier
```
