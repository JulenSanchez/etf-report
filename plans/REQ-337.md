# REQ-337: 三派统一横轴 MDD [-40, -20]

**优先级**: 🔴 P0 | **目标版本**: v3.10.0 | **依赖**: Pool 引擎就绪 | **状态**: 规划中

## 目标

Zen 和 Actuary 改为和 Gambler 一样以 MDD [-40, -20] 为横轴。三派唯一差异 = 纵轴指标（AR / Sortino / Calmar）。

## 改动拆解

### 1. 代码

| 文件 | 改动 |
|------|------|
| `quant_contract.py` — `prune_pool` | 移除 school 分支，统一按 MDD 槽位 (`band`×`per_band`) |
| `iterative_optimizer.py` — `_DF` | zen/actuary 移除 `prune_top`，统一用 `prune_band`/`prune_per_band`；zen 默认 band=1.0 |
| `quant_contract.py` — `build_frontier_output` | 三派统一用 MDD 轴，zen/actuary 也产出 slider 格式 |
| `quant_tuner.py` — `/api/frontier` | zen/actuary 分支从 discrete 改为 slider（已部分就绪） |
| `templates/tuner.html` | zen/actuary 区域渲染散点图 + slider（复用 gambler 逻辑） |

### 2. 数据

| 改动 | 说明 |
|------|------|
| zen pool 裁到 MDD [-40, -20] | 当前 21 条，best Sortino 在 -13.6%（超出范围），裁后顶尖可能下降 |
| actuary pool 裁到 MDD [-40, -20] | 已裁完（53 条），但 prune 键需从 ma_bear_pos 改为 MDD |
| 三派重跑首轮 MDD 范围优化 | 确保在 [-40, -20] 内有充分覆盖 |
