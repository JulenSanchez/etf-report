# REQ-125: 状态栏协议宿主抽离与规则解耦

**优先级**: 🟡 Medium
**估时**: 2-4h
**状态**: done

**目标版本**: -
**依赖**: REQ-123
**创建日期**: 2026-04-15
**最后活动**: 2026-04-17

---

## 动机

`REQ-123` 已完成“协议 + 配置”的首轮落地，但当前状态栏机制仍然挂在 `.codebuddy/rules/etf-report.mdc` 这个项目专属规则文件上。

这会导致一个剩余问题：

- `etf-report` 的业务内容虽然已经对状态网络基本无感知，但**机制本身**仍依赖项目专属入口
- 因此还不能满足“把协议迁到一个全新工作区，再加一份配置文件就能转起来”的目标

本轮需要继续把**通用状态栏协议宿主**从 `etf-report` 老规则中抽出来，让老规则只承担需求看板 / 版本 / Bug 守卫职责。

## 本轮目标

- 新增一个工作区级通用状态栏协议宿主规则
- 将固定三行状态头、渲染约束、配置发现 / 选中逻辑迁入通用宿主
- 将 `.codebuddy/rules/etf-report.mdc` 收窄为项目级需求看板守卫
- 同步 `SKILL.md`、`PLAN.md` 和协议文档中的事实源分工
- 验证 `etf-report` 的状态网络配置仍以 `statusbar.config.md` 为单一事实源

## 范围边界

### 本轮纳入
- 通用宿主规则文件落位
- `etf-report` 项目规则职责收窄
- 文档入口与事实源分工修正

### 本轮不纳入
- 运行时自动缓存与热重载实现
- 多工作区真实迁移演练
- 新的默认兜底配置样例

## 实施步骤

### Step 1: 抽离通用宿主
- 新建通用状态栏协议宿主规则文件
- 承接固定状态头协议与渲染安全约束
- 承接配置发现 / 选中 / 重判原则

### Step 2: 收窄项目守卫
- 删除 `etf-report.mdc` 中与通用协议有关的内容
- 保留入口 / 出口守卫与需求看板治理职责
- 明确项目规则与通用宿主的分工边界

### Step 3: 同步文档锚点
- 更新 `SKILL.md` 的协作模式说明
- 更新 `PLAN.md` 的事实源表与维护原则
- 更新协议草案中的当前分工描述

## 验证方式

| 验证项 | 方法 | 通过标准 |
|--------|------|---------|
| 协议宿主已独立 | 读规则文件 | 状态头协议不再写在 `etf-report.mdc` |
| 项目守卫职责清晰 | 读规则文件 | `etf-report.mdc` 只保留看板 / 版本 / Bug 守卫 |
| 文档口径一致 | 走查 `SKILL.md` / `PLAN.md` / 协议草案 | 不再把老规则误写成协议宿主 |
