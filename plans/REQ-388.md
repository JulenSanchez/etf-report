# REQ-388: Tuner HTML 面板级拆分 — L1 三文件

**优先级**: 🟡 P1 | **状态**: ✅ 已交付 | **目标版本**: v3.14.0 | **日期**: 2026-07-21

## 背景

`templates/tuner.html` ~900 行单文件，292 对 `<div>`。每次 Edit HTML 有非零概率破坏嵌套，导致面板布局崩溃（已多次发生：rebalance 锁定、conf-type 锁定、signal_steps 样式切换）。AI 辅助编辑下这个问题尤为突出——替换一段含 `<div>` 的 HTML 时，旧代码的闭合标签可能和外层共享。

## 方案：L1 面板级拆分

```
tuner.html           (41行)   骨架：head/top bar/panels 空壳/CSS/JS 引用 + sync XHR 加载
tuner-left.html     (245行)   左面板：所有参数区，89 对 div
tuner-right.html    (565行)   右面板：结果区，193 对 div
```

### 加载流程（sync XHR + document.write）

```html
<div id="tuner-panels">
  <script>
   var x = new XMLHttpRequest();
   x.open('GET', '/static/tuner-left.html', false); x.send();
   document.write(x.responseText);
   x.open('GET', '/static/tuner-right.html', false); x.send();
   document.write(x.responseText);
  </script>
</div>
```

同步 XHR 在线脚本中执行，`document.write` 在当前解析位置写入 DOM。面板 DOM 在 tuner.js 的 `load` 事件触发前已就绪，无需改 JS。

### 不改

- JS/CSS 文件不动（DOM 结构、id、class 全部不变）
- 事件委托机制不动
- Flask 路由不动（`send_from_directory` 发静态 HTML）
- CSS 文件引用位置不动

## 收益

- Edit 不再跨文件，div 嵌套错误概率趋零
- 单文件 ~150-400 行，Read 一次可读全
- 后续可局部下沉到 L2（section 级拆分），不阻塞
