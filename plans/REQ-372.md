# REQ-372: 回测输出分级（静默 / 进度 / 详细）

**优先级**: 🟡 P1 | **状态**: ✅ 已交付 | **目标版本**: v3.13.0

## 背景

`run_backtest()` 内部有 29 处裸 `print()`，当前 `verbose` 参数在签名中存在但从未被消费（REQ-368 的守卫被后续损坏版本抹掉）。单次调试和批量 sweep 对输出的需求不同：

| 场景 | 期望 |
|------|------|
| Tuner 单次回测 | 完整输出（区间、NAV、指标） |
| 批量 sweep | 仅进度（如 `MH=3 TB=5 done`） |
| CLI 手动跑 | 完整输出 |

## 设计

三级输出：

```
verbose=2  完整输出（当前行为，Tuner / CLI 手动）
verbose=1  进度条（每 20 日 + 最终指标一行）
verbose=0  完全静默
```

或更简单的布尔 + 进度回调：

```
verbose=True   完整输出
verbose=False  静默（调用方自己按需打印）
progress_cb    进度回调（已在签名中，但未被所有 print 使用）
```

## 改动范围

`scripts/quant_backtest.py`：29 处 `print()` 加 `if verbose:` 守卫，进度条单独保留或走回调。

调用方传参更新：`quant_tuner.py`、`iterative_optimizer.py`、`pareto_optimizer.py` 等批量调用方传 `verbose=False`。
