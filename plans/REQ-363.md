# REQ-363: ETF 池稳健性研究 — 随机子集 + leave-one-sector-out

**优先级**: 🟡 P1 | **状态**: done | **目标版本**: v3.15.0 | **完成日期**: 2026-07-22

## 背景

2026-07-08 讨论"ETF 池子设定对策略的影响"时产生两个方向：
- **A**：随机池子集 → 评估策略稳健性 + 发现关键 ETF
- **B**：动态池/扇区轮动 → 牛市去传统、熊市去科技

经分析，方向 A 不改引擎、成本低、信息量高，且其产出可直接为方向 B 提供决策依据。方向 B 的前提假设（"熊市去科技能提升回报"）尚无数据支撑，应先验证再投入引擎改动。

### 现状事实

- 池子 54 支 ETF，11 个扇区，集中度 HHI=0.102（均匀分布）
- 3 个"薄扇区"：医药(3)/平台经济(3)/新能源(3)，去掉成本低
- 回测引擎已有 `--universe` flag 支持子集筛选
- 每支 ETF 已在 `config/quant_universe.yaml` 中标 `sector`
- 无需改引擎即可完成全部调研
- 研究产出目录：`research/pool/`（换池证据、候选记录见 `research/pool/rounds/`）

## 目标

回答三个问题：

| # | 问题 | 方法 |
|---|------|------|
| 1 | 策略表现是靠少数 ETF 撑起来的，还是普适于全池？ | 随机子集回测（n≥50），看 Sharpe/MDD 分布 |
| 2 | 哪些扇区对策略是正向/负向贡献？ | leave-one-sector-out（11 次回测），对比全池基线 |
| 3 | 池子大小是否有最优区间？ | 不同 k 值（10/15/20/27/35/54）的 Sharpe 均值+方差曲线 |

## 范围

### Phase 1: leave-one-sector-out（最优先）

对每个扇区单独移除，跑 gam-0 × 6Y 窗口（2020-07-03 ~ latest）：

```
pool_all (baseline)   → TR, Sharpe, MDD
pool - 科技(7支)       → TR, Sharpe, MDD, ΔSharpe
pool - 另类(8支)       → ...
pool - 传统(7支)       → ...
pool - 资源周期(5支)    → ...
pool - 金融(5支)       → ...
pool - 海外科技(5支)    → ...
pool - 制造(4支)       → ...
pool - 消费(4支)       → ...
pool - 医药(3支)       → ...
pool - 平台经济(3支)    → ...
pool - 新能源(3支)      → ...
```

人工判断：

- ΔSharpe > 0 → 该扇区是拖后腿的（去掉反而更好）
- ΔSharpe < 0 → 该扇区是正向贡献的（去掉损失 alpha）
- |ΔSharpe| < 0.05 → 该扇区对策略影响不大

### Phase 2: 随机子集分布

- 固定 gam-0，k ∈ {10, 15, 20, 27, 35, 54}
- 每档 n=20 次回测（初始，如果分布很宽再扩到 n=50）
- 每次从 54 支中纯随机抽样
- 收集：TR, Sharpe, MDD, AR，画分布箱线图 + k-Sharpe 曲线

### Phase 3: 扇区分层 vs 纯随机对比（如 Phase 1/2 有明确信号则做）

- 对比 `random.sample(54, k)` vs `stratified（每扇区至少 1 支 + 剩余随机）`
- 固定 k=27（池子一半）
- n=20 次每组

### 不改的范围

- 不碰回测引擎
- 不调 gam-0 参数
- 不做动态池引擎改动
