# BUG-035: nav-chart 红竖线反复失效

**严重度**: 🟡 Medium | **状态**: fixed | **引入版本**: v3.0.0 | **发现日期**: 2026-06-22 | **修复日期**: 2026-06-22

---

## 症状

Tuner nav-chart 中选中日期红色 markLine 反复出问题：不动/两条/消失。

## 根因（确认）

**两个函数在写同一条 markLine，产生竞态。**

```
renderNavChart:     setOption({...}, true)   → notMerge=true，全量替换，包含 markLine
updateNavMarkLine:  setOption({series:[{markLine}]}) → notMerge=false，部分合并
```

ECharts 默认合并模式下，`markLine.data` 是数组，合并行为是**追加**而非替换。所以：
1. `renderNavChart` 画了一条线在 `curDate`
2. `updateNavMarkLine` 想移到新日期 → 实际上在 `data` 数组里**追加**了一条
3. 结果：两条线

kline-replay 只有 1 条 series + `renderNavChart` 的 full replace 每次都清空重建 → 碰巧不受影响。

## 修复方案

**唯一原则：只有一个人管这条线。**

具体做法：`renderNavChart` 不再画 markLine（移除 `markLine: curDate ? {...}` 块）。`updateNavMarkLine` 成为唯一的 markLine 管理者。关键修复：

```javascript
function updateNavMarkLine() {
  if (!navChart || !currentSelectedDate) return;
  // 先清空 series[0] 的 markLine，再设新的
  navChart.setOption({ series: [{ markLine: null }] }, false);
  navChart.setOption({ series: [{ markLine: {...} }] }, false);
}
```

`markLine: null` 清掉旧线，然后设新线。两次 `setOption` 都是 partial merge，不碰 series 数据。

`renderNavChart` 末尾调用 `updateNavMarkLine()` 完成初始画线。

## 验证

1. 初始渲染 → 一条红线在最新日期
2. 点击 snapshot → 红线移动到选中日期
3. 切换周/日频 → 红线保持在当前选中日期
4. 多次切换 → 始终只有一条红线

## 预案：如果这次还不行

| 排查方向 | 具体动作 |
|---------|---------|
| 确认 ECharts 版本 | `echarts.version`，检查 markLine 合并文档 |
| 绕过 ECharts merge | 不碰 markLine API，改用 `series[0].markLine` 直接赋值 + `setOption(notMerge=true)` 全量写回 |
| 放弃 markLine | 在 chart 上 overlay 一个绝对定位的 `<div>` 画 CSS 红线，依赖 `convertToPixel` 算 x 坐标 |
| 对比 kline-replay 差异 | 完整 diff `renderKlineReplay()` vs `renderNavChart()`，找 nav-chart 独有的配置项是否干扰 markLine |
