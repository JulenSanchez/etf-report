## 动机

当前周频调仓陷入一个恶性循环：

```
周调仓 → F2 埋没（日频信号周频执行=无效）→ F2 归零
  → 只剩 F1+F3 → 分差太小（25 支挤在 65-75）
  → 无多因子共振 → 无重仓超额能力
  → 最佳策略退化为纯择时策略（靠信心函数躲暴跌）
```

典型案例：传媒 ETF 2026-01-14 天量见顶，1-16 大幅回落，周调仓死拿至周五才卖，利润大幅回吐。F2 天然是日频过热退出信号，但周调仓埋没了它。

日调仓是解开死结的必要条件，但面临磨损大的问题。分数带让日调仓只在真正有信号差异时才执行交易，大部分交易日换仓频率接近周频，但 F2 触发过热时分数显著拉开，自然逃顶。

## 核心思路

**日打分日调仓 + 分数带，让持仓稳定性从机制中自然涌现。**

- 所有标的用同一套标准打分，一视同仁（**禁止**持仓状态影响打分）
- 新标的的分数必须超过当前持仓**超过一个带宽**，才触发换仓
- 分数差异小于带宽 = 不具备统计显著性，不足以支撑交易决策
- 带宽阈值可通过 Tuner 建模确定

## 实施路径

### Phase 1：调仓频率参数化

在 `quant_backtest.py` 新增 `rebalance_freq` 参数：
- `weekly`（当前默认，保持兼容）：每周五调仓
- `daily`：每个交易日调仓

Tuner UI 新增调仓频率切换控件。

**预期**：周频下最佳策略仍落在 (20,0,80,0)——纯择时策略包圆。日频下（无分数带时）磨损大、换仓频繁，但 F2 有机会参与。

### Phase 2：分数带机制

在调仓逻辑中加入分数带过滤：

```
对当前持仓的每支 ETF：
  如果存在未持仓标的，分数超过该持仓 > score_band → 允许换仓
  否则 → 保持持仓
```

**预期**：日频 + 分数带下，换仓次数回落到接近周频水平，但 F2 过热信号不被阻挡。最佳策略有望落在 F2>0 的区间——F2 被激活，多因子共振，分差拉开。

### Phase 3：双冠对比（完成 2026-04-30）

**周调仓冠军** (w1=40, w2=0, w3=60, sb=0, W-FRI): 年化 94.6%, Sharpe 2.78, MDD -13.0%, Calmar 7.27, 换仓 43/52, 佣金 1.18%

**日调仓冠军** (w1=40, w2=5, w3=55, sb=5%, daily): 年化 103.9%, Sharpe 2.93, MDD -13.5%, Calmar 7.69, 换仓 36/243, 佣金 0.74%

**结论**：日调仓+分数带在所有核心指标上优于周调仓——年化+9.3%, Sharpe+0.15, Calmar+0.42, 佣金-0.44%。F2 在日频下被有效激活（w2=5 贡献 5.5% 年化增量），而在周频下 w2=5 反而伤害（-13.7%）。分数带 sb=5% 将日频换仓从 129 次压至 36 次，佣金从 2.20% 降至 0.74%。

**验证结果**：

| 验证项 | 结果 | 通过？ |
|--------|------|--------|
| 周调仓兼容 | 周频结果与此前一致 | ✅ |
| 日调仓回测 | daily+band 年化/Sharpe/Calmar 均优于 weekly | ✅ |
| 分数带稳定性 | daily+sb=5% 换仓36次，接近周频43次 | ✅ |
| F2 激活 | 最佳策略 w2=5（F2>0 区间） | ✅ |
| 传媒 ETF 逃顶 | F2 过热信号在日频下不被阻挡 | ✅（逻辑验证） |
| 佣金损耗 | daily+band 净值高于 weekly | ✅ |

跑出两个冠军后对比：
- **周调仓冠军**：纯择时策略（信心函数躲暴跌），F2/F4/F5 归零
- **日调仓冠军**：择时 + 择股策略，F2 过热退出贡献，分数带过滤噪声

对比维度：年化收益、最大回撤、换仓次数、持仓集中度、F2 有效信号触发次数。

对比目标：产生新的思路和碰撞，看是否涌现出新的策略方向。

## 设计讨论记录

### 已否决的方案

| 方案 | 否决理由 |
|------|---------|
| 黏着度/鲜度（持仓有禁售衰减期） | 持仓状态影响决策，违反量化公正原则 |
| 周调仓 + 日频 F2 止盈（双层拆分） | 因子工作拆开不协同，F2 不应只做"止盈器" |

### 已认可的方案

- **分数带**：微小的分数差异不构成换仓依据，与交易成本逻辑一致（换仓成本 ~0.05%，分数差异 <1% 不够覆盖）
- 与 F2 dead_zone 设计理念一致：同一个哲学——用阈值过滤噪声
- 日调仓 + 分数带不是两个功能，是同一个方案的两面：日调仓释放 F2，分数带控制磨损

### 已结论的问题

1. **分数带阈值如何定** → **固定值，5%为最优**。数据：sb=0→88.2%, sb=3%→95.8%, sb=5%→98.3%, sb=7%→98.0%, sb=10%→88.6%。自适应方案暂不需要——5%在当前25支ETF池+综合分0-1尺度下稳定有效，若未来ETF池或因子结构大变再考虑。
2. **日调仓下信心函数是否需要调整** → **不需要**。数据：dead_zone/full_zone对日频结果极不敏感（dz=5~20, fz=50~70，年化几乎不变），说明信心函数在日频下仍稳健，无需额外band。
3. **交易成本建模** → **已完成（REQ-186）**。佣金率0.00026已写入yaml，日调仓+sb=5%下佣金仅0.74%，低于周调仓的1.18%，成本可控。
4. **仓位差异化** → **留待未来**。当前日调仓+w2=5已让分差拉开（F2贡献5.5%年化增量），但分差是否足够支撑非等权还需更多分析。这是一个独立议题，不阻塞REQ-185关闭。
