# REQ-292: 量化回测池四类聚合标签

**创建**: 2026-06-18 | **状态**: backlog | **优先级**: 🟡 Normal | **目标版本**: v4.0.0/v4.1.0

## 背景

当前 `config/quant_universe.yaml` 中的 `sector` 主要用于人工查看、Tuner/优化报告归因展示，不参与回测评分、选股或仓位约束。

本需求不是建立严格行业 taxonomy，也不追求申万/中信行业口径。用户只需要一个人工查看用的聚合概念：**4 个一级分类，每个一级分类下面再有若干二级分类**。分类只要有稳定逻辑、便于扫盘和归因即可。

现有标签的问题：

1. `另类` 过载：混合 QDII、海外主题、黄金、豆粕、微盘等，人工查看时信息密度低。
2. `传统` 过宽：混合电力、农业、地产链、红利防御等不同风险源。
3. `科技` / `TMT` / `制造` 边界不清：机器人、工业母机、通信、内容互联网等主题容易混放。
4. 单个 `sector` 很难表达“聚合桶 + 子类”的人工查看结构。

## 目标

1. 保留 `sector` 兼容旧逻辑。
2. 新增人工查看用的一级/二级聚合标签。
3. 一级分类固定为 4 类，二级分类用于解释该 ETF 的主要风险来源。
4. 不改回测选股逻辑，不引入板块上限或行业轮动约束。

## 非目标

- 不直接改变策略收益、评分因子、仓位权重。
- 不把分类作为策略硬约束。
- 不追求严格行业分类。
- 不重做 ETF 全市场筛选逻辑。

## 推荐字段

保留旧字段：

```yaml
sector: 科技
```

新增两个展示字段：

```yaml
group1: 成长进攻
group2: 科技硬件
```

含义：

- `group1`：一级聚合，只能取 4 类之一。
- `group2`：二级聚合，用于解释更具体的风险来源。
- `sector`：旧标签，短期保留，避免影响既有代码和报告。

## 一级分类方案

### 1. 成长进攻

逻辑：靠景气、技术扩散、估值弹性、产业趋势吃饭。

| 二级分类 | 放哪些 ETF |
|---|---|
| 科技硬件 | 通信、芯片、卫星、消费电子、科创新材料 |
| 互联网内容 | 传媒、游戏 |
| 医药成长 | 创新药、A股创新药、医疗器械 |
| 新能源 | 电池、光伏、电网设备 |
| 高端制造 | 机器人、军工、工业母机 |

备注：船舶、工程机械偏订单/周期，不放成长进攻。

### 2. 周期弹性

逻辑：受商品价格、宏观周期、地产链、金融 beta 影响大，涨起来猛，回撤也猛。

| 二级分类 | 放哪些 ETF |
|---|---|
| 资源能源 | 有色、煤炭、石油 |
| 材料化工 | 化工、钢铁 |
| 金融地产 | 非银、证券、银行、金融科技、房地产、建材 |
| 周期制造 | 船舶、工程机械 |

备注：银行可被视为防御，但在当前池子里和金融/地产 beta 更接近，先归入周期弹性。

### 3. 内需防御

逻辑：偏国内需求、民生、公用、红利，更偏稳态/防守/现金流视角。

| 二级分类 | 放哪些 ETF |
|---|---|
| 消费内需 | 酒、旅游、家电 |
| 农业民生 | 养殖、粮食 |
| 公用事业 | 电力 |
| 红利防御 | 红利 |

备注：旅游有复苏弹性，但比放成长/周期更适合作为内需观察桶。

### 4. 跨境另类

逻辑：不是纯 A 股行业逻辑，主要受海外市场、港股市场、商品、风格因子驱动。

| 二级分类 | 放哪些 ETF |
|---|---|
| 港股主题 | 港券、港股红利、港股信息、中概互联、港股消费 |
| 海外宽基 | 纳指、日经、德国、巴西、沙特、亚太精选 |
| 海外主题 | 纳指科技、中韩半导体、标普生物、标普油气 |
| 商品 | 黄金、豆粕 |
| 风格市值 | 微盘 |

这样可以避免争论“港股消费算消费还是 TMT”“中韩半导体算科技还是 QDII”：一级统一放跨境另类，二级再解释具体风险来源。

## 推荐落地方案

### Phase 1: 配置补字段

在 `config/quant_universe.yaml` 的每支 ETF 条目上补充：

```yaml
group1: 成长进攻
group2: 科技硬件
```

只补展示字段，不改变 active、preset、因子、回测逻辑。

### Phase 2: Tuner / 报告展示使用新字段

- Tuner ETF 列表展示 `group1 / group2`。
- ETF contribution 归因可按 `group1` 汇总，再按 `group2` 展开。
- 优化报告中避免把黄金、纳指、微盘都归为同一个“另类”。

### Phase 3: 旧 `sector` 兼容

- 短期保留 `sector`，避免破坏现有读取逻辑。
- 展示层优先用 `group1/group2`。
- 若后续确认没有代码依赖 `sector`，再考虑降级为历史字段。

## 验收标准

1. 54 支 ETF 都有 `group1` / `group2`。
2. `group1` 只出现 4 个值：成长进攻、周期弹性、内需防御、跨境另类。
3. 同一一级分类下的二级分类具有可解释的聚合逻辑。
4. 默认回测结果不因本需求改变。
5. 相关测试通过：

```bash
python -m pytest tests/test_quant_* -q
```
