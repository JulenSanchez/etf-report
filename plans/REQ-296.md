# REQ-296: Stage A 合成杠杆账户模式与 exposure 引擎

**创建**: 2026-06-18 | **状态**: backlog | **优先级**: 🔴 P0 | **目标版本**: v3.9.0

## 背景

`docs/design/margin-account-model.md` 将杠杆账户拆成 Stage A / Stage B。v3.9 先做 Stage A：零融资成本、无追保/强平的 `synthetic_leverage` 研究模型。

本需求负责最底层引擎能力：让回测可以在显式账户模式下支持 `ma_bull_pos > 1.0`，并输出 exposure 序列。

## 目标

1. 新增账户模式：`cash` / `synthetic_leverage`。
2. cash 模式行为与当前完全一致。
3. synthetic_leverage 模式允许 `ma_bull_pos <= 2.0`。
4. 引擎按 `actual_exposure` 放大组合日收益。
5. 输出每日 exposure 序列。

## 非目标

- 不计融资成本。
- 不建真实 margin debt。
- 不模拟追保/强平。
- 不改生产 preset 默认值。

## 设计约束

```text
cash:
  max_gross_exposure = 1.0
  ma_bull_pos <= 1.0

synthetic_leverage:
  max_gross_exposure <= 2.0
  ma_bull_pos <= max_gross_exposure
```

强趋势门槛第一版：

```text
if not (hs300_above_ma and hs300_ma_rising):
    actual_exposure = min(actual_exposure, 1.0)
```

回撤刹车第一版：

```text
if strategy_drawdown < -10%:
    actual_exposure = min(actual_exposure, 1.0)
```

## 预计改动区域

- `src/etf_report/core/quant_contract.py`
- `scripts/quant_backtest.py`
- `config/quant_universe.yaml`
- `tests/test_quant_*`

## 验收标准

1. cash 模式基线不漂移。
2. synthetic_leverage 模式可跑 `ma_bull_pos > 1.0`。
3. `actual_exposure` 不超过 `max_gross_exposure`。
4. 非强趋势或触发回撤刹车时，exposure 回落到 `<= 1.0`。
5. `python -m pytest tests/test_quant_* -q` 通过。
