# REQ-181: 桌面版仪表盘重设计（index.html 全新架构）

> **设计原型参考**：[req181-layout-mockup.html](req181-layout-mockup.html) — 三种布局方案（A: Dashboard Grid / B: Split Panel / C: Overview Table）的交互原型，浏览器直接打开即可查看

## 概述

**端侧分治**：桌面版和移动/传播版彻底分开。

- **REQ-181（本需求）**：index.html 完全重设计为桌面仪表盘，横屏优先、强交互
- **REQ-182**：基于当前 index.html 冻结版本，输出 PNG 海报 / 公众号摘要

两端共享同一份数据 pipeline（`update_report.py` 一次生成，多端消费），
单元 UI 元素（卡片、水位条、涨跌色、判词 chip）保持视觉一致。

**前置已满足**：
- REQ-178: 25 支 ETF 估值引擎全覆盖 (done)
- REQ-177: 多因子量化打分系统 v1 (done)

## 设计原则

1. **横屏优先，不妥协移动端** — 移动端由 REQ-182 海报版承担
2. **仪表盘形态，不是长报告** — 一屏看全局，不翻页
3. **全量 25 支 + 量化高亮** — 结构稳定，只有高亮在动
4. **首页核心价值**：3 秒回答"今天该关注谁"

## 整体架构（全局 Split）

既然不再顾虑移动端，可以采用更激进的 B1 全局 Split：

```
┌──────────────┬─────────────────────────────────┐
│              │ ① 顶栏: 日期 / 全局指标 / 摘要     │
│  25 ETF      ├─────────────────────────────────┤
│  侧栏列表     │ ② 主面板: 随侧栏选中项切换         │
│  (~280px)    │                                 │
│              │    默认视图: 总览对比表（C 方案）    │
│  按 Score 排  │    选中某支: 详情面板（K线/估值/..） │
│  🔥标记 Top  │    量化面板 tab                   │
│  mini 估值条  │    Editorial tab                 │
│  1D 涨跌     │                                 │
│              │                                 │
└──────────────┴─────────────────────────────────┘
```

### 侧栏（左, 固定 280px）
- 25 支 ETF，按 Score 降序
- 每条: 代码+简称 / 板块 / Score / mini 估值条 / 1D 涨跌
- 🔥 + 绿色左边框标记 Top Pick
- 点击切换右侧面板内容
- 底部可加"全局总览"入口回到默认视图

### 主面板（右, 自适应宽度）

**默认视图 — 总览对比表**：
- 25 行 × 12 列表格（C 方案）
- 列: ETF / 行业 / 1D / 5D / 20D / 估值mini条 / 判词 / 总分 / F1 / F2 / F3 / F4
- 🔥 + 行背景色标记 Top Pick
- 表头可点击排序
- 综合分纯数字（无进度条）

**选中视图 — ETF 详情**：
- 点击侧栏某支 → 右侧切换为该 ETF 完整详情
- K 线图（全宽，~1640px，比现在还大）
- 估值水位（大版水位条 + 百分位 + 判词 + 历史分布）
- 持仓 Top 10
- 资金流
- 详情内可用 tab 或纵向滚动组织子模块

**其他 tab**：
- 量化面板（现有 quant-snapshot 的图表类内容）
- Editorial（政策/宏观资讯）
- 可通过顶栏 tab 或侧栏底部入口切换

## 总览对比表 列规划

| # | 列名 | 宽度 | 说明 |
|---|------|------|------|
| 1 | ETF | 120px | 代码 + 简称，Top 行前加 🔥 |
| 2 | 行业 | 60px | 板块标签 |
| 3 | 1D | 60px | 昨日涨跌%，红绿色值 |
| 4 | 5D | 60px | 5 日涨跌% |
| 5 | 20D | 60px | 20 日涨跌% |
| 6 | 估值 | 100px | mini 水位条 + 百分位% |
| 7 | 判词 | 80px | emoji + 文字 |
| 8 | 总分 | 50px | 数字加粗（无进度条） |
| 9 | F1 | 40px | EMA 偏离分 |
| 10 | F2 | 40px | RSI 自适应分 |
| 11 | F3 | 40px | 量比分 |
| 12 | F4 | 40px | 估值分 |

### 视觉分层
- 涨跌列：只用色值（绿/红），不加 emoji
- 估值列：emoji + 色值 + 文字
- 因子分列：极小字号，灰色调
- 密度对标富途 watchlist

## 与现有模块的关系

| 现有模块 | REQ-181 处理方式 |
|---------|----------------|
| 6 支 ETF 详情 section | 替换为 Split Panel 右侧详情视图 |

| Editorial section | 移入主面板某个 tab |

## 实施阶段

- **M1**: 数据层准备 — 25 支全量 JSON payload（涨跌/估值/因子分/持仓），`update_report.py` 输出
- **M2**: 总览对比表 — 独立 section，数据驱动，排序，🔥高亮
- **M3**: 全局 Split 骨架 — 侧栏 + 主面板框架，路由切换
- **M4**: 详情视图 — K 线/估值/持仓/资金流模板化（从硬编码 6 支 → 数据驱动）
- **M5**: 其他 tab 迁移 — 量化图表、Editorial 嵌入主面板

## 不做的事

- 不做移动端适配（归 REQ-182）
- 不做主题切换（归 REQ-167）
- 不做 SPA 框架（保持原生 HTML/JS/CSS）
