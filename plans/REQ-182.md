# REQ-182: 移动/传播版海报生成 + 自动发布流程

## 概述

基于当前 index.html（v2.5.x）冻结为"海报模板"，输出 PNG 摘要图，
用于公众号 / 企微群 / 社交分享等移动端场景。

**与 REQ-181 的关系**：端侧分治
- REQ-181 = 桌面版仪表盘（index.html 全新重设计）
- REQ-182 = 移动/传播版海报（基于冻结模板生成 PNG）
- 两端共享同一份数据 pipeline + 单元 UI 元素样式

## 核心理念

- 当前 index.html 竖版长页面天然适合做海报：信息块纵向堆叠，宽度固定
- **冻结当前版本作为海报模板**，不再跟随桌面版 UI 迭代
- 海报是桌面版的**摘要版**：同一份数据，但只展示 Top 4~6 推荐 + 全局指标
- 后续可接通自动发布管线（企微机器人 / 公众号草稿箱）

## 技术路径

### 生成方式：HTML → PNG

- 用 Playwright / Puppeteer headless 截图，最成熟的 HTML→PNG 方案
- 或 `wkhtmltoimage`（轻量但 CSS 支持弱）
- 推荐 Playwright：`pip install playwright && playwright install chromium`

### 模板管理

```
templates/
  poster-daily.html    # 冻结自当前 index.html，精简为海报版
  poster-weekly.html   # 周报摘要（可选，后续）
```

- 海报模板从 index.html fork 一次，之后独立维护
- 单元 CSS 样式（卡片/水位条/涨跌色/判词 chip）提取为共享 CSS
- 数据注入方式与现有 `inject_valuation_blocks` 类似

### 海报内容（日报版）

```
┌─────────────────────┐
│ ETF 日报 2026-04-28  │
│ 数据截止 15:00       │
├─────────────────────┤
│ 全局: 低估6 合理11   │
│       偏高4 极高4    │
├─────────────────────┤
│ 🔥 量化推荐 Top 4    │
│ ┌─────────────────┐ │
│ │ 512690 酒ETF     │ │
│ │ PE 0.2% 极度低估  │ │
│ │ 1D -0.21%       │ │
│ └─────────────────┘ │
│ (repeat × 4)        │
├─────────────────────┤
│ 25 支估值分布概览     │
│ (紧凑列表或热力图)    │
├─────────────────────┤
│ 数据来源 / 免责      │
└─────────────────────┘
宽度: 750px (手机友好)
```

### 生成流程

```python
# update_report.py 扩展
def generate_poster():
    1. 读取同一份 JSON payload
    2. 注入 poster-daily.html 模板
    3. Playwright 截图 → output/poster_20260428.png
    4. (可选) 推送企微机器人 / 上传公众号草稿
```

## 实施阶段

- **M0**: 从当前 index.html fork 海报模板 `poster-daily.html`，精简内容
- **M1**: 写 `scripts/poster_generator.py`（Playwright 截图）
- **M2**: 接入 `update_report.py` 流程（`--poster` flag）
- **M3**: 企微机器人推送（可选）
- **M4**: 公众号草稿箱对接（远期）

## 不做的事

- 不在海报模板里加交互（纯静态截图）
- 不做海报模板的 responsive（固定 750px 宽）
- 不跟随桌面版 UI 迭代（独立维护）

## 与 CodeBuddy 发布链路的关系

海报生成属于"产出物"，但推送动作（企微/公众号）属于发布。
按双 IDE 节奏协议：
- Claude 侧：可以生成 PNG 到本地
- CodeBuddy 侧：负责实际推送（`--publish` flag 控制）
