## 动机

> REQ-170 / REQ-172 开发过程中接触了 AKShare / Tushare / 集思录 / 韭圈儿 / 东财 / 新浪 / 雪球 / 中证官网 / 国家统计局 / 百度股市通 / 乐咕乐股等 10+ 数据源。用户反馈：
> 1. 做不出来不重要，**过程中接触到的工具、项目、网站本身就是收获**
> 2. 这些信息应该"**综合地沉淀为信源获取的知识**"，**持续进化**，说不定某天会深挖其中一个
> 3. 作者无 Web/App 开发背景，**需要大量科普才能避免踩坑**（反爬、PoC、ToS 这些术语都要解释）
> 4. 需要"**和外部渠道打交道时的风险说明**"，保持兴趣项目的合规边界

**不做会怎样**：
- 每次接触新数据源都从头调研，重复劳动
- 踩过的坑记不住（Tushare 定价误记、csindex PB 接口踩雷等）
- 合规边界不清晰，未来做公开展示时可能越界

## 演化规则

### 何时更新 `DATA_SOURCES_LANDSCAPE.md`
- 新增数据源接入（不论成功失败）
- 发现现有数据源的新限制 / 新接口
- 定价 / 积分规则变动
- 某个数据源关闭或格式大改

### 何时更新 `EXTERNAL_DATA_COMPLIANCE.md`
- 遇到新的技术术语（用户或 AI 理解有困难时追加到扫盲区）
- 踩新坑（追加到 FAQ 或红线清单）
- 本项目定位变化（从 Level 3 → Level 2/4）
- 法律/监管变化（如《数据安全法》新规）

### 两份文档的联动
- `LANDSCAPE` 讲"有什么"和"怎么用"
- `COMPLIANCE` 讲"能用到什么程度"和"怎么保护自己"
- 互相引用，新增时双向同步
