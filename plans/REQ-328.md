# REQ-328: 多 target 补跑 — 填补 gambler 前沿空白

**优先级**: 🟠 P1 | **目标版本**: v3.10.0 | **依赖**: REQ-327 (已完成) | **状态**: ⚠️ 数据已失效 (2026-06-29)

> ⚠️ 2026-06-29: 优化目标从伪COMP修正为纯3Y AR。下面记录的 2071 trials 使用旧指标，**全部失效，需重跑**。保留作为多 target 覆盖策略的方法验证证据。

## 问题

REQ-327 只用了一个 `target_mdd=-25` 跑 5 轮。产出的帕累托前沿在两端薄弱：

| 区间 | 问题 | 证据 |
|------|------|------|
| -40%~-30% | 全部映射到 -29.3% 一个点 | 无更激进端的高 COMP 点 |
| -24%~-20% | COMP 偏低 (28~53%) | 28.7% vs 甜蜜点 92.8% |

保守端缺数据尤其影响滑动条体验——用户滑到 -20%~-24% 时只能拿到 COMP 远低于甜蜜点的参数组。

## 方案

用 `scripts/iterative_optimizer.py` 对 4 个 target_mdd 分别跑独立优化。

```
target_mdd=-20:  3 轮 x 20 trial -> 填补保守端
target_mdd=-30:  3 轮 x 20 trial -> 填补中段
target_mdd=-35:  3 轮 x 20 trial -> 填补激进端
target_mdd=-40:  3 轮 x 20 trial -> 填补极激进端
```

所有 trial 各自独立产出 sub-pareto，最后合并：

```
合并 -> compute_frontier() -> 一条完整前沿
```

### 参数

```bash
# 保守端
python scripts/iterative_optimizer.py --target-mdd -20 --mdd-margin 10 --trials-per-round 20 --max-rounds 3 --output research/params/iter_mdd20

# 中段
python scripts/iterative_optimizer.py --target-mdd -30 --mdd-margin 10 --trials-per-round 20 --max-rounds 3 --output research/params/iter_mdd30

# 激进端
python scripts/iterative_optimizer.py --target-mdd -35 --mdd-margin 10 --trials-per-round 20 --max-rounds 3 --output research/params/iter_mdd35

# 极激进端
python scripts/iterative_optimizer.py --target-mdd -40 --mdd-margin 5  --trials-per-round 20 --max-rounds 3 --output research/params/iter_mdd40
```

- mdd_margin 缩小到 +/-10%（之前 +/-15%），减少跨区重叠
- trials 减少到 20（之前 30），因为已有 376 个 warm-start 点提供初始界
- 4 个任务完全独立可并行

### 合并产出

```bash
python -c "
from etf_report.core.quant_contract import compute_frontier
import json, pathlib

all_data = []
for p in pathlib.Path('research/params').glob('iter_mdd*/pareto.json'):
    all_data.extend(json.loads(p.read_text('utf-8')))

frontier = compute_frontier(all_data, mdd_range=(-40, -20))
pathlib.Path('research/params/frontier_gambler.json').write_text(
    json.dumps({'gambler': {'type': 'slider', 'points': frontier}},
               ensure_ascii=False, indent=2))
print(f'Merged {len(all_data)} trials -> {len(frontier)} frontier points')
"
```

## 预期结果

合并后前沿应有 10-20 个非支配点，均匀覆盖 -40%~-20% MDD，每 2-3pp MDD 间隔至少有一个点。

## 验收

```bash
# 1. 4 个 target 各自产出 pareto.json
ls research/params/iter_mdd20/pareto.json
ls research/params/iter_mdd30/pareto.json
ls research/params/iter_mdd35/pareto.json
ls research/params/iter_mdd40/pareto.json

# 2. 合并前沿覆盖全面
python -c "
import json, pathlib
f = json.loads(pathlib.Path('research/params/frontier_gambler.json').read_text('utf-8'))
pts = f['gambler']['points']
mdds = [p['mdd'] for p in pts]
assert min(mdds) >= -21, f'Conservative end missing'
assert max(mdds) <= -38, f'Aggressive end missing'
assert len(pts) >= 10, f'Only {len(pts)} frontier points'
print(f'Frontier: {len(pts)} points, MDD [{min(mdds):.1f}%, {max(mdds):.1f}%]')
print('PASS')
"

# 3. slider lookup at multiple positions
python -c "
import json, pathlib
f = json.loads(pathlib.Path('research/params/frontier_gambler.json').read_text('utf-8'))
pts = f['gambler']['points']
for t in [-20, -22, -25, -28, -30, -33, -35]:
    candidates = [p for p in pts if p['mdd'] >= t]
    assert candidates, f'No candidate for target={t}%'
    best = max(candidates, key=lambda p: p['comp'])
    print(f'  slider={t:+.0f}% -> MDD={best[\"mdd\"]:+.1f}% COMP={best[\"comp\"]:+.1f}%')
print('PASS')
"
```
