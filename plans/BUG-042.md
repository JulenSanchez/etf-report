# BUG-042: 数据管理面板"强制更新"按钮的前后端语义不一致

**严重度**: minor | **状态**: resolved | **发现日期**: 2026-07-06 | **关闭日期**: 2026-07-06

## 问题

用户选中表格中的连续日期区间点"强制更新"，前端承诺"只更新选中范围"，但早期实现调 `update_single(full=False)` 走的是标准增量路径——从 CSV 末笔拉到今天，不限定区间。这导致：
1. 全量拉取整支 ETF 历史（如未先删区间）
2. 前端承诺与后端实现不匹配

## 修复

`data_refetch` 改为先删后拉：
1. 从 CSV 删除选中的 `[start, end]` 区间
2. 调 `update_single(full=False, end_date=end)`，增量模式自动从新末笔拉到 end
3. `fetch_etf_kline` 带 `start_date` 参数，Tencent API 请求限制为 max(20, days_gap*2) 行（通常 ~20 行）

## 实际的 API 调用

- **Tencent API**：`?param={code},day,,,{count},qfq`，单 ETF 单次请求，返回最近 N 行
- **不支持多 ETF 批量**：后端逐个 ETF 循环，每次一发 HTTP 请求，间隔 1s
- **不支持日期区间**：API 只接受 count，不接收 start/end。区间由客户端过滤实现
- **每次"强制更新"最多拉 20 行**，不会触发全量历史拉取
