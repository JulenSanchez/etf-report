## 动机

当前策略研究完全依赖用户主动发起：注意到异常→提出假设→AI 分析→得出结论。F7 因子的发现源于用户观察到养殖重仓异常。用户希望 AI 能自主完成这个循环。

**目标**：建立一个 AI 定期分析策略运行状态、主动发现异常、提出研究假设、执行实验、输出结论和建议的自治系统。

## 功能规格

### 层级 1：日常体检（全自动）

每次 Tuner 启动或每日定时触发，输出一份简洁的状态报告：

- 当前持仓（集中度、行业分布、是否极端）
- 各因子贡献（F1/F3/F7 各自拉动了哪些 ETF）
- 风险指标（滚动波动率、VaR、回撤深度）
- 异常标记（单支权重 > 80%、连续 N 天同仓位、因子极端值）

### 层级 2：异常升级（半自动）

当体检发现异常时，自动提出研究假设：

- "养殖连续 3 天权重 > 80%，F7 贡献 > 90% → 建议研究：F7 在极端超卖情境下的尾部风险"
- "近 1 月 Sharpe 下降 > 0.3 → 建议研究：是否为市场状态切换导致策略退化"
- "某 ETF 近 1 月从未被选中 → 建议研究：是否应从池中移除"

### 层级 3：自主实验（需用户确认后执行）

对提出的假设自动设计实验方案：

- 对比实验（有/无该因子、不同参数）
- 回测窗口选择
- 指标对比模板
- 输出结论报告

## 技术方案

```
scripts/strategy_watchdog.py
  ├─ daily_check()      → 体检报告（markdown）
  ├─ anomaly_detect()   → 异常检测 + 假设生成
  └─ propose_experiment() → 实验方案

触发方式：
  - Tuner 启动时自动运行 daily_check
  - CLI: python scripts/strategy_watchdog.py --deep
  - Cron: 每日 18:00 运行并推送报告
```

## 与 REQ-275 的关系

依赖 REQ-275 的基准快照能力：对比历史基准才能判断"当前是否异常"。
