# REQ-371: 移除 max_gross_exposure 冗余参数

**优先级**: 🔴 P0 | **状态**: ✅ 已交付 | **目标版本**: v3.13.0

## 背景

A 股两融账户中，担保品买入与融资买入分两步走：先现金全仓买 ETF（100% NAV），再用担保品折算价值一次性融资买入。ETF 折算率实测约 90%，因此理论最大杠杆 = 1 + 0.9 = 190%，安全冗余后取 180%。

`max_gross_exposure` 原用于钳制总仓位上限（默认 200%），但在当前系统中是多余的：

- **信号层**：`actual_exposure = min(total_target, max_gross_exposure)` — `total_target` 本身由 `ma_bull_pos`/`ma_bear_pos` 决定，外部再套一层 `min` 纯属重复
- **执行层**：`_execute_rebalance` 用 `remaining = max_gross_exposure * tradable_tv - total_hv` 限制买入，但 `target_positions` 已被分配层精确约束到 `total_target`，执行层只需忠实跟随

详见 [`docs/knowledge/margin-leverage.md`](../docs/knowledge/margin-leverage.md)。

## 改动

### 引擎层

| 文件 | 改动 |
|------|------|
| `quant_backtest.py` | `_execute_rebalance` 移除 `max_gross_exposure` 参数和 `remaining` 计算；信号层 `actual_exposure = total_target`（去 `min`）；买逻辑简化为 `buy_value = max(diff, 0)`（末位）/ `buy_value = diff`（非末位） |
| `quant_contract.py` | `tuner_params_to_config_override` 移除 account 块；`preset_to_tuner_params` 和 `seed_params_from_presets` 移除 `max_gross_exposure` 字段；移除两个死变量 |

### 配置层

| 文件 | 改动 |
|------|------|
| `config/defaults.yaml` | 移除 `account.max_gross_exposure` |
| `config/quant_universe.yaml` | 移除全局 + 3 个 preset（gam-0/1, cst-1）的 `max_gross_exposure` |

### 测试

| 文件 | 改动 |
|------|------|
| `tests/test_quant_contract.py` | 移除 4 个 `max_gross_exposure` 相关测试 |

### 存量数据

pool.json、results.json 等历史 trial 数据中残留的 `max_gross_exposure` 字段不影响回测（`tuner_params_to_config_override` 不再消费该字段），无需清理。

## 控制链

```
ma_bull_pos / ma_bear_pos → total_target → actual_exposure
                                             ↓
                                   target_positions（精确分配）
                                             ↓
                                   _execute_rebalance（忠实执行，不额外钳制）
```

## 验证

- 26 tests 全过
- gam-0 Q1 2026：2026-02-02 leverage=158.0%（= ma_bull_pos），最大 163.5%（执行漂移）
