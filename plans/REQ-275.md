# REQ-275: 基准版本数据快照

**优先级**: 🟡 Normal
**估时**: 4-6h
**状态**: backlog
**目标版本**: v4.1.0
**依赖**: 无
**创建日期**: 2026-06-09

---

## 动机

2026-06-09 的 `rebuild_weekly_from_daily` 过滤逻辑变更导致 CSV 丢失 20 条周线数据、TR 从 741% 掉到 485%。排查时发现：没有旧 CSV 的 bar 数量、EMA 值、因子分数等元信息可供对比，只能靠当场复现旧逻辑。

**目标**：每次大改动前自动存储一份完整的基准快照，改动后自动对比差异。

## 快照内容

| 层级 | 内容 | 空间估算(45 ETFs × 6y) |
|------|------|----------------------|
| 指标摘要 | 各 preset 的 AR/Sharpe/Calmar/MDD/TR | ~1 KB |
| NAV 序列 | 每日 NAV | ~45 KB |
| 因子分数 | 每日每 ETF 的 F1/F3/F7 原始值+映射分+总分 | ~3 MB |
| 数据元信息 | CSV bar 数量、首尾日期、样本 EMA 值 | ~10 KB |
| 代码版本 | 关键函数的 hash 或 git commit | ~1 KB |

总计约 3 MB，可接受。

## 实施要点

- 存储路径：`data/baselines/{version_tag}/`
- 触发时机：CLI 命令 `python scripts/save_baseline.py --tag v4.0.0-before-f1-refactor`
- 对比工具：`python scripts/compare_baselines.py --old <tag> --new <tag>`
- 自动集成到 RELEASE_RUNBOOK.md 的发布前检查
