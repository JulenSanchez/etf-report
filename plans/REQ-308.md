# REQ-308: Tuner 十大成分股展示

**优先级**: 🟡 Normal | **目标版本**: v3.9.0 | **状态**: backlog | **创建**: 2026-06-23

---

## 动机

用户点击 snapshot 表格中的 ETF 行时，希望看到该 ETF 的前十大重仓股，理解策略为什么选了它。

## 设计

### 位置

在 `tuner-snapshot-kline-block` 内，K 线图下方新增一个紧凑的持仓表。

```
┌─ K 线回放 (280px) ──────────────────────┐
├─ 十大重仓 (max 140px, 可滚动) ──────────┤
│ 股票名        代码    权重    行业        │
│ 紫金矿业    601899   9.4%   有色金属      │
│ ...                                      │
└──────────────────────────────────────────┘
```

K 线高度从 380px 缩到 280px，为持仓表腾出 100px。总高度不变。

### 数据源

`data/quant/etf_metadata.json` 的 `top10` 字段。Tuner API 启动时已加载到内存。

### 缺数据处理

部分 ETF（如 QDII、新上市）没有 `top10` 数据 → 显示"暂无成分股数据"。不报错、不隐藏面板。

### 高度约束

nav-block（NAV 图 320px）和 snapshot-section（表格+K线+持仓）通过 CSS flex 保持等高。新增持仓表后在 snapshot-section 内部做高度再分配，不影响外部布局。

## 改动范围

| 文件 | 改动 |
|------|------|
| `scripts/quant_tuner.py` | `/api/run` 或 presets 响应中透传 `top10` 数据 |
| `templates/tuner.html` | K 线区下方加持仓表 + JS 渲染逻辑 |

## 验证

- 点击 snapshot 行 → K 线 + 持仓表同时更新
- 点击无 top10 的 ETF → 显示"暂无成分股数据"
- 左右两栏高度保持一致
