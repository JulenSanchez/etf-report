# REQ-390: GitHub Actions 远端部署 — 收盘前信号兜底

**优先级**: 🟡 P1 | **状态**: in_progress | **目标版本**: v3.15.0 | **依赖**: 无 | **创建日期**: 2026-07-22

## 动机

当前 14:50 收盘前信号完全依赖用户 PC 的计划任务。PC 关机/断网/故障 → 信号不推送 → 当天无法调仓。需要一个远端兜底方案。

## 范围

### Phase 1: GitHub Actions 兜底推送（本 REQ）

- GitHub Actions workflow，cron 14:50 触发
- 全量拉取数据（`quant_data_fetcher.py --full`）→ 跑回测 → Server酱推送
- 与本地计划任务并行运行（先到先推，不冲突）
- 只做信号推送，不做数据持久化（每次从零拉取）

### Phase 2: 数据缓存优化（后续）

- GitHub Actions cache 保留 CSV，避免每次全量拉
- 增量更新替代全量拉取

### 不改的范围

- 不碰 `preclose_push.py` 正式推送流程
- 不改 Tuner（GitHub Actions 直接用 CLI 模式）
- 不替换本地计划任务（远端是兜底，不是替代）

## 技术方案

```yaml
# .github/workflows/signal_push.yml
on:
  schedule:
    - cron: '50 6 * * 1-5'  # 14:50 CST = 06:50 UTC
  workflow_dispatch:          # 手动触发测试

jobs:
  push:
    runs-on: ubuntu-latest
    steps:
      - checkout
      - setup python + install deps
      - inject secrets.yaml from GitHub Secrets
      - python quant_data_fetcher.py --full
      - python backtest_cli.py → Server酱 push
```

### 与 `preclose_push.py` 的差异

| | 本地 `preclose_push.py` | GitHub Actions |
|---|---|---|
| 触发 | Windows 计划任务 | cron |
| 回测方式 | Tuner API | CLI 直接调用 `run_backtest()` |
| 数据 | 本地 CSV（增量） | 每次全量拉取 |
| 推送 | Server酱 | Server酱（同为 sendkey） |
