# REQ-390: GitHub Actions 远端部署 — 收盘前信号兜底

**优先级**: 🟡 P1 | **状态**: in_progress | **目标版本**: v3.15.0 | **依赖**: 无 | **创建日期**: 2026-07-22

## 动机

当前 14:50 收盘前信号完全依赖用户 PC 的计划任务。PC 关机/断网/故障 → 信号不推送 → 当天无法调仓。需要一个远端兜底方案。

## 范围

### Phase 1: GitHub Actions 兜底推送 ✅（已完成，已废弃）

- GitHub Actions workflow，cron 14:50 触发
- `signal_push_cli.py`：全量拉取数据（`quant_data_fetcher.py --full`）→ CLI 直调 `run_backtest()` → Server酱推送
- **问题**：`--full` 模式只拉 CSV 历史数据，`_latest_allowed_date()` 盘中返回昨天，当日实时价格缺失 → 推送信号基于昨天收盘价，无法用于实盘调仓

### Phase 2: Tuner 模式 + 数据缓存 ← 当前

**根因**：远端必须拿到当日实时价格才有实盘意义。盘中数据路径在 Tuner 里（`intraday_cache` + `_get_daily_with_cache`），CLI 直调 `run_backtest()` 绕过了这条路径。

**方案**：远端也走 Tuner API，和本地 `preclose_push.py` 统一：

```
GitHub Actions (14:50 CST)
  ├─ checkout + pip install
  ├─ actions/cache restore data/quant/     ← 持久化 CSV 跨 run
  ├─ python scripts/preclose_push.py       ← 起 Tuner + refresh_data（增量 + 新浪盘中价）+ 回测 + 推送
  └─ actions/cache save data/quant/        ← 为下次 run 缓存
```

**关键改动**：

| 文件 | 改动 |
|------|------|
| `scripts/preclose_push.py` | `_ensure_tuner()` 跨平台（Linux 支持）、新增 `--dry-run` 标志、CI 环境自动标题加 `(远端)` |
| `.github/workflows/signal_push.yml` | 加 `actions/cache@v4`、脚本从 `signal_push_cli.py` 改为 `preclose_push.py` |
| `scripts/signal_push_cli.py` | 废弃，保留文件加注释指向 `preclose_push.py` |

**为什么本地和远端现在可以统一**：
- 本地 `preclose_push.py` 已经封装了 Tuner 生命周期管理（启动/等待/调 API/推送）
- `_ensure_tuner()` 改为跨平台后，GitHub Actions Ubuntu 上同样能起 headless Tuner
- Cache 解决了远端每次全量拉取的问题（首次 ~8min，有 cache 后 ~2min）
- Sina 实时价拉取是 HTTP 请求，GitHub Actions 网络可达

### 不改的范围

- 不替换本地 Windows 计划任务（远端是兜底，不是替代）
- 不改 Tuner 核心逻辑（只改 `_ensure_tuner` 的平台兼容性）

## 技术方案

```yaml
# .github/workflows/signal_push.yml
on:
  schedule:
    - cron: '45 6 * * 1-5'  # 14:45 CST = 06:45 UTC（首次全量拉取 ~6min，推送约在 14:55 前完成）
  workflow_dispatch:
    inputs:
      dry_run:
        type: boolean
        default: true

jobs:
  push:
    runs-on: ubuntu-latest
    timeout-minutes: 15
    steps:
      - checkout
      - setup python + install deps
      - actions/cache restore data/quant/
      - run: python scripts/preclose_push.py [--dry-run]
      - actions/cache save data/quant/
```

### 与 `preclose_push.py` 的差异（Phase 2 统一后）

| | 本地 Windows | GitHub Actions |
|---|---|---|
| 触发 | Windows 计划任务 | cron / workflow_dispatch |
| 回测方式 | Tuner API | **Tuner API（统一）** |
| 数据 | 本地 CSV（增量）+ 新浪盘中 | Cache CSV（增量）+ 新浪盘中 |
| 当日实时价 | ✅ | ✅（统一后） |
| 推送 | Server酱 | Server酱（同为 sendkey） |
| 标题 | `XX/XX 赌徒 调仓信号` | `XX/XX 赌徒 调仓信号 (远端)` |

## 产出

- `.github/workflows/signal_push.yml`
- `scripts/preclose_push.py`（跨平台改造）

## 验证

1. `workflow_dispatch` 手动触发，确认 Tuner 启动 → 数据刷新 → 回测 → 推送成功
2. 推送内容中现价应与当日盘中实时价一致（非昨日收盘价）
3. 手机上收到 Server酱 推送，标题含 `(远端)` 以区分本地推送
4. 第二次手动触发时，cache 命中，数据阶段只做增量（日志可见）
