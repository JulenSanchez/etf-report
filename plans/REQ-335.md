# REQ-335: TPE 收敛突破 — Sobol 注入 + 界松弛 + 多 seed 并行

**优先级**: 🟠 P1 | **目标版本**: v3.10.0 | **依赖**: Pool 引擎就绪 | **状态**: 设计中

## 问题

首轮 Gambler 优化 5 轮 × 30 trial + 50 Sobol，最佳 AR=46.6% 卡在 Round 1 再未突破。三个根因：

1. **边界吸引**：Sobol 撞到极端点 `bull=1.97 bear=0.04` → TPE 的 KDE 被拉偏 → 反复在边界试探
2. **界太窄**：`narrow_bounds_from_trials` margin=0.3，top-15 的好 trial 集中在窄区 → TPE 探索半径小
3. **单线程**：一次只有一个 TPE 进程，一个 random seed → 收敛路径单一

## 方案

### A. Sobol 定期注入（打破 TPE 局部收敛）

每 N 轮迭代后，插入一轮 Sobol：

```
Round 1: narrow_bounds → TPE 30
Round 2: narrow_bounds → TPE 30
Round 3: SOBOL 20 (全界, seed=新)     ← 注入多样性
Round 4: narrow_bounds → TPE 30
Round 5: narrow_bounds → TPE 30
```

Sobol 不依赖历史分布，纯低差异序列覆盖全空间。新点进入池子 → 扩宽 KDE 分布 → TPE 看到新区域。

### B. 界松弛 `--margin`

```python
narrow_bounds_from_trials(trials, top_n=15, margin_pct=0.5)  # 0.3→0.5
```

CLI: `--bounds-margin 0.5`

更大 margin → 搜索界更宽 → TPE 有更多空间跳出局部最优。

### C. 多 seed 并行 + 合并

```
python iter.py --school gambler --seed 100 --output runs/g_seed100.json &
python iter.py --school gambler --seed 200 --output runs/g_seed200.json &
python iter.py --school gambler --seed 300 --output runs/g_seed300.json &

# 完成后合并
cat runs/g_seed*.json → merge → dedupe → prune → pool.json
```

三个独立 TPE 从不同初始路径探索 → 合并池子 → 覆盖更广。

### D. Optuna enqueue_trial（种子注入 TPE）

把已验证的种子 params 用 `study.enqueue_trial()` 注入 TPE 的采样器，让 KDE 从一开始就包含高质量预设的参数分布。

## 改动范围

| 文件 | 改动 |
|------|------|
| `scripts/iterative_optimizer.py` | `--sobol-every` (每 N 轮注入), `--bounds-margin`, `--output` 合并 |
| `src/etf_report/core/quant_contract.py` | `narrow_bounds_from_trials` margin 参数化, `enqueue_trial` 种子注入 |

## 验收

1. 第二轮 Gambler 优化后前沿点 >= 8 个（MDD [-40, -20]）
2. 甜点区 [-35%, -25%] 至少 2 个前沿点
3. 最佳 AR 不卡在同一点超过 2 轮
