## 动机

当前公开导航已经补得较全，但存在公开入口分散的问题：`README.md`、`SKILL.md` 与 `docs/AGENT_USAGE_GUIDE.md` 三者同时承接 Agent 导航，容易让外部与内部认知边界再次混杂。

用户希望进一步收敛成更稳定的形态：
- `SKILL.md` 负责解释“这个技能是什么、从哪里入手”；
- `README.md` 负责解释“怎么配置、怎么运行”；
- 本地额外增加一个不出远端的 `CONTRIBUTING.md`，只服务开发 Agent 的内部认知与治理入口。

## 实施步骤

### Step 1: 收敛公开职责
- 重写 `SKILL.md`，让其成为技能定位 + 导航总入口
- 重写 `README.md`，让其聚焦安装、配置、运行、发布准备
- 清理对外多余导航层，避免 AI 在公开仓里多头读取

### Step 2: 建立本地开发入口
- 新增本地 `CONTRIBUTING.md`
- 明确开发 Agent 的首读顺序、治理事实源、公开/私有边界
- 通过 `.gitignore` 保证该文件不进入远端

### Step 3: 验证一致性
- 检查公开文档是否仍自洽
- 检查内部开发入口是否明确指向 `PLAN.md` / `plans/Board.md`
- 检查 `.gitignore` 与最终工作区状态是否符合预期

## 文件影响分析

| 文件 | 改动类型 | 改动说明 |
|------|---------|---------|
| `README.md` | 修改 | 收敛为安装、配置、运行入口 |
| `SKILL.md` | 修改 | 收敛为技能定位与 Agent 导航入口 |
| `docs/AGENT_USAGE_GUIDE.md` | 删除或退场 | 清理公开多余导航层 |
| `.gitignore` | 修改 | 忽略本地 `CONTRIBUTING.md` |
| `CONTRIBUTING.md` | 新增 | 只服务本地开发 Agent 的私有说明 |
