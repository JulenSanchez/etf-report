# REQ-374: 分组约束优化框架 — 替代无脑网格扫描

**优先级**: 🔴 P0 | **状态**: ✅ 已交付 | **目标版本**: v3.13.0

## 背景

MH/TB/C/CS/F7_window 等参数的优化是逐个孤立的——每轮锁死其他、扫一个。结果是：
- F7_window=17 对 gambler 可能不是最好，但锁死 C/CS 后 TPE 无法找到更好组合
- 赌徒三个预设的 C 取 0.62/0.29/0.40，而 TB=8/12/16，C 的差异部分补偿了 TB 的缺位

问题根源：参数不是独立变量，存在组内代偿/协同，必须联合优化。

## 设计

### 参数树（功能视角）

```
1. 因子评分 ─ "谁是好的"
   ├── F1 子组 ── w1, f1_sensitivity, f1_ema_period
   ├── F3 子组 ── w3, f3_sensitivity, f3_vol_window
   └── F7 子组 ── w7, f7_t, f7_k, F7_window

2. 信心 ─ "下注多少"
   └── bear, MA_period

3. 集中度 ─ "怎么分"
   ├── C, CS, TB     ← 代偿组
   └── band, BS
```

### 优化组（可跨功能域）

一个参数可以出现在多个优化组中：
- 权重比例组：{w1, w3, w7} — 跨 F1/F3/F7，w1+w3+w7=100%
- 灵敏度组：{f1_s, f3_s, f7_t, f7_k} — 跨 F1/F3/F7
- 集中度组：{C, CS, TB} — 同功能域

### 分组约束优化方法

```
Objective:     max AR (或 Sortino / Calmar)
Constraints:   MDD >= -40% (软约束 penalty)
Lock:          组外所有参数锁定当前值
TPE:           optuna TPESampler, 15~30 trials
```

启动前先跑一次当前参数作为基线 (`study.enqueue_trial`)，确认不退化再展开探索。

### 验收标准

1. **不退化**：新参数在目标指标上 >= 当前值（gambler→AR, zen→Sortino, actuary→Calmar），且 MDD 不劣于当前 -2%
2. **产出物**：每轮后更新 YAML 预设 + `preset_metrics.json`，Tuner 前沿图刷新可见
3. **可复用**：优化逻辑封装为一个函数 `optimize_group(presets, vary_keys, metric, mdd_bound)`，换参数名和列表即可复用
4. **结束条件**：连续 2 轮无改善，或全部 7 组跑完

### 权重特殊处理

w1, w3, w7 和为 100%，只有 2 个自由度：

```python
scale = (100 - w7) / (w1_raw + w3_raw)
w1 = round(w1_raw * scale)
w3 = round(w3_raw * scale)
```

### 风控线

每个优化组可设 MDD 软约束。TPE 在约束内自由探索参数间的代偿路径——不需要人预设规则。

## 优化路线图

| 轮次 | 优化组 | 参数 | 锁定 | trials |
|------|--------|------|------|--------|
| 1 | 集中度 | C, CS, TB | 全部 locked | 15×5=75 |
| 2 | F7 | w7, f7_t, f7_k, F7_window | 其余锁定 | 15×5=75 |
| 3 | F1 | w1, f1_s, f1_ema | 其余锁定 | 15×5=75 |
| 4 | F3 | w3, f3_s, f3_vol_window | 其余锁定 | 15×5=75 |
| 5 | 信心 | bear, MA_period | 其余锁定 | 15×5=75 |
| 6 | 权重比例 | w1, w3, w7 | 其余锁定 | 15×5=75 |
| 7 | band | band, BS | 其余锁定 | 15×5=75 |

每轮产出最优值，更新预设和 `preset_metrics.json`。

- `optuna.TPESampler` - 已确认可用
- `config/preset_metrics.json` - 每轮完成后更新
