# REQ-112: Tuner inline onclick → 事件委托

**优先级**: 🟡 P1 | **状态**: done | **目标版本**: v3.13.0

## 背景

原 REQ-112 范围是"index.html HTML/JS/CSS/数据 四层分离"，scope 过大，已废弃。

重新定位：`templates/tuner.html` 当前有 **93 处 inline `onclick`/`onchange`**，违反 `string-escaping.md` 规则：

> 禁止跨上下文 inline 字符串拼接。用 `data-attribute` + 事件委托。

## 范围

仅处理 `templates/tuner.html` 中的 inline event handler，不拆 CSS、不拆 JS 文件、不动 `index.html`。

### 高危（涉及动态值拼接，必须改）

类似 `onclick="dmFoo('${code}')"` 的模式，code 来自 Python 变量注入或 JS 字符串拼接，引号嵌套易出错。

### 低危（纯函数调用，无动态参数）

类似 `onclick="pulseBtn()"` 的静态调用，虽不违反转义但也不一致——统一改为委托更干净。

## 方案

```
1. 给每个可交互元素加 data-action 属性标记行为类型
2. 在父容器上注册一个全局 click 委托
3. 根据 data-action + dataset 分发到对应处理函数
4. 删除所有 inline onclick/onchange
```

## 不改

- CSS 不拆（保持 `<style>` 块）
- JS 不拆（保持 `<script>` 块）
- ECharts 事件（`chart.on('click', ...)`）不受影响——那些是 JS 侧绑定
- `tuner.html` 之外的文件

## 验证

1. 启动 Tuner → 所有按钮/卡片/行点击行为与改动前一致
2. `grep -c 'onclick=' templates/tuner.html` → 归零（或仅保留 ECharts 无关项）
3. 快照表行点击、预设卡片点击、DM 面板操作、分布面板跳转 → 全部正常
