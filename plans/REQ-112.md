## 动机

当前 `index.html` 是一个 ~443KB 的单体文件，包含 HTML 骨架、CSS 样式、JS 逻辑和动态数据。理想状态是三者分离，数据更新只替换 JSON 文件。

**为什么降级为 Low**：

| 论点 | 分析 |
|------|------|
| 性能收益有限 | `update_report.py` 总耗时 ~11s，HTML 注入只占 ~1s，分离省不了多少 |
| 系统运行稳定 | v2.1.0 已稳定运行，没有因为单体 HTML 产生的维护痛点 |
| 改动风险高 | 拆分涉及 JS 变量作用域、事件绑定、懒加载，回归风险大 |
| 收益场景不明 | "AI 消费 data.json" 等场景尚未出现实际需求 |
| 有更优先的事 | REQ-108（备用源）、REQ-113（单测）比这个更紧迫 |

## 实施步骤

> 以下方案保留备查，实际开发前需重新评估。

### Step 1: 提取内联 JS
- 将 `<script>` 中的逻辑拆到 `js/app.js`
- 注意变量作用域、DOM ready 时序

### Step 2: 提取内联 CSS
- 将 `<style>` 拆到 `css/style.css`

### Step 3: 数据加载改造
- 内嵌 `const klineData = {...}` 改为 `fetch('data/report_data.json')`
- 前端加 loading 状态

### Step 4: 保持样式一致
- 逐 ETF 对比分离前后渲染结果

### Step 5: 更新构建流程
- `update_report.py` 改为只生成 `data/report_data.json`

## 风险与缓解

| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|---------|
| JS 作用域问题 | 高 | 变量未定义，功能异常 | 逐函数提取，每步验证 |
| 加载时序变化 | 中 | 数据未加载完就渲染 | 加 loading 状态 + Promise 等待 |
| CSS 优先级变化 | 低 | 样式错乱 | 保持选择器不变，只搬位置 |
| GitHub Pages 路径 | 低 | 静态资源 404 | 用相对路径引用 |
