# REQ-344: 四区并行优化 — 每个 MDD 区间独立 TPE

**优先级**: 🔴 P0 | **目标版本**: v3.11.0 | **状态**: ✅ 完成 (2026-07-02)

## 问题

Gambler 前沿只有 7 pts，[-40,-30) 完全空白。需要四个 5% MDD 区间各自独立 TPE，并行优化，共享池子。

## 方案

4 个 5% MDD 区间各自跑独立 TPE，并行优化，共享池子。

```
区间 A: MDD [-40,-35)  TPE-A  目标: 该区间最高 AR
区间 B: MDD [-35,-30)  TPE-B
区间 C: MDD [-30,-25)  TPE-C
区间 D: MDD [-25,-20)  TPE-D
```

每轮流程：
1. 从共享池子加载所有 trial（snapshot）
2. 各区间筛出自己范围内的 trial，独立缩界
3. 四个 TPE 并行跑（各自 N trial）via ProcessPoolExecutor
4. 每 trial 回测后若 MDD 超出 [-40,-20] 直接丢弃（`_run_trials` 已有）
5. 四区间结果合入池子，去重减负，写回
6. 重复直到各区收敛或总轮次上限

Per-zone convergence: < 2% improvement × 2 consecutive rounds → zone 退出。所有 zone 退出 → 全局结束。

## 实现

| 文件 | 改动 |
|------|------|
| `iterative_optimizer.py` — `_run_zone_round()` | 新增 module-level worker（Windows spawn picklable） |
| `iterative_optimizer.py` — `_print_multi_zone_status()` | 新增 round summary helper |
| `iterative_optimizer.py` — `--multi-zone` / `--max-concurrent` | 新增 CLI flags |
| `iterative_optimizer.py` — Phase 2 dispatch | `if args.multi_zone: ... else: ...` 替代原有 Phase 2 |

### 使用

```bash
# 四区并行（默认 4 并发）
python scripts/iterative_optimizer.py --school gambler --multi-zone --max-rounds 5

# 仅 2 区并行（测试/调试）
python scripts/iterative_optimizer.py --school gambler --multi-zone --max-concurrent 2

# 单区模式不变
python scripts/iterative_optimizer.py --school gambler --zone A
```

## 验收

1. ✅ 烟雾测试：`--multi-zone --max-rounds 2 --trials-per-round 5` 四进程并行无报错
2. ✅ 正式跑测：前沿 4 pts, 激进端 AR=124.6%@-34.7%（融资利率上线后：AR=111.4%@-35.2%，见 `baseline.yaml`）
3. ⚠️ 前沿点数 4（低于目标 8）— 根因是激进端高 AR 支配了中段，非代码缺陷
4. ✅ MDD 覆盖: [-34.7%, -19.7%]，激进端从空白到 -34.7%（当前: [-35.2%, -19.7%]）
5. ✅ 四进程并行无写入冲突
6. ✅ `--zone A` 单区模式回归正常

## 已知局限

- **前沿中段稀疏**：-34.7% 到 -26.3% 之间有 8.4% 空档。需要中段 TPE 补 AR
- **Zone B ([-35,-30)) 始终产出困难**：该区间的参数空间天然难以同时满足高 AR 和约束 MDD
- **Zen/Actuary 未跑**：四区并行目前只在 Gambler 验证，其他两派待后续
