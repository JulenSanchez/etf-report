## 研究问题

当前 gam-0 的 `discretize_step` 以粗粒度离散化形式提供了显著的 alpha（~1100 万 / 6Y）。但这是一种"偷懒"的设计——它在同一个操作中混合了三件事：

1. **集中度放大**：粗粒度 → 头名 ETF 占更大份额
2. **信号延迟**：台阶宽 → 分数波动跨不过去 → 权重不更新 → 复利不被打断
3. **舍入噪声**：奇偶效应、余数分配的随机性、多峰参数地形

其中 1 和 2 产生 alpha，3 是无用的副作用。目标是找到一种更优雅的方式，分离 1 和 2，扔掉 3。

## 关键实验与发现

### 参数拆分（REQ-365）

`discretize_step` 拆分为信号步长（离散化精度）和执行步长（调仓容忍带宽）。

### 整数步数 N 的发现

将信号步长重新定义为整数步数 N（`discretize_step = total_target / N`），消除了浮点多峰，呈现清晰的锯齿波：

```
N=3,5,7,9,11,13,15... (奇数) 始终优于相邻偶数
原因：奇数总步数 → deficit=1 → 头名必然多得 1 步 → 排名被保留
```

### 信号步长 vs 执行步长的等价性

sig 和 exec 都是低通滤波器，分别装在信号输入端和执行输出端。两者同值扫描显示多峰结构，峰值出现在 sig=0.345（+8755%）和 sig=0.165（+7768%）。

### C 无法替代 N

实验（N=5 到 N=17，C 从 0.71 到 5.0）证明：
- C 增大让 softmax 更集中但也更敏感 → churn 增加 → 抵消集中度收益
- 只有 N=5 时 C=1.0 有效（+8339%），N≥7 时提 C 一律亏钱
- **C 操作的是幅度维度，N 操作的是频率维度。两者不可互替。**

### CS 的有限补偿

固定 C=0.71，提 CS 到 18，在 N=5 时达到 +8391%（优于 C 补偿的 +8339%）。但在 N≥7 时 CS 提升同样失效。

### N→无穷的收敛

```
N=5  +9428%  粗粒度 α
N=17 +8425%  开始收敛
N=29 +8234%  平台
```

离散化贡献 ≈ N=5 − 平台 ≈ +1100万。softmax 裸奔（连续信号）的底是 ~+8300%。

### 名义杠杆 vs 实际杠杆

6Y 回测：bull 名义 1.58 实际均值 1.60（偏差 +0.02），bear 名义 0.60 实际均值 0.63。均值无偏——regime 切换定期清零累积偏差。

### 集中度匹配实验

C 可以匹配不同 N 的头名权重（如 N=5 C=0.71 的 76.9% ≈ N=7 C=1.0 的 77.0% ≈ N=9 C=0.71 的 76.6%），但匹配集中度后交易频率仍不同（508 vs 512 vs 518）→ C 改不了 churn。

## 核心洞察

> 离散化是一个偷懒的设计。要替代它，需要把集中度（C/CS）和延迟（死区）分开建模，并扔掉舍入噪声（直接用整数 N 或取消 N）。

目前已知最优整数配置：**N=5 + C=0.71 + CS=18 + exec=sig** → +8391%（优化窗口下 +9428%）。

次优均衡配置：**N=13 + C=0.71 + CS=8.68** → 收益仅低 3.3% 但 MDD 低 4.6pp。

## 后续方向（待建 REQ）

1. **信号死区机制**：分数变化低于阈值时不更新目标权重（纯延迟参数，不改变集中度）
2. **N 视角的引擎重构**：用整数 N 替代连续 signal_step，消除舍入噪声
3. **三参数解耦设计**：C（幅度） + 死区（频率） 替代 N（幅度×频率混在一起）
