# REQ-330: Actuary 派前沿升级 — bear 约束优化 + 前端对齐

**优先级**: 🟡 P1 | **目标版本**: v3.10.0 | **依赖**: pool 引擎就绪 | **状态**: 规划中

## 背景

Actuary 派风险轴 = `ma_bear_pos`（熊市仓位参数，直接可控），目标 = max Sortino。

当前 `iterative_optimizer.py --school actuary` 已原生支持（`target_metric=6y_sortino`, `prune` 按 `ma_bear_pos` 槽位 0.01 步长）。

## 方案

```
优化: python scripts/iterative_optimizer.py --school actuary
      → research/params/actuary/pool.json
      → build_frontier_output(school='actuary')
      → research/params/frontier_actuary.json

风险轴: ma_bear_pos (输入参数, PARAM_BOUNDS step=0.01)
目标: 6y_sortino
prune: 每 0.01 bear 步长保留最优 Sortino, pool 自动维持前沿

前端: type="slider" — X轴=ma_bear_pos, Y轴=Sortino, 散点图点击选点
```

## 关键决策

- **ma_bear_pos 是输入参数而非输出指标**：不依赖 bear_exposure 度量，直接可控
- **prune 槽位 = 参数步长**：band=0.01 对齐 PARAM_BOUNDS，[0.15, 0.30] 区间 ~15 个槽
- 约束 [0.15, 0.30] 待后续在优化器中实现（当前 prune 按全范围槽位）

## 改动范围

| 文件 | 改动 |
|------|------|
| `templates/tuner.html` | Actuary 区域 type="slider" (bear 轴), 散点图 |
| `scripts/quant_tuner.py` | `/api/frontier` actuary 分支 (已就绪) |

## 验收

1. Actuary 优化产出 >= 5 个 bear 前沿点
2. 前端展示 bear 滑动条 + 散点图 (X=ma_bear_pos, Y=Sortino)
3. `/api/frontier` 的 actuary 分支返回有效数据
