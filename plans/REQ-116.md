**创建日期**: -

## 动机

当前报告只有 HTML 格式。有时候需要离线分享（PDF）或程序化消费数据（JSON）。支持多格式导出能扩展报告的使用场景。

**现状问题**：
- 想分享给同事只能发 GitHub Pages 链接，对方需要网络
- 想用报告数据做二次分析，需要手动从 HTML 提取
- 无法归档到本地文档系统

**不做会怎样**：HTML 报告已满足基本需求，多格式是锦上添花。

## 实施步骤

### Step 1: JSON 报告
- 从现有 `etf_full_kline_data.json` + `etf_realtime_data.json` 合并为一份结构化报告
- 输出 `outputs/report_{date}.json`，包含：元数据（日期、版本）、K 线数据、实时数据、健康检查摘要
- **涉及文件**：
  - `scripts/update_report.py` — 修改，生成完 HTML 后额外输出 JSON

### Step 2: PDF 报告
- 选型：`weasyprint`（HTML→PDF，保留样式）或 `playwright`（无头浏览器截图）
- 将 index.html 渲染为 PDF，保留 K 线图和表格样式
- 注意：K 线图是 Canvas/SVG 渲染，需要确保 PDF 能正确包含
- 输出 `outputs/report_{date}.pdf`
- **涉及文件**：
  - `scripts/pdf_generator.py` — **新增**
  - `scripts/update_report.py` — 修改，集成 PDF 生成

### Step 3: CLI 参数
- `update_report.py` 新增 `--format` 参数：
  ```
  python update_report.py                  # 默认 html
  python update_report.py --format json    # 只生成 JSON
  python update_report.py --format pdf     # 只生成 PDF
  python update_report.py --format all     # 全部格式
  ```
- **涉及文件**：
  - `scripts/update_report.py` — 修改 argparse

## 验证方式

| 验证项 | 方法 | 通过标准 |
|--------|------|---------|
| JSON 导出 | `python update_report.py --format json` | 生成结构化 JSON，字段完整 |
| PDF 导出 | `python update_report.py --format pdf` | 生成可打印 PDF，图表和表格正确渲染 |
| 默认行为 | `python update_report.py` | 只生成 HTML（向后兼容） |
| all 格式 | `python update_report.py --format all` | 同时生成 HTML + JSON + PDF |
| PDF 图表 | 打开 PDF 查看 | K 线图、涨跌色、表格均正确 |
| 回归 | 跑一次默认报告 | 报告正常生成，行为不变 |

## 风险与缓解

| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|---------|
| Canvas 图表无法转 PDF | 中 | PDF 中 K 线图空白 | 用 playwright 无头浏览器截图方案，兼容性最好 |
| weasyprint 安装复杂 | 中 | 环境配置困难 | 备选 playwright，或用 Docker 封装 |
| PDF 样式与 HTML 不一致 | 低 | 分享效果差 | 先本地预览 PDF，确认后再发布 |
| JSON 格式不稳定 | 低 | 消费方解析失败 | 定义 JSON Schema，版本化数据格式 |
