# REQ-298: Tuner 合成杠杆参数与风险展示

**创建**: 2026-06-18 | **状态**: backlog | **优先级**: 🟠 High | **目标版本**: v3.9.0

## 背景

Stage A 合成杠杆需要在 Tuner 中可控、可解释、可审查。用户不能只看到收益提升，还必须看到杠杆暴露、尾部风险和融资成本压力估算。

## 目标

Tuner 新增账户区域：

```text
账户模式：cash / synthetic_leverage
最大总暴露：max_gross_exposure
融资成本压力估算利率：financing_rate_annual
```

结果区新增：

```text
avg_exposure
max_exposure
days_above_100
days_above_150
days_above_180
max_daily_loss
worst_month
interest_drag_estimate
```

## 非目标

- 不展示真实追保/强平。
- 不连接券商账户。
- 不把 synthetic_leverage 说成真实两融。

## UI 口径

必须明确标注：

```text
合成杠杆为零融资成本研究模型，不代表真实两融账户收益。
```

## 预计改动区域

- `templates/tuner.html`
- `scripts/quant_tuner.py`
- `src/etf_report/core/quant_contract.py`
- `tests/test_quant_contract.py`

## 验收标准

1. Tuner 可切换 cash / synthetic_leverage。
2. cash 模式下杠杆参数不可误用。
3. synthetic_leverage 模式可设置 `max_gross_exposure <= 2.0`。
4. 风险指标和融资成本压力估算显示在结果区。
5. 参数保存不污染现有生产 preset，除非用户明确保存。
