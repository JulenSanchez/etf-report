## 背景

2026-04-21 BUG-018 修复把 AKShare 的 `fund_etf_hist_sina` 接入到 `fix_ma_and_benchmark.py`，
**只用它补拉 `amount`（成交额）一个字段**，OHLC/volume 仍从新浪原生 `getKLineData` 抓。结果是现在日线链路同时调两个接口：

```
fetch_kline_sina(symbol, scale=240, days=60)
  ├── new Sina getKLineData → {day/open/high/low/close/volume}
  └── fetch_amounts_via_akshare() → ak.fund_etf_hist_sina → {date/...amount}
                                                           ↓
                                          按 dates 对齐后合并 → amounts
```

**这是 BUG 修复路径上的合理折中，但长期不优雅**：

1. 同一只 ETF 同一天被两个接口各拉一次数据，明显冗余
2. 两边的字段一旦口径出现差异（复权规则 / 最新 bar 完整性 / 时区），会成为诡异的隐性 bug 源
3. AKShare 的接口返回 `['date','open','high','low','close','volume','amount']` **完全包含新浪 getKLineData 所有字段**，所以根本没必要留新浪原生接口
4. 统一到 AKShare 还能顺带消化 REQ-108 ("腾讯财经备用源") 的需求——**AKShare 本身就是多源聚合器**，它内部已在做备用源降级

## 目标

把**主行情链**从"新浪 getKLineData + AKShare 打补丁"的双源模式，切换为"AKShare `fund_etf_hist_sina` 单一主源"。

## 技术要点

### AKShare `fund_etf_hist_sina` signature 已实测

```python
def fund_etf_hist_sina(symbol: str = 'sh510050') -> pandas.DataFrame:
    """返回全历史，没有 period/start/end/adjust 参数。"""
```

- 只接 symbol 一个参数
- 返回**全历史**（sh512070 返回 2599 行，sh512400 返回 2092 行）
- 需要应用层自行按 `days` 末尾截取

### 指数用另一个接口

```python
ak.stock_zh_index_daily(symbol='sh000300')
# cols = ['date','open','high','low','close','volume']
# 没有 amount，但指数本身也没成交额语义，用 volume 即可
```

### 周线仍走 `build_weekly_from_daily`

已经是现状，不改。amounts 在 BUG-018 修复后已能正确聚合。

### 符号格式与现状一致

AKShare `fund_etf_hist_sina` 接受 `sh512400` / `sz159566` 格式，与 `config.yaml` 里的 `market + code` 拼接后一致，**零改动**。
