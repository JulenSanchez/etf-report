# REQ-345: Tuner UI 布局设计框架

**优先级**: 🟠 P1 | **目标版本**: v3.11.0 | **状态**: ✅ 完成 | **完成日期**: 2026-07-02

## 背景

Tuner 页面的 UI 改动（增减卡片、调整容器大小）越来越频繁，但缺乏一个统一的布局约束文档。每次改动都靠视觉反馈调整，容易引入容器高度跳动、溢出、留白不均等问题。

## 目标

建立一份布局设计框架文档，作为未来所有 UI 改动的约束。

## 核心内容

### 1. 布局哲学

- **左参数、右回测**：为什么这样分——参数调优是输入，回测结果是输出，左右分栏符合"调参→看结果"的心智模型
- **上汇总、下明细**：策略业绩在上（快速判断），ETF/快照在下（深入分析）

### 2. 容器预算表

基于用户的默认屏幕分辨率（待确认：2560×1440? 1920×1080?）和浏览器缩放（待确认），精确测量每个一级容器的 `width × height`：

| 容器 | 宽度 | 高度 | 备注 |
|------|------|------|------|
| 左侧参数面板 | ? px | ? px | 含 preset 选择器 + 参数滑块 |
| 右侧上：策略业绩 + 指标卡片 | ? px | ? px | 含 nav-chart + dd-chart + metric cards |
| 右侧下左：单个 ETF 快照 | ? px | ? px | |
| 右侧下右：调仓快照 | ? px | ? px | |
| **页面总高** | — | ? px | = 视口高度，无滚动 |

### 3. 预算约束规则

1. **总高宽不变原则**：页面整体 `width × height` 为常量，不出现滚动条
2. **内部补偿规则**：任一元素的高度/宽度变化，必须由同级或父级容器的另一个元素反向补偿
3. **卡片网格规则**：指标卡片区采用固定数量网格，减卡用占位卡填补，加卡替换占位卡
4. **容器间零溢出**：所有直接子元素的高度之和 = 父容器高度

### 4. 测量方法

- 使用浏览器 DevTools 测量实际渲染尺寸
- 记录用户的屏幕分辨率、OS 缩放、浏览器缩放
- 最终数据写入 `docs/design/tuner-layout.md`

## 改动范围

## 验收

1. 文档包含用户默认分辨率下的精确容器尺寸
2. 文档包含预算约束规则，可在未来 REQ 中引用
3. 任意未来 UI 改动 REQ 需先对照框架评估是否触发布局预算变化
