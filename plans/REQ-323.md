**优先级**: 🔴 P0 | **目标版本**: v3.14.0 | **依赖**: 无 | **状态**: 规划中

## 背景

Walk-Forward 讨论全记录见 `research/strategy/walk-forward-discussion.md`。

核心思路：训练窗优化参数（层 1），测试窗样本外验证（层 2），串联多段评估"每年重优化"的真实表现。

Phase 1 的任务：用 gambler × MDD=-40% 铁律，跑通完整两层流程，验证方法可行。

## 前置：参数框架成熟度审查（← REQ-386 并入）

WF 优化开始前，先跑 4 维诊断确认框架在 3Y 窗口上的稳定性：

| # | 诊断 | 方法 |
|---|------|------|
| 1 | 3Y vs 6Y 敏感度一致性 | 10 个 WF 参数 Sobol ~30 trials × 2 窗，看方向性偏好是否一致 |
| 2 | 3Y 窗口 TPE 收敛质量 | 同一窗 2 次独立 TPE（不同 seed），AR 差 < 5pp 为可接受 |
| 3 | 分组假设 3Y 验证 | {bull, MH} 和 {C, N} 联合扰动 3Y vs 6Y，对比响应面崎岖度 |
| 4 | 锁定参数确认 | 验证 f7_lookback/band/band_sensitivity 等在 3Y 上也无需优化 |

产出：锁定参数清单 + 分组方案（每组参数列表 + bounds + trial 数）。

## 两层模型

### 层 1: in-sample 优化

```
每步独立运行:
  训练窗: 3Y (固定长度, 按年滚动)
  目标:   max 3Y_AR
  约束:   MDD ≥ -40%（铁律，不缩放）
  方法:   分组串行 TPE（4 组），每组 30~50 trials
  产出:   该步的最优参数 P
```

3 步时间表：

```
Step  Train           Test
  1   2020-06~2023-06  2023-06~2024-06  → P1
  2   2021-06~2024-06  2024-06~2025-06  → P2
  3   2022-06~2025-06  2025-06~2026-06  → P3
```

start_date 固定 `2020-06-25`。

### 层 2: walk-forward 验证

```
用 P1 跑 2023-06~2024-06 → NAV 序列 1
用 P2 跑 2024-06~2025-06 → NAV 序列 2
用 P3 跑 2025-06~2026-06 → NAV 序列 3
拼接成连续 NAV → 读总 AR / MDD / Sharpe（← REQ-385 决策：方案 C）
```

拼接点天然对齐调仓日，直接换仓。

### 通过标准

1. ✅ 前置审查报告完成，产出锁定清单+分组方案
2. ✅ 3 步层 1 各自完成，AR/MDD 有效（无 None/NaN）
3. ✅ 拼接 NAV 产出有效 OOS 指标
4. ✅ 参数在步间确实变化（≥ 2 组不同值，证明不是原地踏步）
5. ✅ OOS AR 不低于 IS AR 的 50%（否则框架不稳定，需回退诊断）

### 失败处理

如果验收 5 失败（OOS AR < IS AR 的 50%），说明框架在 3Y 上过拟合严重。此时：

1. 回看审查报告：哪个诊断维度最差？
2. 进一步锁定跳变最剧烈的参数组
3. 缩小不稳定参数的 bounds
4. 重跑 Phase 1

如果 3 轮调整后仍未通过 → 参数框架需要结构性简化（降维、合并参数），再启动 WF。
