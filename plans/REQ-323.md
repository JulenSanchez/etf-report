# REQ-323: Walk-Forward Phase 1 — 单轨迹方法验证

**优先级**: 🟠 P1 | **目标版本**: v3.10.0 | **依赖**: REQ-301 (real_margin) | **状态**: 规划中

## 背景

完整讨论记录见 `research/strategy/walk-forward-discussion.md`。

当前所有策略评估基于 in-sample 回测（优化和测试用同一段数据），偏乐观。Walk-forward 是量化圈验证策略稳定性的标准方法：训练窗优化参数，测试窗样本外验证。串联多段样本外结果评估"每年重优化"的真实表现。

在投入全量工程之前，先用一条轨迹验证方法论正确。

## 目标

对 gambler 的 **一个 MDD 约束值**（-25%），跑通完整的层 1 + 层 2 流程，产出第一条 walk-forward 轨迹：

```
Step  Train (3Y)         Test (1Y)        参数
──────────────────────────────────────────────
  1   2020-06 ~ 2023-06   2023-06 ~ 24      P1
  2   2021-06 ~ 2024-06   2024-06 ~ 25      P2
  3   2022-06 ~ 2025-06   2025-06 ~ 26      P3
```

## 两层模型

### 层 1: in-sample 优化

```
每步独立运行:
  训练窗: 3Y (固定长度, 滚动)
  目标:   max 3Y_AR
  约束:   MDD ≥ -25%
  算法:   Sobol 50 + TPE 50 (标准优化, 不 prune)
  产出:   该步的最优参数 P
```

P1/P2/P3 可能不同——因为训练窗数据滑动了一年，最优参数会变。

### 层 2: walk-forward 验证

```
每步用层 1 产出的 P 在测试窗上跑一次回测:
  测试窗: 1Y (参数 P 从未见过这段数据)
  产出:   实际 MDD 和 AR（样本外）
  
  3 个样本外点串联 → 一条 walk-forward 轨迹
```

### 关键设计决策

1. **训练窗 3Y**：足够长以稳定优化，足够短以反映近期市场
2. **测试窗 1Y**：标准样本外验证周期
3. **不 prune**：所有 trial 保留，作为前沿样本
4. **start_date 固定 2020-06-25**：过渡期不滚动
5. **MDD 轴用实际值**：层 2 的 MDD 是样本外真实值，不是约束值

## 验收

```bash
# 1. 层 1: 3 步优化各自完成
ls research/params/wf_gam_m25/step1/signal/.optimizer_done
ls research/params/wf_gam_m25/step2/signal/.optimizer_done
ls research/params/wf_gam_m25/step3/signal/.optimizer_done
# → 三个文件都存在

# 2. 层 2: 3 步样本外测试有结果
python -c "
import json, pathlib
for step in ['step1','step2','step3']:
    r = json.loads(pathlib.Path(f'research/params/wf_gam_m25/{step}/test_result.json').read_text('utf-8'))
    print(f'{step}: AR={r[\"ar\"]:+.1f}% MDD={r[\"mdd\"]:+.1f}%')
    assert r['ar'] is not None and r['mdd'] is not None
    assert r['mdd'] < 0, 'MDD should be negative'
print('PASS')
"

# 3. 参数确实在变化（证明每步独立优化）
python -c "
import json, pathlib
params = []
for step in ['step1','step2','step3']:
    r = json.loads(pathlib.Path(f'research/params/wf_gam_m25/{step}/test_result.json').read_text('utf-8'))
    p = r['params']
    params.append((p['ma_bull_pos'], p['max_holdings']))
# 至少有两步的参数不同（证明 walk-forward 不是原地踏步）
assert len(set(params)) >= 2, f'All 3 steps have identical params: {params}'
print('Params vary across steps:', params)
print('PASS')
"

# 4. 至少一个测试窗 MDD 比约束值差
python -c "
import json, pathlib
constraint = -25
violations = 0
for step in ['step1','step2','step3']:
    r = json.loads(pathlib.Path(f'research/params/wf_gam_m25/{step}/test_result.json').read_text('utf-8'))
    if r['mdd'] < constraint:
        violations += 1
        print(f'{step}: MDD={r[\"mdd\"]:.1f}% < constraint {constraint}%')
assert violations >= 1, 'No step violated the constraint (unrealistic)'
print(f'{violations}/3 steps violated constraint')
print('PASS')
"
```

### 通过标准

1. ✅ 3 步层 1 各自完成，产出 `.optimizer_done`
2. ✅ 3 步层 2 各自产出 (MDD, AR)，无 None/NaN
3. ✅ 参数在步间确实变化（≥ 2 组不同值）
4. ✅ 至少一步样本外 MDD 比约束值更差（证明样本外退化存在）
