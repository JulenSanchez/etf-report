# REQ-353: 实盘调仓执行表 — 推送内容优化

**优先级**: 🔴 P0 | **目标版本**: v3.11.0 | **状态**: 设计中

## 当前痛点

Server酱推送的 Top-12 表格是"信号摘要"——展示得分、仓位、动作，但和实盘执行脱节：

- 用户需要自己算：总资金 × 目标仓位% = 分配金额 → 金额 ÷ 价格 = 股数
- Top-12 包含零仓位 ETF（无操作），信息冗余
- 没有价格信息，不知道一张 ETF 多少钱

## 目标

推送内容从"信号摘要"改为"执行参照表"——用户对照自己的大致总金额，直接看到每支 ETF 该买多少股。

## 推送内容设计

### 简化信号区（仅需买入的 ETF）

```
策略: 赌徒 | 07-03 15:05 收盘确认

需买入:
  恒生医药 (513120)  目标 58%  现价 1.234
  有色金属 (512400)  目标 42%  现价 2.156
```

不展示零仓位 ETF，只展示需要买入的。

### 执行参照表（核心）

```
┌────────┬─────────────────────┬─────────────────────┐
│ 总金额  │ 恒生医药 58%         │ 有色金属 42%         │
│        │ 金额      股数       │ 金额      股数       │
├────────┼─────────────────────┼─────────────────────┤
│  50w   │ 29.0w   235,000    │ 21.0w    97,400    │
│  51w   │ 29.6w   240,000    │ 21.4w    99,400    │
│  52w   │ 30.2w   244,000    │ 21.8w   101,400    │
│  ...   │ ...                │ ...                │
│  60w   │ 34.8w   282,000    │ 25.2w   116,900    │
└────────┴─────────────────────┴─────────────────────┘
```

- 每行 = 一个总金额档位（50w~60w，步长 1w）
- 每列 = 一支需买入 ETF
- 每格 = 金额(万元) + 股数(取整到 100 股)
- 用户找到接近自己实际金额的行，直接按股数下单

### 计算规则

- 金额 = 总金额 × 目标仓位%
- 股数 = floor(金额 / 现价 / 100) × 100（A 股 ETF 最小交易单位 100 股）
- 现价来自回测信号执行日的收盘价

| 文件 | 改动 |

## 验收

1. 推送消息只包含需要买入的 ETF（仓位 > 0）
2. 执行参照表覆盖 50w-60w 共 11 档
3. 股数取整到 100 股的整数倍
4. 与 REQ-348（盘后定价）协同——推送时间改为 15:05，现价为收盘确认价
