# REQ-338: 删除 account_mode — bull 隐式表达账户行为

**优先级**: 🔴 P0 | **目标版本**: v3.10.0 | **依赖**: REQ-337 (三派统一) | **状态**: 设计中

## 背景

`account_mode` (cash / synthetic_leverage) 引入了 4 个 bug，且逻辑上冗余：
- `bull ≤ 1.0` = 不需要借钱 = 天然 cash
- `bull > 1.0` = 需要借钱 = 天然 synthetic_leverage

引擎统一走融资路径，`max_gross_exposure` 固定 2.0，`buy_value` 永远用 `min(diff, remaining)`。

## 改动范围

| 文件 | 改动 |
|------|------|
| `config/quant_universe.yaml` | 所有 preset 的 `account` 块去掉 `mode`，保留 `max_gross_exposure: 2.0`；顶层 account 块同样 |
| `quant_contract.py` | `PARAM_BOUNDS` 删 `account_mode`；`tuner_params_to_config_override` 删 account_mode 分支；`preset_to_tuner_params` 删 `account_mode` 字段；`PARAM_SCHEMA` 删账户模式组 |
| `quant_backtest.py` | `load_config` account 块只读 `max_gross_exposure`；主循环去掉 `if account_mode == "synthetic_leverage"` 分支；`_execute_rebalance` 统一走融资路径 |
| `iterative_optimizer.py` | 去掉硬编码 `account_mode` |
| `templates/tuner.html` | 前端删账户模式选择器 |

## 验证

1. **语法**：所有 .py 编译通过
2. **gam-0 6Y**：AR=110.0% MDD=-30.4%（当时验证正确；REQ-339 融资利率上线后变为 AR=102.6% MDD=-30.9%，见 `baseline.yaml`）
3. **gam-0 2M**：total_target=1.58，持仓总和=158%（不变）
4. **三派 optimize 干跑**：`--school gambler --trials-per-round 3 --max-rounds 1` 不报错
5. **Tuner 启动**：`/api/presets` 不报错，账户模式区消失
6. **前端回测**：gam-0 近两月 totalPosition=158%
