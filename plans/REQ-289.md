# REQ-289: ETF 默认勾选机制（active 标记 + 5% 阈值）

**创建**: 2026-06-16 | **状态**: done | **关联**: REQ-274（R15 换池）

## 背景

R15 换池新增 9 支 ETF，仅 1 支正向贡献，累计拖累 TR -195.75pp。筛选流与回测流应功能分离，但换池时须有异常监测。引入 5% TR 降幅阈值 + `active` 标记机制，让负面 ETF 默认休眠而非彻底拒绝。

## 目标

1. Config 增加 `active: false` 可选标记，控制 ETF 的默认勾选状态
2. Tuner 前端初始只勾选 `active !== false` 的 ETF，休眠 ETF 不参与默认回测
3. Tuner 增加"保存标的池"按钮，勾选变动手动持久化到 config
4. 所有回测入口统一参数语义：不传 universe → 默认仅 active ETF；显式传参 → 以参数为准
5. `pool_change.py` 新增/替换触发 5% TR 降幅 → `active: false`

## 改动点

### Config 层
- `config/quant_universe.yaml`: ETF 条目增加可选字段 `active: false`（缺省=true）

### 传输层
- `quant_contract.py:398`: `_universe_options` 每条增加 `"active": true/false`
- `quant_contract.py:215`: `parse_universe_filter` 语义不变（空串→None）

### 回测层
- `quant_backtest.py:427`: `run_backtest` — `universe_filter=None` → 仅取 `active:true` ETF
- `quant_backtest.py:1204`: CLI 增加 `--universe` 参数（逗号分隔 code 列表，不传=默认）

### Tuner 后端
- `quant_tuner.py`: 新增 `POST /api/universe/save`，接收 `{active_codes: [...]}`，回写 config
- `quant_tuner.py:1111`: `run_tuner_backtest` — `universe_filter=None` → 仅 active ETF

### Tuner 前端
- `tuner.html:1935`: `UNIVERSE_SELECTED` 初始化改为仅收集 `active !== false` 的 code
- `tuner.html:2068`: `getUniverseParam()` 始终发显式 code 列表（移除 `''` 全选快捷）
- `tuner.html`: 标的池区域新增"保存标的池"按钮 + 保存结果文字提示

### 调度任务
- `preclose_push.py:161`: 删除 `"universe": ""` 硬编码
- `pool_change.py`: 不改——不传 universe，自动走默认池做基线

## 行为规范

| 场景 | universe 参数 | 行为 |
|------|-------------|------|
| Tuner 打开 | — | 仅勾选 active ETF |
| Tuner 勾选/取消 | — | 仅 UI 变化，不持久化 |
| Tuner 点"保存" | POST active_codes | 写入 config，显示保存结果 |
| Tuner 跑回测 | 显式列表（当前勾选） | 以勾选为准 |
| CLI `--universe a,b,c` | 显式列表 | 以参数为准 |
| CLI 不传 `--universe` | None | 默认池（active only） |
| pool_change subprocess | None | 默认池做基线 |
| preclose_push | None（删 `""` 后） | 默认池 |
| update_report | None（不传） | 默认池 |

## 5% 阈值规则（pool_change.py 内）

- 新增/替换后 `|Δ TR / 基线 TR| > 5%` → 写入 `active: false`
- 移除触发阈值 → 取消移除（回退 config），登记原因

## 验证

1. Config 加一支 `active: false` ETF，启动 Tuner → 确认默认不勾选
2. 手动勾选后跑回测 → 确认该 ETF 参与回测
3. 点"保存" → 确认 config 已更新 → 刷新 Tuner → 确认状态保持
4. CLI 不传 `--universe` 跑回测 → 确认仅 active ETF 参与
5. CLI 传 `--universe "code1,code2"` → 确认仅指定 ETF 参与
6. `python scripts/pool_change.py --add ...` → 确认默认池做基线
