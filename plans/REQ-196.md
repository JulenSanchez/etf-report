# REQ-196: 实盘调仓信号（内置到 Tuner）

## 背景

回测系统偏向研究（输出3年NAV曲线），而实盘需要的是"今天买/卖/持仓什么"的即时指令。
7月6日后盘后固定价格交易扩展到全部A股+ETF，可以在15:05-15:30以收盘价成交。

## 实现方式：Tuner 内置刷新

不单独建脚本，直接在 Tuner 调试页内置"刷新数据"功能：

1. 点击"刷新数据"按钮
2. 盘中（<15:10）：Sina API 取实时价 → 写入 intraday_cache → 估算全天成交量
3. 盘后（≥15:10）：quant_data_fetcher 增量拉取确认收盘价 → 更新 CSV → 清 cache
4. 点击"运行回测" → 回测区间右端始终到今天 → cache 数据参与计算但不写 CSV

### 数据分离原则

| | CSV（持久化） | intraday_cache（内存） |
|---|---|---|
| 来源 | 腾讯K线API（收盘后确认） | 新浪实时行情API |
| 写入 | 仅盘后 fetcher 写入 | 盘中每次刷新覆盖 |
| 生命周期 | 永久 | 会话级，Tuner关了就没了 |
| 计算 | 回测直接用 | 回测时动态合并到 daily_df |

### 时序

**盘中（14:45点击刷新 → 运行回测）：**
```
14:45  刷新数据 → Sina API 取实时价 → cache
14:46  运行回测 → cache数据参与计算 → 看到盘中信号
14:55  再次刷新 → cache更新 → 重新回测 → 信号更准
```

**盘后（15:10后点击刷新）：**
```
15:10  刷新数据 → fetcher增量拉收盘价 → CSV更新 → cache清空
15:12  运行回测 → CSV里有确认收盘数据 → 信号=最终版
```

## 关键设计决策

1. **盘中数据不进CSV**：避免混淆盘中价和收盘价，CSV永远是纯收盘数据
2. **单份cache，每次覆盖**：时间不会倒流，只保留最新的盘中快照
3. **成交量估算**：盘中volume按交易时间比例推算全天量，让F3量比更准确
4. **回测右端始终是今天**：不受CSV最新日期限制
5. **与回测引擎逻辑完全一致**：同一套因子计算代码，cache只是"补齐今日bar"
