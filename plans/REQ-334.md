# REQ-334: 架构包化 — scripts→src 迁移，正式页与量化调试器拆分

**优先级**: 🔴 P0 | **目标版本**: v4.0.0 | **依赖**: 无 | **状态**: 规划中

## 背景

当前 `scripts/` 同时承担 CLI、库逻辑、兼容入口三种角色，导致：
- 测试直接 import 脚本模块（依赖 `sys.path.insert`）
- CLI 行为和库行为混杂
- wrapper 与主入口行为可能不一致
- 包化边界不清，无法拆分为独立项目

评审报告 §6.2 将其列为"主要架构风险"。

## 目标

将核心逻辑从 `scripts/` 迁入 `src/etf_report/`，使 `scripts/` 只保留薄 CLI/兼容入口。最终支持正式页（index.html + report）和量化调试器（Tuner + backtest）拆分为两个独立项目。

## 目标目录结构

```text
scripts/                          # 只保留薄 CLI / 兼容入口
  update_report.py                # → from etf_report.report import build
  quant_tuner.py                  # → from etf_report.quant import serve
  ...

src/etf_report/
  core/                           # 共享模块（已完成）
    paths.py                      # ✅
    quant_contract.py             # ✅
    quant_data_utils.py           # ✅
    calendar.py                   # 待迁
  report/                         # 正式页生成逻辑（新增）
    update_report.py              # 从 scripts/ 迁入
    deployer.py                   # 从 scripts/ 迁入
    editorial.py                  # 待评估
  quant/                          # 量化核心逻辑（新增）
    backtest.py                   # 从 scripts/quant_backtest.py 迁入
    tuner.py                      # 从 scripts/quant_tuner.py 迁入（业务逻辑）
    optimizer.py                  # 从 scripts/ 迁入（三优化器合并后）
    factors.py                    # 从 scripts/quant_factors.py 迁入
    data_fetcher.py               # 从 scripts/quant_data_fetcher.py 迁入
```

## 原则

- **不一次性大重构**——每次迁移保留旧入口，旧入口 `runpy` 转发到新位置
- **每次迁移都有测试兜底**——迁移前后 `pytest tests -q` 必须全绿
- **每步独立可交付**——不要求整个 v4.0.0 串行完成
- **旧 wrapper 不改行为**——`scripts/report_site/` 和 `scripts/quant_lab/` 的 wrapper 在迁移期间继续可用，逐步退役

## 迁移步骤

### Step 1: 量化核心抽取
- `quant_backtest.py` 核心逻辑 → `src/etf_report/quant/backtest.py`
- `scripts/quant_backtest.py` 变为薄 wrapper：`from etf_report.quant.backtest import run_backtest`
- `scripts/quant_lab/quant_backtest.py` wrapper 不变

### Step 2: 正式页核心抽取
- `update_report.py` 核心逻辑 → `src/etf_report/report/`
- `scripts/update_report.py` 变为薄 wrapper

### Step 3: Tuner 抽取
- `quant_tuner.py` 业务逻辑 → `src/etf_report/quant/tuner.py`
- `scripts/quant_tuner.py` 保留 Flask app + CLI

### Step 4: 优化器合并后迁入
- 三个优化器统一后 → `src/etf_report/quant/optimizer.py`

### Step 5: 独立项目拆分
- `etf-report` → 正式页发布项目（index.html + report）
- `etf-tuner` → 量化调试器项目（Tuner + backtest + optimizer）
- 共享 `src/etf_report/core/` 作为公共依赖包

## 验收

```bash
# 每步迁移后
pytest tests -q                    # 全量测试通过
python scripts/update_report.py    # 正式页生成正常
python scripts/quant_tuner.py      # Tuner 启动正常
```

## 阻塞

无。可与 v3.10.0 / v3.11.0 并行推进。
