# REQ-139: 按 ID 聚焦页面元素的调试工具

**估时**: 3-5h
**状态**: done
**目标版本**: -

**创建日期**: 2026-04-18

## 动机

当前页面已经有“调试功能”和“热区模式”两个调试入口，但它们更偏向整体浏览和温度审视，缺少一个**针对单个 `id` 的精确定位工具**。

当用户想验证 `strong-buy-card-overview-item-*`、`report-card-text-*` 之类节点时，往往已经知道目标 `id`，却仍然需要手动翻子页面、肉眼查找和滚动定位，排查成本偏高。

## 目标

在现有调试工具栏中新增第三个工具，支持：
- 输入任意页面元素 `id`
- 自动切换到目标元素所在的子页面 / 面板
- 自动滚动页面，让目标元素尽可能位于视口中部
- 通过高亮、涂色或框选方式显式提示目标位置
- 对不存在的 `id` 提供清晰反馈，并尽量不打扰正常浏览

## 设计原则

- **沿用既有调试工具栏**：不再发明单独入口，作为第三个工具接入
- **输入驱动**：提供轻量输入框/确认动作，直接围绕 `id` 查询
- **先切页再定位**：若目标在非当前面板，先自动切换，再做聚焦
- **滚动居中**：定位成功后尽量把目标滚到视口中部，提升识别效率
- **视觉提示清晰**：复用现有高亮层能力，但要明显区别于普通 hover/热区态

## 实施步骤

### Step 1: 梳理目标元素与面板映射
- 识别各 ETF 详情面板、概览区、宏观区等容器边界
- 设计“通过元素反推应切到哪个子页面”的规则

### Step 2: 扩展调试工具栏交互
- 新增第三个工具按钮
- 设计输入框、提交、取消与键盘交互
- 保持与前两个工具的视觉节奏一致

### Step 3: 实现按 ID 聚焦流程
- 查找目标元素
- 自动切页 / 展开对应面板
- 居中滚动并加高亮/框选
- 处理未命中、隐藏节点、重复触发等边界

### Step 4: 验证体验与回归
- 验证概览项、详情项、文本项都能稳定定位
- 验证与现有调试开关、热区模式不会互相打架
- 跑一次完整报告，确保生成产物正常

## 验证方式

| 验证项 | 方法 | 通过标准 |
|--------|------|---------|
| 输入有效 id | 手动输入 `strong-buy-card-overview-item-1` 等样本 | 自动切页、居中并高亮目标 |
| 输入无效 id | 手动输入不存在的 id | 有 toast 反馈，页面不进入异常状态 |
| 调试工具共存 | 与调试模式/热区模式一起切换 | 三个工具互不打架，视觉反馈清晰 |
| 回归生成 | 跑一次完整报告 | 报告正常生成 |
