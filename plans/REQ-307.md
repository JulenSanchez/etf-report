## 动机

当前 `ma_trend_confidence()` 只用单一指数（沪深 300）做牛熊判断。单指数在某些市场风格下给出假信号：
- 2016-2017 蓝筹牛：HS300 走强但创业板还在熊 → 单看 HS300 会过早切 bull
- 2024 年初小盘崩：HS300 平稳但 CSI1000 暴跌 → 单看 HS300 察觉不到风险

多宽基投票：N 个指数各自独立做 MA 判断，投票过半才切换牛熊状态。任何单一指数的风格偏差被其他指数对冲。

**与 REQ-306 的关系**：完全独立。投票指数是信号源，不进选股池。不需要等宽基 ETF 入库。

## 投票池设计

### 方案选择

当前 plan 原方案（HS300 + CSI500 + ChiNext + STAR50）有问题：ChiNext 和 STAR50 都是科技/成长导向，产生科技权重偏见。

| 方案 | 指数 | 覆盖 | 评价 |
|------|------|------|------|
| **A（推荐）** | SSE50 + HS300 + CSI500 + ChiNext | 超大盘价值 → 大盘 → 中盘 → 创业板 | 风格跨度最宽，无冗余 |
| B（最小） | HS300 + CSI500 + ChiNext | 大盘 → 中盘 → 成长 | 3 票奇数天然可决，最简单 |
| C（含小盘） | HS300 + CSI500 + CSI1000 + ChiNext | 大盘 → 中盘 → 小盘 → 成长 | CSI1000 对小盘流动性敏感 |

**推荐方案 A**。理由：
- SSE50 是"价值锚"——银行/保险/红利在极端行情中往往与科技反向，提供真正的多样化
- HS300 是机构基准——最宽的大盘信号
- CSI500 是中盘——风格轮动的中间地带
- ChiNext 是成长——Beta 最高的信号源
- 四者相关性足够低，不会被单一风格绑架

## 投票机制

```
单指数层（每票规则）:
  该指数 close > MA(close, N周) 且 MA斜率 > 0 → 投 bull 票
  任一条件不满足 → 投 bear 票

投票层（汇总）:
  bull票 / 总票数 > 0.5  → 切换/维持 bull
  bull票 / 总票数 < 0.5  → 切换/维持 bear
  bull票 / 总票数 = 0.5  → 维持当前状态（平票不切换）
```

## 统一 MA 周期

**所有指数共用一个 MA 周期参数**。不允许分别设置——否则可以通过差异化窗口制造想要的投票结果。

```yaml
confidence:
  type: ma_trend
  ma_trend_period: 19        # 周，所有投票指数统一
  benchmarks:                # 投票委员会
    - "000300"               # 默认只勾 HS300
  direction_confirm: true    # 价格+斜率都同意才投票
```
