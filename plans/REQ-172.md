## 动机

> REQ-170 v2.6.0 上线后，6 支 ETF 的估值水位 section 只有 **515880 通信**（PE 主指标）有真实数据（77.9 偏高），**5 支显示 no-data**。
> 用户反馈："只有通信有数据，感觉很奇怪"，希望提高覆盖率到"全部/大多数 ETF 有数据"。

**受限支现状**：
| ETF | 主指标 | 受限原因 |
|-----|--------|----------|
| 512400 有色 | PB | csindex API 只给 PE 不给 PB |
| 159865 养殖 | PB | 同上 |
| 512070 证保 | PB | 真实指数无 csindex 直连 + PB 拿不到 |
| 513120 港创药 | PB | csindex 不收港股系 |
| 159566 储能 | PE | csindex 不收国证系 |

**4 支需要 PB**，**2 支需要指数体系外覆盖**。

**不做会怎样**：
- 估值水位模块持续"演示价值 > 实用价值"
- 与 ashare-trading-system 技能的估值联动无法真正启动
- 用户对"为什么不展示数据"产生持续疑问

## 关键洞察（2026-04-23 四轮 PoC 结论）

### 洞察 1：中证指数 API 天花板
- `stock_zh_index_value_csindex`：只给 PE、只返回 20 天
- `stock_zh_index_hist_csindex`：**返回 1464 行日级历史（PE only）！数据停在 2024-06-04**
- csindex JSON API `/csindex-home/indexInfo/index-basic-info`：只有元数据
- csindex 其他估值端点（index-valuation 等）：**全部 404**
- **结论**：csindex 系完全无 PB 接口，PB 只存在于人工下载 Excel 文件中

### 洞察 2：PB 数据源调研（四条路全部受阻）
- oss 文件（`csi-web-dev.oss-...indicator.xls`）：**403** 需签名认证
- 乐咕 `stock_index_pb_lg` / `stock_market_pb_lg`：AkShare 1.18.40 时间戳解析 bug
- 东财 push2 API（`f167` 市净率字段）：**连接断开**，非本地网络问题就是东财反爬
- 百度 `stock_zh_valuation_baidu`：**KeyError: 'chartInfo'**，AkShare 接口实现变更

### 洞察 3：用户"统一公开数据源"诉求指向万得/申万一级
- 万得（Wind）行业分类 = GICS 翻版，需 Wind 终端（¥30k+/年）
- 申万一级 = A 股散户标准，**中证指数有对应指数**（000819 申万有色 / 000928 申万金融 ...），但 PB 仍需 Excel
- **结论**：万得本身不是数据源，是"分类 + 汇总口径"；申万一级通过中证指数拉 PE 可以，PB 同样卡 Excel

### 洞察 4：历史 PE 可以**先跑起来**
- `stock_zh_index_hist_csindex` 拉到 1464 行日级 PE 历史
- **方法 B' 可立即切历史百分位模式**，不必等 1 年累积
- 这是比 PB 攻关更快见效的路径（但不能解决周期股反向问题）

## 实施路线选择（分阶段）

### Phase A：历史 PE 百分位落地（低风险高收益，推荐先做）

让 4 支 csindex 覆盖的 ETF（512400/512070 若有/515880/159865）**立即用历史 PE 百分位**，无论主指标是 PE 还是 PB。诚实标注"PE 代理" + `window_note` 警示周期股 PE 反向问题。

- 估时 2h
- 覆盖 no-data 数量：从 5 减到 3（仅剩 513120/159566/512070）
- 风险：周期股（有色/养殖）PE 反向可能给出"底部显示低估"的误导，需 UI 重点说明

### Phase B：港股/国证系同行业代理（用户批准方向）

- 513120 港创药 → 用 A 股中证创新药 **931152** 或中证生物医药 **930726** 做代理
- 159566 储能 → 用中证新能源 **931153** 或中证新能源车 **931463** 做代理
- 估时 1.5h
- 覆盖 no-data 数量：3 → 1（仅剩 512070 代理问题）
- 风险：代理指数的估值不等于原 ETF 成分真实估值，偏差可能 10-20%

### Phase C：PB 数据源深挖（高风险，成败难料）

三条候选路径，按工作量 + 成功概率排序：

1. **模拟浏览器下载 csindex Excel**（估时 3-5h，成功概率 60%）
   - 用 `playwright` / `selenium` + 渲染 SPA + 抓下载 URL
   - 月度维度估值（月更即可，PB 波动小）
   - 维护成本高（selenium 版本漂移）

2. **Tushare Pro 积分制 API**（估时 1-2h，成功概率 90%）
   - 官方 `index_dailybasic` 接口含 PB
   - 需注册 + 积分（免费账号部分接口需 2000 积分）
   - 稳定性最高

3. **爬东财 AJAX + Playwright**（估时 4-6h，成功概率 70%）
   - 绕过 push2 反爬，走 SPA 内部 API
   - 维护成本中

### Phase D：512070 真实指数直连（深度优化）

- 512070 跟踪"沪深300非银行金融指数"，真实代码可能是 `H30093` 或类似
- AkShare 可能需要走 `index_stock_info` + 新 endpoint 探测
- 当前用"沪深300金融地产 000914"代理已 10% 偏差
- 估时 1h（简单核查）

## 推荐实施顺序

**1→2→3（分批验证）**：
1. **Phase A**（历史 PE）：最安全，2h 落地，覆盖 4 支
2. **Phase B**（代理指数）：1.5h 追加 2 支
3. **Phase C 路径 2**（Tushare）：若用户同意注册，1-2h 拿 PB 精准值

**Phase C.1 和 C.3（Playwright）**：当 Phase C.2 无法或用户不想走 Tushare 时启用，作为兜底

**Phase D**：独立小任务，任何时候可插入
