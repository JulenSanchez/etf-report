🛠 全流程总览 end-to-end ▶

每个调仓日（每周五），对 25 支 ETF 执行以下流水线：

Phase 1: 因子计算（每支 ETF 独立）

1. F1 = 周线EMA偏离度（原始值，如 +5.2%）

2. F2 = RSI 偏离 z-score（最近 20 日 RSI 自身均值的标准化偏离）

3. F3 = 方向性量比（原始值，如 1.8）

4. F4 = 估值因子（已映射为 0-100 分）

Phase 2: 连续映射（每支 ETF 独立，不做截面排名）

5. F1 → sigmoid(ema_dev / f1_sensitivity) → [0, 1]

6. F2 → 死区 |z|<1.0 → 0.5；外 → 0.5 ± 0.5(1−exp(−|z|−1)) → [0, 1]

7. F3 → sigmoid(log(ratio) / f3_sensitivity) → [0, 1]

8. F4 → val_score / 100 → [0, 1]（不再做"排名的排名"）

Phase 3: 加权合成

9. 综合分 = mapped_f1 × w1 + mapped_f2 × w2 + mapped_f3 × w3 + mapped_f4 × w4 + bias(如有)，值域 [0, 1]

Phase 4: 仓位分配

10. 取综合分 Top-N 支 ETF

11. 信心函数: composite < dead_zone% → 0, ≥ full_zone% → 1, 中间 quadratic

12. 总仓位上限 = min(95%, avg_confidence × 1.2)

13. 目标仓位 = 相对权重 × 总仓位上限，按 step 离散化

关键差异（vs 截面标准化）：连续映射保留因子的绝对水平和极端性。牛市时所有因子都高 → 综合分普遍高 → 信心高 → 重仓；熊市时所有因子都低 → 综合分低 → 信心低 → 多留现金。截面标准化则无论牛熊，综合分均值恒为 ~52%，信心函数无法识别市场状态。

📈 F1 周线 EMA 偏离度 w1 ▶

Step 1: 计算 EMA

取周线收盘价序列，用指数移动平均公式计算：

EMA_t = close_t × α + EMA_{t-1} × (1-α)，其中 α = 2/(span+1)，span = ema_period（默认20周）

Step 2: 算偏离度

F1_raw = (最新周收盘价 - EMA值) / EMA值 × 100

结果为百分比，如 +5.2%（价格在均线上方5.2%）或 -3.8%（下方3.8%）

Step 3: sigmoid 连续映射

mapped_f1 = 1 / (1 + exp(-ema_dev / f1_sensitivity))

特性：ema_dev=0 → 0.5（中性），+sensitivity → ~0.73，-sensitivity → ~0.27，极端值自然饱和。

f1_sensitivity：控制"多偏离算极端"。小值(3-5)更敏感——小幅偏离就拉开差距；大值(10-15)更迟钝——需要大偏离才有区分度。默认8，即±8%偏离对应~0.27/0.73。

与截面标准化的核心区别：sigmoid 保留绝对偏离水平。全市场-10%时mapped_f1≈0.13（低分），全市场+10%时≈0.87（高分），信心函数可以识别牛熊。

策略视角：

价值型：低权重(15%)+迟钝尺度(10)，忽略短期趋势 技术型：高权重(40%)+敏感尺度(5)，追趋势核心 景气型：中权重(30%)+默认尺度(8)，趋势辅助
🎯 F2 RSI 异动检测 w2 ▶

设计理念

F2 不再用"健康区"的绝对值评分，而是检测 RSI 相对自身近期均值的异动： 显著超卖 → 博反弹， 显著超买 → 逃顶， 中性区沉默（让 F1/F4 主导）。 这样能让 ETF 在"自身热度涌现"时，F2 自然爆发出超越 F1/F4 的影响力。

Step 1: 计算 RSI 序列

取日线收盘价，标准 RSI 算法：

RS = avg_gain / avg_loss，RSI = 100 - 100/(1+RS)

用 EMA 平滑（α = 1/period），rsi_period 默认 14 日。得到 RSI 序列（每日一个值）。

Step 2: 计算 RSI 偏离 z-score

取最近 rsi_window = 20 日的 RSI 子序列，算自身基线：

μ = mean(RSI[-20:])，σ = std(RSI[-20:])

定义 z-score：z = (current_RSI − μ) / max(σ, 5)

说明：

  • z 表示"今天 RSI 偏离自己最近 20 日均值多少个标准差"
  • 分母 max(σ, 5) 是 sigma 兜底，避免 RSI 异常平稳时 z 爆炸
  • z 最终 clip 到 [-3, +3]，超出视为黑天鹅

这一步的关键洞察：同样是 RSI=80，对长期震荡 RSI 60-70 的强势品种来说不算异常；对长期 RSI 30-40 的弱势品种来说是显著超买。z-score 把"个体基线"考虑进去了。

Step 3: 死区 + 饱和映射

把 z 映射到 [0, 1] 的 F2 分数：

  • |z| < dead_zone（默认 1.0） → 输出 0.5（中性，不投票）
  • z << -dead_zone → 趋向 1.0（博反弹高分）
  • z >> +dead_zone → 趋向 0.0（逃顶低分）

具体公式：

overshoot = |z| - dead_zone

magnitude = 1 - exp(-overshoot)（饱和函数）

F2 = 0.5 ± 0.5 × magnitude（z<0 取 +；z>0 取 −）

Step 4: 涌现效应

因为大多数时候 |z| < 1（统计上正常），F2 输出 0.5 不影响综合分；只有当 ETF 自身确实出现极端 RSI 异动时，F2 才会推到 0.85+ 或 0.15-，把综合分明显拉离 F1+F4 决定的基调。这就是"自身涌现"。

策略视角：

价值型：低权重(10%)，仅极端时干预 技术型：高权重(20-25%)，捕捉异动 景气型：中权重(15-20%)，反弹+逃顶共振
📤 F3 方向性量比 w3 ▶

Step 1: 取窗口数据

取最近 vol_window 日（默认20）的日线，计算每日涨跌 change = pct_change(close)

Step 2: 按涨跌分组

上涨日 = change > 0 的交易日，下跌日 = change < 0 的交易日

Step 3: 算方向性量比

优先用成交额（amount 列），无则退回成交量（volume 列）：

F3_raw = mean(上涨日成交额) / mean(下跌日成交额)

边界：全涨无跌 → 封顶 3.0（强势上限），全跌无涨 → 返回 0（极弱），无跌日且有涨 → 3.0

Step 4: 连续映射（log + sigmoid）

新版采用 log + sigmoid 替代旧的纯指数饱和，让高位（ratio > 5）不再锁死，能继续拉伸：

log_r = log(ratio)（对 ratio < 1 取对称负 log）

mapped_f3 = sigmoid(log_r / f3_sensitivity)

对照样本（f3_sensitivity = 1.0）：

  • ratio = 0.5 → 0.34（量缩，下跌日成交比上涨日大）
  • ratio = 1.0 → 0.50（持平，中性）
  • ratio = 2.0 → 0.66（温和放量）
  • ratio = 5.0 → 0.83（爆量，仍能继续拉伸）
  • ratio = 10.0 → 0.91（极端爆量，区别于一般放量）

核心改进：旧版 ratio > 4 后基本饱和锁死，"温和放量"和"爆量"拉不开差距；新版让真正异动的标的能涌现出来。

vol_window：短窗口(5-10日)捕捉短期放量异动，长窗口(20-40日)看中期资金趋势。

f3_sensitivity：默认 1.0；越小越敏感（高位拉得更陡），越大越平滑。

策略视角：

价值型：低权重(15%)，量能非核心 技术型：高权重(30%)，量价配合核心 景气型：中权重(25%)，量能确认
💰 F4 估值因子 w4 ▶

Step 1: 取当前估值

每支 ETF 有一个主指标（PE-TTM 或 PB），从 csindex 拉取完整历史序列（2010~至今，1400~4000 行/支）。启动时一次性预计算，回测中不再联网。

Step 2: 算历史百分位

百分位 = 历史数据中 ≤ 当前值的占比 × 100

Step 3: 逆向映射

F4_raw = 100 - 百分位。越低估 → 百分位越低 → 分数越高

举例：有色 PE=25，历史 3000 日中 300 个 ≤25 → 百分位 10% → F4_raw = 90

Step 4: 连续映射（替代截面标准化）

F4_raw 已是 0-100（100 - 历史百分位），直接归一化到 [0, 1]：

mapped_f4 = F4_raw / 100 → [0, 1]

不再做"排名的排名"（历史百分位 → 截面排名 = 双重排名）。全市场一起低估（熊市底部）→ F4_raw 普遍高（80+）→ 综合分普遍高 → 信心函数配重仓。这才是连续映射的核心价值：保留绝对水平，让信心函数识别市场状态。

每日更新一次（非实时），波动慢。数据源：csindex 历史PE/PB + X6 东方财富BPS反推。

策略视角：

价值型：核心权重(50%)，低估是信仰 技术型：极低权重(5%)，忽略估值 景气型：中权重(20%)，安全边际
➕ 综合分合成 score ▶

加权求和（连续映射版）

四个因子各自经过连续映射后，加权求和：

综合分 = mapped_f1 × w1 + mapped_f2 × w2 + mapped_f3 × w3 + mapped_f4 × w4

其中 w1+w2+w3+w4 = 1，各 mapped_fi ∈ [0, 1]，所以基础综合分 ∈ [0, 1]（显示时 × 100 为百分比）。

偏好加成（bias）

对标记 bias 的 ETF（有色/创新药/非银），综合分额外加 bias/100（默认 0.04）：

综合分 += bias/100（仅对 bias 标的）

加成在连续映射之后叠加。高 bias → 该 ETF 综合分更高 → 更容易进 Top-N、获得更高仓位。

bias=0 = 纯客观，bias=3-5 = 适度偏好，bias=8+ = 重度倾斜（几乎一定进持仓）。

📐 信心函数 conf ▶

将综合分（[0, 1]）映射为信心系数 [0, 1]，控制"敢配多少仓"。

计算公式（当前仅 quadratic 实际生效）

设 t = (综合分 - dead_zone%) / (full_zone% - dead_zone%)（内部 [0, 1] 尺度）

  • 综合分 < dead_zone% → 信心 = 0（零仓位）
  • 综合分 ≥ full_zone% → 信心 = 1（满配）
  • 中间区域：信心 = t²（quadratic，当前唯一实现）

quadratic 的效果：t=0.3 → 信心=0.09，t=0.5 → 信心=0.25，t=0.8 → 信心=0.64。前期慢后期快 → 保守型，多数分数只给低仓位，只有高分才重仓。

间距调节（显示尺度 0-100%）：dead_zone% 与 full_zone% 的间距 = 过渡带宽度。窄间距(如 30%-55%) → 几乎开关型，宽间距(如 15%-85%) → 渐进型。

注意：前端图表支持 linear / sigmoid 形状预览，但回测引擎当前只实现了 quadratic。linear/sigmoid 选择不影响回测结果。

策略视角：

价值型：宽带(20%-55%), 确信才重仓 技术型：窄带(30%-70%), 跟随信号 景气型：中带(25%-65%), 均衡
⚖ 仓位分配 pos ▶

Step 1: Top-N 选股

取综合分最高的 max_holdings 支 ETF（默认6支）。

Step 2: 相对权重

相对权重_i = 综合分_i / Top-N 综合分之和

综合分越高 → 相对权重越大 → 仓位越重。

Step 3: 总仓位上限

计算 Top-N 的平均信心系数 avg_conf = mean(信心函数(综合分_i))

总仓位上限 = min(0.95, avg_conf × 1.2)

牛市时信心高 → 上限接近95%，熊市信心低 → 上限降低。乘以1.2是放大系数（避免仓位永远偏低），封顶95%留5%现金。

Step 4: 分配仓位

目标仓位_i = 相对权重_i × 总仓位上限

Step 5: 离散化

目标仓位_i = round(目标仓位_i / step) × step

step = discretize_step：5% → 精细调仓，10% → 中等省手续费，15% → 粗放少交易。

例：目标仓位 0.128，step=0.05 → round(0.128/0.05)×0.05 = round(2.56)×0.05 = 3×0.05 = 0.15（15%）

策略视角：

价值型：5支/10%步长/低bias(3), 少动 技术型：6支/5%步长/高bias(5), 灵活 景气型：6支/5%步长/中bias(4), 均衡
