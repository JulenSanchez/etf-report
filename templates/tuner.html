🛠 全流程总览 end-to-end ▶

每个调仓日，对标的池内的全部 ETF 执行以下流水线：

Phase 1: 因子计算（每支 ETF 独立）

1. F1 = 周线EMA偏离度（原始值，如 +5.2%）— 权重 F1 EMA 偏离度

2. F3 = 方向性量比（原始值，如 1.8）— 权重 F3 方向性量比

3. F6 = 动能衰竭惩罚保留率（1.0=无惩罚，0.15=强惩罚）— 权重 F6 动能衰竭惩罚

Phase 2: 连续映射（每支 ETF 独立，不做截面排名）

4. F1 → sigmoid(F1raw / F1 sigmoid 尺度) → [0, 1]

5. F3 → sigmoid(log(ratio) / F3 指数尺度) → [0, 1]

6. F6 → F6score = 保留率 - 1.0 → [0, -0.85] 这类负值惩罚

Phase 3: 加权合成

7. 综合分 = F1 × w1 + F3 × w3 + (F6保留率 - 1) × w6，值域以 F1/F3 为主，F6 只扣分

Phase 4: 仓位分配

8. 取综合分 Top-N 支 ETF

9. 仓位控制: 沪深300周线在MA趋势周期均线上方 → MA上方仓位，下方 → MA下方仓位（二值信号，非综合分驱动）

10. posi = wi × total_pos，按 离散化步长 离散化

关键差异（vs 截面标准化）：连续映射保留因子的绝对水平和极端性。牛市时所有因子都高 → 综合分普遍高 → 仓位由MA信号决定为满仓；熊市时所有因子都低 → 综合分低 → 且MA信号转bear → 自动降仓。仓位控制由市场趋势驱动（MA20），而非自身NAV回撤驱动，避免频繁误触发导致踏空。

📈 F1 周线 EMA 偏离度 w1 ▶

Step 1: 计算 EMA

取周线收盘价序列，用指数移动平均公式计算：

EMAt = closet × α + EMAt-1 × (1-α)，其中 α = 2/(span+1)，span = EMA 周期（默认20周）

Step 2: 算偏离度

F1raw = (closet - EMAt) / EMAt × 100

结果为百分比，如 +5.2%（价格在均线上方5.2%）或 -3.8%（下方3.8%）

Step 3: sigmoid 连续映射

F1 = 1 / (1 + exp(-F1raw / f1_sensitivity))

特性：F1raw=0 → 0.5（中性），+sensitivity → ~0.73，-sensitivity → ~0.27，极端值自然饱和。

F1 sigmoid 尺度：控制"多偏离算极端"。小值(3-5)更敏感——小幅偏离就拉开差距；大值(10-15)更迟钝——需要大偏离才有区分度。默认8，即±8%偏离对应~0.27/0.73。

下图展示不同灵敏度下 F1raw → F1 的映射曲线：

与截面标准化的核心区别：sigmoid 保留绝对偏离水平。全市场-10%时F1≈0.13（低分），全市场+10%时≈0.87（高分），仓位控制可以识别牛熊。

策略视角：

价值型：低权重(15%)+迟钝尺度(10)，忽略短期趋势 技术型：高权重(40%)+敏感尺度(5)，追趋势核心 景气型：中权重(30%)+默认尺度(8)，趋势辅助
🎯 F2 RSI 异动检测 w2 ▶

设计理念

F2 不再用"健康区"的绝对值评分，而是检测 RSI 相对自身近期均值的异动： 显著超卖 → 博反弹， 显著超买 → 逃顶， 中性区沉默（让 F1/F4 主导）。 这样能让 ETF 在"自身热度涌现"时，F2 自然爆发出超越 F1/F4 的影响力。

Step 1: 计算 RSI 序列

取日线收盘价，标准 RSI 算法：

RS = avg_gain / avg_loss，RSI = 100 - 100/(1+RS)

用 EMA 平滑（α = 1/period），RSI 周期 默认 14 日。得到 RSI 序列（每日一个值）。

Step 2: 计算 RSI 偏离 z-score

取最近 20 日的 RSI 子序列，算自身基线：

μ = mean(RSI[-20:])，σ = std(RSI[-20:])

定义 z-score：z = (RSIt − μ) / max(σ, 5)

说明：

  • z 表示"今天 RSI 偏离自己最近 20 日均值多少个标准差"
  • 分母 max(σ, 5) 是 sigma 兜底，避免 RSI 异常平稳时 z 爆炸
  • z 最终 clip 到 [-3, +3]，超出视为黑天鹅

这一步的关键洞察：同样是 RSI=80，对长期震荡 RSI 60-70 的强势品种来说不算异常；对长期 RSI 30-40 的弱势品种来说是显著超买。z-score 把"个体基线"考虑进去了。

Step 3: 死区 + 饱和映射

把 z 映射到 [0, 1] 的 F2 分数：

  • |z| < dead_zone（F2 死区半宽 默认 1.5） → 输出 0.5（中性，不投票）
  • z << -dead_zone → 趋向 1.0（博反弹高分）
  • z >> +dead_zone → 趋向 0.0（逃顶低分）

具体公式：

overshoot = |z| - dead_zone

magnitude = 1 - exp(-overshoot)（饱和函数）

F2 = 0.5 ± 0.5 × magnitude（z<0 取 +；z>0 取 −）

下图为映射函数的可视化（左：博反弹端 / 右：逃顶端）：

Step 4: 涌现效应

因为大多数时候 |z| < 1（统计上正常），F2 输出 0.5 不影响综合分；只有当 ETF 自身确实出现极端 RSI 异动时，F2 才会推到 0.85+ 或 0.15-，把综合分明显拉离 F1+F4 决定的基调。这就是"自身涌现"。

策略视角：

价值型：低权重(10%)，仅极端时干预 技术型：高权重(20-25%)，捕捉异动 景气型：中权重(15-20%)，反弹+逃顶共振
📤 F3 方向性量比 w3 ▶

Step 1: 取窗口数据

取最近 量比窗口 日（默认20）的日线，计算每日涨跌 change = pct_change(close)

Step 2: 按涨跌分组

上涨日 = change > 0 的交易日，下跌日 = change < 0 的交易日

Step 3: 算方向性量比

优先用成交额（amount 列），无则退回成交量（volume 列）：

F3raw = mean(amountup) / mean(amountdown)

边界：全涨无跌 → 封顶 3.0（强势上限），全跌无涨 → 返回 0（极弱），无跌日且有涨 → 3.0

Step 4: 连续映射（log + sigmoid）

新版采用 log + sigmoid 替代旧的纯指数饱和，让高位（F3raw > 5）不再锁死，能继续拉伸：

log_r = log(F3raw)（对 F3raw < 1 取对称负 log）

F3 = sigmoid(log_r / f3_sensitivity)

对照样本（f3_sensitivity = 1.0）：

  • F3raw = 0.5 → 0.34（量缩，下跌日成交比上涨日大）
  • F3raw = 1.0 → 0.50（持平，中性）
  • F3raw = 2.0 → 0.66（温和放量）
  • F3raw = 5.0 → 0.83（爆量，仍能继续拉伸）
  • F3raw = 10.0 → 0.91（极端爆量，区别于一般放量）

核心改进：旧版 F3raw > 4 后基本饱和锁死，"温和放量"和"爆量"拉不开差距；新版让真正异动的标的能涌现出来。

量比窗口：短窗口(5-10日)捕捉短期放量异动，长窗口(20-40日)看中期资金趋势。

F3 指数尺度：默认 1.0；越小越敏感（高位拉得更陡），越大越平滑。

下图展示不同灵敏度下 F3raw → F3 的映射曲线：

策略视角：

价值型：低权重(15%)，量能非核心 技术型：高权重(30%)，量价配合核心 景气型：中权重(25%)，量能确认
💰 F4 估值因子 w4 ▶

Step 1: 取当前估值

每支 ETF 有一个主指标（PE-TTM 或 PB），从 csindex 拉取完整历史序列（2010~至今，1400~4000 行/支）。启动时一次性预计算，回测中不再联网。

Step 2: 算历史百分位

百分位 = 历史数据中 ≤ 当前值的占比 × 100

Step 3: 逆向映射

F4raw = 100 - 百分位。越低估 → 百分位越低 → 分数越高

举例：有色 PE=25，历史 3000 日中 300 个 ≤25 → 百分位 10% → F4raw = 90

Step 4: 连续映射（替代截面标准化）

F4raw 已是 0-100（100 - 历史百分位），直接归一化到 [0, 1]：

F4 = F4raw / 100 → [0, 1]

不再做"排名的排名"（历史百分位 → 截面排名 = 双重排名）。全市场一起低估（熊市底部）→ F4raw 普遍高（80+）→ 综合分普遍高 → MA趋势仓位控制给高仓位。这才是连续映射的核心价值：保留绝对水平，让仓位控制识别市场状态。

每日更新一次（非实时），波动慢。数据源：csindex 历史PE/PB + X6 东方财富BPS反推。

策略视角：

价值型：核心权重(50%)，低估是信仰 技术型：极低权重(5%)，忽略估值 景气型：中权重(20%)，安全边际
⚡ F6 动能衰竭惩罚 w6 ▶

设计理念

F6 专门识别"高位过热后放量下跌"：它不是奖励强势，而是在动能衰竭时给 ETF 扣分，避免刚出现高位破位的标的继续排在前面。

Step 1: 计算触发条件

同时满足三件事才触发：

  • 前一日 RSI ≥ F6 RSI触发门槛（默认80），说明此前已过热
  • 当日跌幅 ≥ F6 跌幅门槛（默认2.5%），说明价格开始破位
  • 当日成交量 / 量比窗口均量 ≥ 1.5，说明下跌伴随放量

Step 2: 输出惩罚保留率

F6保留率 ∈ [base_penalty, 1.0]：1.0 表示无惩罚；触发当日降到 F6 惩罚强度（默认0.15），之后 3 个交易日内线性恢复到 1.0。

Step 3: 转成综合分扣分项

F6score = F6保留率 - 1.0，因此无惩罚时贡献 0；触发后贡献负数，例如 0.15 - 1.0 = -0.85。

综合分扣分 = (F6保留率 - 1.0) × F6 动能衰竭惩罚权重

快照表里的 F6 显示扣分百分比：0 代表无惩罚，-85 代表强惩罚。排序时使用的是上面的负值扣分。

策略视角：

价值型：低权重或关闭，避免过度惩罚低估反弹 技术型：中高权重，重点过滤高位破位 景气型：小权重，作为趋势过热后的刹车
📉 F7 对数收益偏离 w7 ▶

Step 1: 日对数收益率

rt = ln(closet / closet-1)

对数收益可加、中轴天然为 0、涨跌对称。20 日对数收益之和 = ln(closet / closet-20)。

Step 2: 20 日累计对数收益

cumt = Σi=019 rt-i

窗口 = F7 对数收益偏离 权重滑块旁的 20 日（代码常量），≈ 4 个交易周。

Step 3: 滚动 Z-score

对 cumt 序列，回溯 lookback = 250 日计算：

μt = mean(cumt-lookback … cumt-1)

σt = max(std(cumt-lookback … cumt-1), σfloor)

Zt = (cumt - μt) / σt

σfloor = 0.01，防止早期样本过少时 Z 值爆炸。数据不足 60 日 → Zt = NaN，该 ETF 跳过 F7（仅用 F1+F3 打分）。

Step 4: 幂次加速映射

F7 = 0.5 + 0.5 × (−sign(Zt) × (|Zt|/k)t)

参数（奇数幂次 t = 7，标准差倍数 k = 3.0）：

  • Zt = 0 → F7 = 0.50（中性，F1+F3 主导）
  • Zt = +1 → F7 = 0.43（轻度高估，小幅减分）
  • Zt = +3 → F7 = 0.00（3σ高估 → 评分 -100，加速惩罚）
  • Zt = −1 → F7 = 0.57（轻度低估，小幅加分）
  • Zt = −3 → F7 = 1.00（3σ低估 → 评分 +100，加速奖励）
  • Zt = +4 → F7 < 0（4σ → 极端高估，加速惩罚无上限）

凸性设计：Z > 0（高估）→ 凸函数，加速下跌；Z < 0（低估）→ 凹函数，加速上涨。t 越大，两端曲率越陡，极端偏离时加速响应。

无上限惩罚：加速惩罚不截断（用户要求），Z > 3 后 F7 继续下降，对应评分 < −100。等回测结果再决定是否 clip。

Step 5: 加权合成

综合分贡献 = F7 × F7 对数收益偏离权重

F7 是加法因子：最终综合分 = F1×w1 + F3×w3 + F7×w7（+ F6 惩罚项若启用）。与 F6 的减法扣分不同，F7 是双向连续调节。

策略视角：

价值型：中高权重(15-20%)，均值回归是价值投资核心逻辑 技术型：中低权重(5-10%)，辅助逃顶，不对抗主趋势 景气型：中权重(10-15%)，景气拐点时常伴随极端偏离
➕ 综合分合成 score ▶

加权求和（连续映射版）

当前实际生效因子为 F1 / F3 / F7（F6 已移除）：

综合分 = F1 × F1 EMA 偏离度权重 + F3 × F3 方向性量比权重 + F7 × F7 对数收益偏离权重

F7 是连续数值因子，使用幂次加速映射：Z > 0（高估）→ 凸函数加速减分，Z < 0（低估）→ 凹函数加速加分。t 控制曲率，k 控制敏感度。建议与 F1+F3 配合使用。

F2 RSI 自适应和 F4 估值因子已从当前实际计算链路中关闭，保留配置位置仅为未来因子重启预留。

📐 仓位控制 conf ▶

将市场趋势信号直接映射为总仓位比例，控制"整体配多少仓"。

当前策略：MA趋势

极简二值信号：每期检查沪深300周线收盘价是否在自身MA趋势周期均线上方。

  • 沪深300 在周线MA 上方 → 总仓位 = MA上方仓位（默认100%，满仓）
  • 沪深300 在周线MA 下方 → 总仓位 = MA下方仓位（默认40%，降仓防守）

设计理念：高默认仓位 + 极少触发的保护层

回测发现：熊市满仓（不降仓） vs 转现金的收益率差距并不大，说明只要选股本身有抗跌能力，不需要频繁调仓躲避。因此仓位控制的核心不是"精准逃顶"，而是"极少误触发 + 关键时刻提供保护"。

  • 默认满仓（MA上方100%），让选股策略充分运作
  • 仅当MA20翻空时才降仓到40%，避免频繁操作导致踏空
  • 信号频率：约每年切换1-2次，极低频

为什么不用NAV驱动（旧方案已移除）

旧方案（regime / momentum_crash / quadratic）基于自身NAV回撤判断市场状态，问题在于：ETF组合的NAV波动与大盘并非同步，NAV回撤时可能大盘已企稳，NAV上涨时可能大盘已在崩盘。MA20是市场级信号，直接反映趋势，更可靠且不易误触发。

⚖ 仓位分配 pos ▶

Step 1: Top-N 选股

取综合分最高的 最大持仓 支 ETF（默认6支）。

Step 2: 相对权重

wi = scorei / ∑scorej

综合分越高 → 相对权重越大 → 仓位越重。

Step 3: 总仓位

由 仓位控制 策略决定：

MA趋势模式下，沪深300在MA上方 → 总仓位 = MA上方仓位（默认100%），下方 → 总仓位 = MA下方仓位（默认40%）。MA均线周期由MA趋势周期决定（默认20周）。

posi = wi × total_pos

Step 4: 离散化

posi = round(posi / step) × step

step = 离散化步长：5% → 精细调仓，10% → 中等省手续费，15% → 粗放少交易。

例：目标仓位 0.128，step=0.05 → round(0.128/0.05)×0.05 = round(2.56)×0.05 = 3×0.05 = 0.15（15%）

策略视角：

价值型：5支/10%步长, 少动 技术型：6支/5%步长, 灵活 景气型：6支/5%步长, 均衡
