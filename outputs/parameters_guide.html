🟢 每日必须更新的参数 (DAILY UPDATE)

这些参数通过自动脚本每天更新，包含最新的市场数据。

参数名称 存储位置 更新内容 更新频率 触发脚本
K线数据 data/etf_full_kline_data.json OHLCV (收盘价、开盘价、最高、最低、成交量) 每天 update_report.py (Step 1)
MA均线 computed in memory 20日、50日、200日均线数据 每天 fix_ma_and_benchmark.py
实时行情 data/etf_realtime_data.json ETF 涨跌幅、成分股涨跌、成交量 每天 realtime_data_updater.py (Step 4)
日期信息 outputs/index.html 报告日期、数据截止、生成时间 每次更新 update_report.py (Step 5)
一键更新命令： python scripts/update_report.py 自动执行步骤： ✓ Step 1: K线数据获取 (3-5s) ✓ Step 2: MA 均线计算 (1-2s) ✓ Step 3: 基准指数获取 (2-3s) ✓ Step 4: 实时行情获取 (2-3s) ✓ Step 5: HTML 更新 (0.5-1s) ✓ Step 6: 健康检查 (0.5-1s) 总耗时：约 11 秒

🟠 可选调整参数 (季度/半年调整)

这些参数可根据需求灵活调整，不是必须项。

参数名称 配置位置 当前值 调整说明 影响范围

MA 预热周期 kline → daily → warmup_days 19 天 改为 49 计算 50日均线 均线精度

周线显示周数 kline → weekly → display_weeks 52 周 改为 26/104 显示不同周期 周线显示范围
🟡 系统参数 (自动计算，不要手动修改)

这些参数由系统自动计算，修改可能导致错误。

# ❌ 不要手动修改以下字段： # 自动计算字段 fetch_days: 79 # 自动 = display_days (60) + warmup_days (19) fetch_weeks: 71 # 自动 = display_weeks (52) + warmup_weeks (19) # 文件名配置 data_files: kline: "etf_full_kline_data.json" # 固定 realtime: "etf_realtime_data.json" # 固定 fund_flow: "fund_flow_data.json" # 固定 # 系统检查列表 system_check: required_scripts: [...] # 系统维护 required_data_files: [...] # 系统维护

✅ 最佳实践 (DO's)
  • 自动化日常更新 - 设置 cron 任务每天执行 update_report.py
  • 监控健康状态 - 每周运行 health_check.py 确认系统正常
  • 备份配置文件 - 修改前备份 config.yaml
  • 版本控制 - 使用 git 追踪配置变更历史
  • 定期审查日志 - 发现异常及时处理
  • 一次改一个参数 - 便于排查问题
⚠️ 避免这些做法 (DON'Ts)
  • 不要手动修改 K线 JSON - 这会导致数据不一致
  • 不要改动 fetch_days - 这是自动计算的，改了会错
  • 不要删除 API 端点 - 除非官方更新
  • 不要修改文件名 - 脚本依赖这些名称
  • 不要同时改多个参数 - 难以排查问题
  • 不要在 config.yaml 中删除必需字段 - 会导致脚本崩溃
📌 快速参考 - 常见操作

新增 ETF

# 1. 编辑 config/config.yaml，在 etfs 列表中添加： - code: "新代码" name: "新ETF名称" market: "sh" # 或 "sz" benchmark: code: "基准代码" name: "基准名称" # 2. 编辑 config/holdings.yaml，为新 ETF 添加成分股数据 # 3. 下次运行 update_report.py 时自动加载新 ETF

调整显示范围

# 1. 编辑 config/config.yaml kline: daily: display_days: 120 # 改为需要的天数 (如 90, 120, 180) warmup_days: 19 # 通常保持不变 # fetch_days 会自动计算为 120 + 19 = 139 # 2. 保存并下次运行时生效
