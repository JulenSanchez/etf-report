# 数据源与工具生态 · A 股视角

> 这是一份面向**个人投资者、量化爱好者、技术博客作者**的工具全景索引。
>
> 围绕 A 股 / 港股 / ETF / 基金场景，梳理"数据去哪里拿、工具怎么选、合规如何把握"。

---

## 为什么需要这份文档

投资信息工具分散在十几家网站、APP、SDK 里，每家都有自己的长板和局限。想做一件有点技术含量的事情——比如给自己的持仓建个估值看板——往往要在多个来源之间折腾：

- 行情去新浪，财务去东财，估值去集思录，研报去雪球，持仓去天天基金……
- 每个工具都有自己的 ToS（服务条款）、限流规则、数据口径
- 免费与付费的边界常常不清晰

这份文档的目标：**一次查阅，多次复用**。把国内主流工具分三类，每个都给出"定位、数据口径、免费/付费、能做什么、不能做什么"。

---

## 一、数据源分三类

按"**和平台的关系**"来分：

### 类 1：官方授权型（数据商 + SDK）

平台签了协议、按规则授权你访问数据。最合规，但可能付费。

代表：**Tushare Pro**、**聚宽 JoinQuant**、**米筐 RiceQuant**、**Wind 万得**。

### 类 2：开源聚合型（社区维护的 SDK）

第三方开发者把多个公开源聚合成一个 Python 库。合规度**继承于上游**——每个接口对应的源头各不相同。

代表：**AKShare**（国内最活跃）、**tushare 免费版**（历史遗留）、**efinance**（社区小众）。

### 类 3：公开页面爬取型（灰色地带）

没有官方 API，只有面向人类用户的网页/APP。爬取它们的结构化数据属于"技术可行但合规需判断"。

代表：**集思录**、**韭圈儿**（App）、**东方财富**（网页和 API 都有）、**雪球**。

## 二、主流工具逐个介绍

### 💼 数据商与 SDK（类 1 / 类 2）

#### AKShare
- **官网**: https://akshare.akfamily.xyz
- **身份**: 开源 Python 库（pip install akshare）
- **数据源**: 聚合新浪 / 东财 / 中证官网 / 央行 / 国家统计局 / 券商公开 API 等
- **付费**: 完全免费
- **强项**:
  - 覆盖最广（股票 / 基金 / 期货 / 债券 / 外汇 / 宏观 / 新股日历 / 基金拆分 …）
  - Python 一行接入，文档示例丰富
  - 社区活跃，更新频繁
- **短板**:
  - 各接口口径不一（因上游不统一）
  - 部分接口随上游网站变动不稳定
  - **指数 PB 覆盖差**（如中证官网 API 只返回 PE 不返回 PB）
  - 历史深度参差（部分接口 20 天，部分全量）
- **典型用法**:
  ```python
  import akshare as ak
  df = ak.fund_etf_hist_em(symbol="512400", period="daily")
  df = ak.stock_zh_index_value_csindex(symbol="000300")
  ```
- **适合**：任何需要免费、Python 原生的个人项目

#### Tushare Pro
- **官网**: https://tushare.pro
- **身份**: 独立数据商，提供统一口径数据
- **数据源**: 自建数据中心（部分合作上游）
- **付费**: **积分制 + 部分接口单独年费**
  - 120 分（注册+手机认证+资料完善）：`index_basic`、`trade_cal`、基础股票
  - 2000 分（持续调用 / 捐赠约 1:10 即 188 元）：`index_dailybasic`（**含指数 PB**）、财务深度
  - 5000+ 分（高额捐赠 / 付费会员）：分钟级、L2、专业财报
- **强项**:
  - 口径统一（Wind / 中证标准化）
  - 历史完整（从 2005 年起）
  - 稳定（不随免费源变动）
- **短板**:
  - 关键接口有积分门槛
  - 部分高阶接口（L2 / 分钟线）即使 5000 分仍单独付费
- **典型用法**:
  ```python
  import tushare as ts
  ts.set_token("<your-token>")
  pro = ts.pro_api()
  df = pro.index_dailybasic(ts_code="000300.SH", start_date="20240101", end_date="20240401",
                             fields="trade_date,pe_ttm,pb,dv_ratio")
  ```
- **适合**：需要严肃历史回测、指数 PB、财务报表深度的项目

#### 聚宽 JoinQuant
- **官网**: https://www.joinquant.com
- **身份**: 量化研究平台（含 Web IDE + API + 数据）
- **付费**: 免费试用 + 年费（数千元）
- **强项**: 学术口径严谨、因子库完整、回测引擎强
- **短板**: 付费门槛远高于 Tushare
- **适合**: 严肃量化回测工作者

#### 米筐 RiceQuant
- **官网**: https://www.ricequant.com
- **身份**: 同聚宽的量化平台
- **付费**: 免费档 + 付费档
- **适合**: 同聚宽

#### Wind 万得
- **官网**: https://www.wind.com.cn
- **身份**: 中国金融数据终端天花板（类似 Bloomberg）
- **付费**: ¥30000+/年（机构定价）
- **强项**: 数据最全、一级行业分类是事实标准
- **借光方式**: 个人用户通过 AKShare / Tushare **间接拿到** Wind 上游数据（申万/中证一级指数）

#### Bloomberg / Refinitiv / MSCI
- **定位**: 全球金融数据
- **付费**: 机构级
- **个人用户**: 基本无法负担

### 🗞️ 财经门户与社区（类 3，**人类优先**）

这些是普通用户每天在用的网站和 App，也都有一些 AJAX 接口（非官方）被散户抓包利用。

#### 东方财富（东财）eastmoney.com
- **官网**: https://www.eastmoney.com
- **定位**: 国内使用最广的散户金融门户，公司业务覆盖券商 + 数据 + 基金销售
- **目标用户**: 全体散户
- **数据特色**:
  - 行情 API `push2.eastmoney.com/api/qt/...`（非官方，AJAX）
  - 天天基金网 `fundf10.eastmoney.com` 提供基金完整资料（跟踪指数 / 净值 / 持仓 / 业绩比较基准）
  - 基金排行 / 板块排行 / 研报整理
- **免费 or 付费**: 基础数据免费；Choice 终端（专业版）付费
- **合规**: ToS 禁止"大规模商业化再发布"；个人浏览和小规模脚本抓取通常容忍
- **反爬等级**: 中（有限流，但不强）
- **关键工具页**:
  - 天天基金基本资料页：`http://fundf10.eastmoney.com/jbgk_<基金代码>.html`
  - 基金十大持仓：`http://fundf10.eastmoney.com/ccmx_<基金代码>.html`

#### 富途 futunn.com / futubull
- **官网**: https://www.futunn.com
- **定位**: 高端券商 + 社区，UI 极美
- **目标用户**: 港美股玩家、新一代年轻投资者
- **数据特色**: 以美股/港股为主，A 股齐全；深色主题是产品设计标杆
- **免费 or 付费**: 基础免费；Level-2 / 实时推送付费
- **合规**: 严格 ToS，抓数据几乎无法
- **参考价值**: **UI 设计学习样本**（本项目 etf-report 的视觉借鉴对象）

#### 雪球 xueqiu.com
- **官网**: https://xueqiu.com
- **定位**: 社交 + 投资社区 + 自选股
- **目标用户**: 有一定投资经验的散户 / 大 V / KOL
- **数据特色**:
  - 完整行情（A / 港 / 美 / 基金）
  - 个股"估值"面板（PE/PB/ROE/股息率）
  - 讨论区内容丰富（投资大 V 的实盘记录）
- **免费 or 付费**: 基础免费；雪球+ 会员付费（组合跟单等）
- **合规**: ToS 严格，反爬强（sign 签名）
- **人类使用建议**: 关注优秀投资博主、浏览讨论；**不要尝试抓取**

#### 同花顺 10jqka.com.cn / 手机同花顺
- **官网**: https://www.10jqka.com.cn
- **定位**: 券商软件鼻祖，PC 客户端用户最多
- **目标用户**: 全体散户
- **数据特色**: 传统 K 线 + 盘口 + F10 基本面 + 资金流
- **付费**: 基础免费；Level-2 / 投资大师付费
- **特色**: **问财**（iwencai）—— 自然语言选股，"PE 小于 20 且 ROE 大于 15% 的银行股" 直接出结果

#### 新浪财经 finance.sina.com.cn
- **地位**: 老牌门户，K 线数据源鼻祖
- **API**: `hq.sinajs.cn` K 线接口被 AKShare 广泛封装
- **数据特色**: 稳定、历史悠久、文本新闻多
- **短板**: 港股覆盖不全，Web UI 陈旧

#### 腾讯财经 gu.qq.com
- **官网**: https://gu.qq.com
- **定位**: 腾讯自选股的数据后端，前复权 K 线可直接获取
- **API**: `web.ifzq.gtimg.cn/appstock/app/fqkline/get?param={code},{period},,,{count},qfq`
- **数据特色**:
  - 原生前复权（`qfq` 参数），无需自行处理份额变动
  - 沪深 ETF 均可覆盖（`sh510300`/`sz512400` 格式）
  - 日线/周线/月线/分钟线均支持
  - 无需注册、无 Token、无频率限制（低频使用）
- **返回格式**: JSON，日线 `["2026-05-08","2.188","2.206","2.215","2.165","5402754.000"]`（date,open,close,high,low,volume）
- **短板**:
  - 无 `amount`（成交额）字段，需用 `close * volume * 100` 估算
  - 单次请求最大 ~800 条（约 3 年日线），全量历史需分页
  - 沪市 ETF 返回的 key 是 `qfqday` 而非 `day`（深市是 `day`），需适配
  - 不支持服务端日期范围过滤，只能指定返回条数
- **合规**: 低频个人使用通常容忍；高频请求可能被限流
- **反爬等级**: 低（无 sign 签名、无验证码，但短时间大量请求可能触发 IP 封禁）

#### 华尔街见闻 wallstreetcn.com
- **定位**: 快讯 + 深度财经
- **特色**: 全球宏观和大类资产的新闻反应快
- **付费**: 部分深度内容会员付费

这些是**数据的源头**，其他商业平台很多都是从这里拿的。

#### 中证指数有限公司 csindex.com.cn
- **官网**: https://www.csindex.com.cn
- **身份**: 上交所子公司，负责发布中证系列指数（000XXX / 9XXXXX / H3XXXX 等）

## 四、典型场景的工具组合

### 场景 A：个人 ETF 持仓感知（无需商业化、无需精确）
**推荐**：AKShare（日常数据）+ 蛋卷/集思录（估值温度参考）+ 东财天天基金（基金资料）

**工程路径**：Python + AKShare 脚本，日更跑一次，占用时间 5-10 分钟/天。

### 场景 B：严肃回测 / 量化因子研究
**推荐**：Tushare 2000+ 分或聚宽付费 + 自建 PB 聚合 + ROE / PEG / 因子合成

### 场景 C：公开博客/展示
**推荐**：AKShare 为主 + 注明每个数据的来源 + 避免再发布原始数据表格（展示加工后的可视化结果）

### 场景 D：专业选股工具
**推荐**：Wind / 理杏仁 + 同花顺问财（自然语言）+ 雪球（舆情）

## 七、常见误区

### 误区 1：免费 = 数据不准
实际上 AKShare 拿的就是中证官网 / 东财 / 新浪的数据。准确性和付费版本差别不大，差异在"口径统一度"和"历史完整性"。

### 误区 2：Tushare 188 包年
捐赠换积分、积分可获得 2000 分基础接口权限；但 L2、分钟线、专业财报等**部分接口即使有 5000 分仍单独付费**。188 是**关键基础接口的入场券**，不是全功能解锁。

### 误区 3：爬虫 = 违法
技术和法律要分开看：
- **技术上**：大多数网页都能爬
- **合规上**：看 ToS、robots.txt、是否商业化、是否影响网站服务
- **个人自用**且低频请求，**通常法律风险极低**
- **商业化再发布**，**风险显著上升**
- 详细讨论 → [外部数据合规入门](./02-外部数据合规入门.md)

### 误区 4：Wind 才是唯一标准
Wind 确实是 A 股数据天花板，但其**一级行业分类**就是 GICS 翻版，**申万一级** ≈ Wind 一级。通过 AKShare / Tushare 能拿到相同或极接近的数据口径。

## 九、本项目（etf-report）数据源映射

> 本项目当前有 8 个脚本调用外部数据，分布在不同数据源上。数据源问题在此追踪，不再由 REQ-108 管理。

### 9.1 数据源 → 脚本映射

| 脚本 | 用途 | 数据源 | 复权 | amount |
|------|------|--------|------|--------|
| `quant_data_fetcher.py` | 25 支 ETF 日/周线 CSV（量化回测） | **腾讯财经** fqkline | 前复权 ✅ | 估算（close×vol×100） |
| `fix_ma_and_benchmark.py` | 6 支 ETF 日/周线 JSON（正式页 K 线） | 新浪 getKLineData + AKShare fund_etf_hist_sina | 不复权（份额变动清洗管线补偿） | AKShare 补取 |
| `realtime_data_updater.py` | 实时行情（日涨跌 + 成分股） | 新浪 hq.sinajs.cn | — | — |
| `valuation_fetcher.py` | 指数 PE 估值 | AKShare csindex | — | — |
| `stock_bps_fetcher.py` | 个股 PB/BPS → ETF 加权 PB | AKShare stock_value_em | — | — |
| `editorial_fetcher.py` | 新闻/舆情 | 新浪/富途/华尔街见闻/东财搜索 | — | — |
| `corporate_action_source.py` | 基金份额变动检测 | AKShare fund_cf_em | — | — |
| `backfill_csindex_pe.py` | 指数 PE 历史回填（一次性） | AKShare csindex | — | — |

### 9.2 数据源切换历史

| 时间 | 脚本 | 变化 | 触发原因 |
|------|------|------|---------|
| 2026-04 前 | quant_data_fetcher | akshare `fund_etf_hist_sina`（新浪源） | 初始实现 |
| 2026-04-28 | quant_data_fetcher | → 东财 push2his API（前复权） | REQ-185：新浪源不复权，改为东财前复权 |
| 2026-05-07 | quant_data_fetcher | → 腾讯 fqkline API（前复权） | 东财 API IP 被封；腾讯验证可用 |

### 9.3 当前风险点

| 风险 | 影响脚本 | 可能性 | 缓解 |
|------|---------|--------|------|
| 新浪 getKLineData 限流/封 IP | fix_ma_and_benchmark, realtime_data_updater | 中 | 降级为缓存数据 |
| AKShare 上游（东财数据源）不可用 | stock_bps_fetcher, corporate_action_source | 低-中 | 3s 间隔 + 重试 |
| 腾讯 fqkline 限流 | quant_data_fetcher | 低 | 3s 间隔 + 800 条/请求上限 |

### 9.4 两套数据结构差异（待统一）

| 维度 | 正式页（6 支 ETF） | 量化（25 支 ETF） |
|------|-------------------|------------------|
| 格式 | JSON（`etf_full_kline_data.json`） | CSV（`{code}_daily.csv`） |
| 日线字段 | dates, kline[[O,C,L,H]], volumes, amounts, ma5, ma20 | date, open, close, high, low, volume, amount |
| 复权 | 不复权 + 份额变动清洗管线补偿 | 前复权（腾讯 qfq） |
| 量纲 | volumes 单位=股（×100 拉伸） | volume 单位=手 |
| 周线 | 手动聚合（同脚本内） | pandas resample W-FRI |
| 额外数据 | benchmark, etf_normalized, name | 无 |

> **统一方向**：取两者并集，统一为 CSV + JSON 双输出。详见 REQ-194。
